### 1. 판매자 상품 등록 API ⭐⭐⭐

#### 왜 필요한가?
**현재 문제**: 판매자가 자신의 상품을 등록할 방법이 없음

**실제 시나리오**:
```
판매자 김철수씨가 새로운 상품을 판매하려고 함:
1. 판매자 대시보드 접속
2. "상품 등록" 버튼 클릭
3. ❌ API가 없어서 등록 불가능!
4. 😡 관리자에게 요청해야 함 (불편)
```

**API가 있으면**:
```
1. 판매자 대시보드 접속
2. "상품 등록" 버튼 클릭
3. ✅ 상품명, 가격, 이미지, 재고 입력
4. ✅ "등록" 버튼 → 즉시 등록 완료!
5. 😊 바로 라이브 스트리밍 시작 가능
```

**비즈니스 영향**:
- ❌ **없으면**: 판매자가 독립적으로 상품 관리 불가
- ❌ **없으면**: 관리자가 모든 상품을 수동 등록 (운영 부담)
- ✅ **있으면**: 판매자 자율성 증가 → 플랫폼 확장 가능

**필요한 API**:
```typescript
POST /api/seller/products
Header: X-Session-Token: seller_1_xxx
Body: {
  name: "프리미엄 무선 이어폰",
  price: 129000,
  description: "노이즈 캔슬링 기능",
  image_url: "https://...",
  stock: 100,
  category: "전자제품",
  live_stream_id: 2  // 어느 라이브에서 판매할지
}
Response: {
  success: true,
  data: { id: 123, name: "프리미엄 무선 이어폰", ... }
}
```

### 2. 판매자 상품 목록 API ⭐⭐⭐

#### 왜 필요한가?
**현재 문제**: 판매자가 **자신의 상품만** 볼 수 없음

**실제 시나리오**:
```
판매자 김철수씨가 자기 상품을 관리하려고 함:
1. 판매자 대시보드 접속
2. 상품 목록 조회
3. ❌ 현재: 모든 라이브의 모든 상품이 보임 (다른 판매자 상품 포함)
4. 😡 내 상품이 어디 있는지 찾기 어려움
```

**API가 있으면**:
```
1. 판매자 대시보드 접속
2. 상품 목록 조회
3. ✅ 자신이 등록한 상품만 표시
4. ✅ 재고 관리, 가격 수정 쉬움
5. 😊 다른 판매자 상품에 영향 없이 관리 가능
```

**비즈니스 영향**:
- ❌ **없으면**: 판매자가 다른 판매자 상품도 다 보임 (보안/프라이버시 문제)
- ❌ **없으면**: 상품 수정/삭제 시 실수로 다른 사람 상품 건드릴 위험
- ✅ **있으면**: 각 판매자가 자기 상품만 안전하게 관리

**현재 vs 개선**:
```typescript
// ❌ 현재: 모든 상품 조회
GET /api/streams/:streamId/products
→ 스트림 ID를 알아야 하고, 모든 판매자 상품이 섞여 나옴

// ✅ 필요: 내 상품만 조회
GET /api/seller/products
Header: X-Session-Token: seller_1_xxx
Response: {
  success: true,
  data: [
    // 이 판매자가 등록한 상품만
    { id: 1, name: "내 상품 A", seller_id: 1, ... },
    { id: 5, name: "내 상품 B", seller_id: 1, ... }
  ]
}
```

### 3. 주문 상태 변경 API ⭐⭐⭐⭐⭐ (가장 중요!)

#### 왜 필요한가?
**현재 문제**: 주문 후 배송 관리가 불가능!

**실제 시나리오**:
```
고객이 상품을 주문했음:
1. 주문 완료 (결제 완료)
2. 판매자가 상품을 포장하고 발송함
3. ❌ 주문 상태를 "배송중"으로 변경 불가능!
4. ❌ 송장번호를 입력할 수 없음!
5. 😡 고객이 "내 주문 어디까지 왔어요?" 물어봐도 답 못 함
```

**API가 있으면**:
```
1. 주문 완료 (결제 완료) → 상태: "결제완료"
2. 판매자가 상품 포장 → "준비중"으로 변경
3. 택배 발송 → "배송중" + 송장번호 입력
4. 배송 완료 → "배송완료"
5. ✅ 고객이 주문 내역에서 실시간 배송 상태 확인 가능!
6. 😊 "CJ대한통운 1234567890" 클릭하면 배송 추적 가능
```

**주문 상태 플로우**:
```
결제완료 (paid)
    ↓
판매자가 상품 준비 (preparing)
    ↓
발송 완료 (shipped) + 송장번호 입력
    ↓
배송중 (in_transit)
    ↓
배송완료 (delivered)
```

**비즈니스 영향**:
- ❌ **없으면**: 고객이 배송 상태를 모름 → 불안함 → 환불 요청 증가
- ❌ **없으면**: 판매자가 수동으로 고객에게 연락해야 함 (운영 부담)
- ❌ **없으면**: 배송 분쟁 발생 시 증거 없음
- ✅ **있으면**: 자동화된 배송 추적 → 고객 만족도 증가
- ✅ **있으면**: 판매자 운영 효율성 증가

**필요한 API**:
```typescript
// 1. 주문 상태 변경
PUT /api/seller/orders/:orderNo/status
Header: X-Session-Token: seller_1_xxx
Body: {
  status: "shipped"  // preparing, shipped, delivered, cancelled
}

// 2. 송장번호 입력
PUT /api/seller/orders/:orderNo/tracking
Header: X-Session-Token: seller_1_xxx
Body: {
  courier: "CJ대한통운",
  tracking_number: "1234567890"
}
```

**필요한 DB 마이그레이션**:
```sql
-- orders 테이블에 컬럼 추가
ALTER TABLE orders ADD COLUMN courier TEXT;
ALTER TABLE orders ADD COLUMN tracking_number TEXT;
ALTER TABLE orders ADD COLUMN shipped_at DATETIME;
ALTER TABLE orders ADD COLUMN delivered_at DATETIME;
```
