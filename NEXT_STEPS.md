## 🎯 추천: 다음에 구현할 것

### 옵션 1: 셀러 주문 처리 완성 ⏱️ 2-3시간
**왜 중요한가?**
- 셀러가 주문을 처리하지 못하면 배송이 안됨
- 현재 주문 조회만 가능할 가능성

**구현 내용**:
1. 주문 상태 변경 버튼
   - 결제완료 → 상품준비중
   - 상품준비중 → 배송중
   - 배송중 → 배송완료
2. 송장번호 입력 모달
3. 주문 상세 보기

---

### 옵션 2: 실시간 재고 관리 ⏱️ 1-2시간
**왜 중요한가?**
- 품절 상품을 계속 판매하면 환불 폭증
- 라이브 중 재고 소진 시 혼란

**구현 내용**:
1. 상품 재고 수량 체크
2. 장바구니 담기 시 재고 검증
3. 결제 시 재고 차감
4. 품절 시 "품절" 표시
5. 재고 부족 경고 (10개 이하)

---

### 옵션 3: 결제 시스템 검증 ⏱️ 1시간
**왜 중요한가?**
- 실제 결제가 안 되면 매출 불가능
- 현재 Mock일 가능성

**확인 내용**:
1. CheckoutPage 코드 확인
2. 실제 결제 API 호출 여부
3. 결제 승인 후 주문 생성
4. 결제 실패 처리

---

### 옵션 4: 메인 페이지 고도화 ⏱️ 2-3시간
**왜 중요한가?**
- 첫인상이 중요
- 라이브 목록, 인기 상품 등

**구현 내용**:
1. 진행 중인 라이브 목록
2. 예정된 라이브
3. 인기 상품 섹션
4. 검색 기능

---

### 옵션 5: 상품 상세 페이지 ⏱️ 3-4시간
**왜 중요한가?**
- 상품을 자세히 볼 수 없으면 구매율 하락

**구현 내용**:
1. 상품 이미지 갤러리
2. 상품 설명
3. 옵션 선택 (사이즈, 색상)
4. 리뷰 섹션
5. 바로 구매 버튼
