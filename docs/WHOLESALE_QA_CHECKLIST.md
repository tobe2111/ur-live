# 도매몰(유통스타트) 동작 점검(QA) 체크리스트

> "서비스 전체가 실제로 잘 동작하는지"를 **사람이 직접 클릭하며** 확인하는 기능 점검표.
> 각 항목 = [무엇을 한다] → [이렇게 나오면 정상]. 이상하면 ❌ + 메모 → 개발 전달.
>
> 📌 코드 레벨은 자동 테스트 **2200+건 통과**로 검증됨. 이 표는 **실제 화면/실데이터** 동작 확인용
> (자동 테스트가 못 잡는 UI·연동·실결제·권한 체감). 라이브가 막혀 제가 직접 클릭은 못 해, 운영 환경에서 진행.

## 표기
- ✅ 정상 / ❌ 문제(메모) / ⏭️ 자원 미준비(예: 전자계약 키 전)
- 🔴 = 돈/계약/노출 관련 — 문제 시 즉시 중단·보고

## A. 판매사(구매자) — 가입~첫 발주

| # | 무엇을 | 정상 기준 | 체크 |
|---|---|---|---|
| A1 | `/wholesale` 첫 진입(비로그인) | 카탈로그/시작 화면 정상, 스켈레톤 오래 안 걸림 | ☐ |
| A2 | 판매사 가입 신청(사업자번호·등록증 업로드) | "승인 대기" 안내, 어드민에 승인요청 알림 뜸 | ☐ |
| A3 | 잘못된 사업자번호/등록증 누락 | 친절한 오류로 막힘(빈 화면/500 아님) | ☐ |
| A4 | 어드민이 승인 → 판매사 로그인 | 로그인됨, 도매 대시보드 진입 | ☐ |
| A5 | 🔴 카탈로그 상품 카드 | **내 등급가**가 보임, **공급가(원가)·제조사 이름 안 보임** | ☐ |
| A6 | 상품 상세 | 등급가·MOQ·박스단위·재고 정상, 이미지 뜸 | ☐ |
| A7 | 장바구니 담기(MOQ 미만/박스 안 맞게) | 자동보정 또는 안내, 합계 정확 | ☐ |
| A8 | 🔴 발주(선결제) → 결제 | 금액 정확, 결제 성공, 주문 생성, 재고 차감 | ☐ |
| A9 | 🔴 외상(credit) 결제(가능 등급) | 여신 한도 내 정상, 한도 초과 시 막힘 | ☐ |
| A10 | 발주 직후 내 주문/명세서 | 방금 주문이 즉시 보임(빈 화면 아님) | ☐ |

## B. 판매사 — 대량주문(엑셀)

| B2 | 수량 채워 업로드(.xlsx/.csv) | 검증 결과: N개 담김·M개 오류(사유) 표시 | ☐ |

| B4 | 유효분 장바구니 담기 → 결제 | 카트로 이동, 합계 정확, 발주 완료 | ☐ |

## C. 판매사 — 부가 기능

## D. 제조사(공급자) 대시보드

| # | 무엇을 | 정상 기준 | 체크 |
|---|---|---|---|

| D2 | 상품 등록(단건) | 저장됨, 카탈로그 노출(노출등급 정책 반영) | ☐ |
| D3 | 대량 등록(엑셀 템플릿) | 템플릿 다운→업로드→다건 등록 | ☐ |

| D7 | 단가 변경 신청 / 네이버 최저가 대조 | 신청→승인 흐름, 가격대조 표시 | ☐ |
| D8 | 🔴 본인 주문/정산만 보이나(IDOR) | 타 제조사 데이터 안 보임 | ☐ |
