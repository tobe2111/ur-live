## 🧪 2. 테스트 시나리오 (5가지 핵심)

### 🎯 **시나리오 1: 라이브 쇼핑 전체 플로우** (최우선)

**목적**: 일반 사용자가 라이브를 보고 상품을 구매하는 전체 과정 검증

1. **비회원 라이브 시청**
   ```
   1. https://live.ur-team.com 접속
   2. 메인 페이지에서 "LIVE" 배지 있는 스트림 클릭
   3. ✅ 체크: 비회원도 라이브 시청 가능
   4. ✅ 체크: YouTube 영상 자동 재생
   5. ✅ 체크: 실시간 채팅 보임
   6. ✅ 체크: 시청자 수 표시됨
   ```

2. **회원가입 및 로그인**
   ```
   1. 우측 상단 "로그인" 클릭
   2. "회원가입" 클릭
   3. 이메일/비밀번호 입력 후 가입
   4. ✅ 체크: 회원가입 성공 메시지
   5. 로그인 페이지에서 로그인
   6. ✅ 체크: 메인 페이지로 리다이렉트
   ```

3. **장바구니 담기**
   ```
   1. 라이브 페이지 재접속
   2. 하단 상품 정보에서 "담기" 버튼 클릭
   3. 상품 시트에서 수량 선택 후 "장바구니 담기"
   4. ✅ 체크: "✅ 장바구니에 1개 담았습니다" 토스트
   5. 우측 상단 장바구니 아이콘에 빨간 점 표시
   ```

4. **결제 진행**
   ```
   1. 장바구니 아이콘 클릭 → /cart 이동
   2. 상품 확인 후 "주문하기" 클릭
   3. 배송지 선택 (없으면 "배송지 추가")
   4. ✅ 체크: 배송지 선택 필수
   5. "결제하기" 버튼 클릭
   6. ✅ 체크: 버튼이 "결제 처리중"으로 변경 (disabled)
   7. Toss 결제 위젯에서 카드 정보 입력
   8. 결제 승인
   9. ✅ 체크: /payment/success 페이지로 이동
   10. ✅ 체크: 주문 번호 표시
   ```

5. **주문 내역 확인**
   ```
   1. 마이 페이지 → "주문 내역" 클릭
   2. ✅ 체크: 방금 주문한 내역 표시
   3. ✅ 체크: 주문 상태 "결제 완료" 표시
   ```

**예상 소요 시간**: 5~7분

---

### 🎯 **시나리오 2: 셀러 라이브 컨트롤** (핵심 신규 기능)

**목적**: 셀러가 라이브 중 상품을 변경하고 시청자 화면에 반영되는지 검증

1. **셀러 로그인**
   ```
   1. https://live.ur-team.com/seller/login 접속
   2. seller@ur-team.com / 비밀번호 입력
   3. ✅ 체크: 셀러 대시보드로 이동
   ```

2. **라이브 스트림 생성 (이미 있으면 Skip)**
   ```
   1. 대시보드 → "라이브 관리" → "새 라이브 만들기"
   2. 제목, YouTube Video ID 입력
   3. 상품 2개 이상 연결
   4. ✅ 체크: 라이브 생성 성공
   ```

3. **셀러가 라이브 페이지 접속**
   ```
   1. 대시보드 → "라이브 보기" 클릭
   2. 본인 라이브 페이지로 이동
   3. ✅ 체크: "소개 중" 보라색 배지 표시 (상단 좌측)
   4. ✅ 체크: 하단 "🔄 변경하기" 버튼 표시 (일반 유저는 "구매하기")
   ```

4. **상품 변경 테스트**
   ```
   1. 릴카드를 스와이프하여 다른 상품으로 이동
   2. "🔄 변경하기" 버튼 클릭
   3. ✅ 체크: 버튼이 "⏳ 전환 중..."으로 변경
   4. ✅ 체크: 2초 후 버튼이 "✅ 소개 중"으로 변경
   5. ✅ 체크: "소개 중" 배지가 표시됨
   ```

5. **시청자 화면 동기화 확인**
   ```
   1. 다른 브라우저 (시크릿 모드)에서 같은 라이브 페이지 접속
   2. 일반 유저로 로그인
   3. ✅ 체크: 3초 이내에 변경된 상품 정보 표시
   4. ✅ 체크: 상품 이름, 가격이 변경됨
   ```

**예상 소요 시간**: 3~5분

### 🎯 **시나리오 3: 재고 예약 시스템 (Race Condition 방지)**

**목적**: 동시 결제 시도 시 오버셀링이 발생하지 않는지 검증

**⚠️ 주의**: 이 테스트는 **재고 1개 상품**으로 진행해야 합니다.

1. **테스트 상품 준비**
   ```
   1. 셀러 로그인 → 상품 등록
   2. 상품명: "재고 테스트 상품"
   3. 재고: 1개 (중요!)
   4. 가격: 10,000원
   5. 라이브에 연결
   ```

2. **2명의 사용자 동시 주문 시도**
   ```
   # 브라우저 A (사용자 A)
   1. 라이브 페이지 접속
   2. "재고 테스트 상품" 장바구니 담기
   3. 결제 페이지로 이동
   4. 배송지 선택
   5. "결제하기" 버튼 클릭 준비 (⚠️ 아직 누르지 말 것)
   
   # 브라우저 B (사용자 B)
   1. 동일하게 장바구니 담기
   2. 결제 페이지로 이동
   3. 배송지 선택
   4. "결제하기" 버튼 클릭 준비
   
   # 동시 클릭 (5초 이내)
   1. 사용자 A, B가 거의 동시에 "결제하기" 클릭
   2. ✅ 체크: 한 명만 결제 성공
   3. ✅ 체크: 다른 한 명은 "죄송합니다. 방금 상품이 모두 판매되었습니다." 에러
   ```

3. **재고 확인**
   ```
   1. 셀러 대시보드 → 상품 관리
   2. "재고 테스트 상품" 재고 확인
   3. ✅ 체크: 재고 0개 (정확히 1개만 차감)
   4. ✅ 체크: reserved_stock: 0개
   ```

**예상 소요 시간**: 5분

### 🎯 **시나리오 4: 결제 실패 시 재고 복구**

**목적**: 결제 실패 또는 취소 시 예약된 재고가 정상적으로 복구되는지 검증

1. **테스트 상품 준비**
   ```
   1. 셀러 로그인 → 상품 등록
   2. 상품명: "롤백 테스트 상품"
   3. 재고: 5개
   4. 라이브에 연결
   ```

2. **주문 생성 (결제 전)**
   ```
   1. 일반 유저로 로그인
   2. "롤백 테스트 상품" 2개 장바구니 담기
   3. 결제 페이지로 이동
   4. ✅ 체크: 이 시점에서 재고 예약됨 (stock: 5, reserved_stock: 2)
   ```

3. **결제 취소**
   ```
   # 방법 1: 브라우저 닫기
   1. 결제 페이지에서 브라우저 닫기 (10분 대기)
   2. 10분 후 재고 확인
   3. ✅ 체크: reserved_stock이 자동으로 0으로 복구
   
   # 방법 2: 결제 실패
   1. Toss 결제 위젯에서 잘못된 카드 정보 입력
   2. 결제 실패 → /payment/fail 페이지
   3. ✅ 체크: 즉시 reserved_stock 0으로 복구
   ```

4. **재고 확인**
   ```
   1. 셀러 대시보드 → 상품 관리
   2. "롤백 테스트 상품" 재고 확인
   3. ✅ 체크: stock: 5개 (변동 없음)
   4. ✅ 체크: reserved_stock: 0개
   ```

**예상 소요 시간**: 3분 (또는 10분 + 대기 시간)

   3. ✅ 체크: 관리자 대시보드로 이동
   ```

2. **셀러 승인 테스트**
   ```
   1. 새 브라우저에서 셀러 회원가입
   2. 사업자 정보 입력 후 가입
   3. ✅ 체크: "승인 대기 중" 메시지
   4. 어드민 대시보드 → "셀러 관리"

   6. ✅ 체크: 상태 "pending"

   8. ✅ 체크: 상태가 "approved"로 변경
   9. 셀러 계정으로 다시 로그인

4. **전체 통계 확인**

## 📞 6. 긴급 문제 발생 시 대응

### 🔥 **재고 오버셀링 발생 시**

**증상**: 재고 1개인데 2명 이상 주문 성공

**원인**: DB 마이그레이션 미적용

**조치**:
```bash
# 1. 즉시 라이브 중단
# 2. DB 마이그레이션 적용
npx wrangler d1 migrations apply toss-live-commerce-db --remote

# 3. 중복 주문 확인
SELECT o.order_number, oi.product_id, oi.quantity, p.stock
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id
WHERE p.stock < 0;

# 4. 고객 연락 (재고 부족 안내)
```

### 🔥 **결제 시스템 오류 시**

**증상**: 결제 버튼 클릭 시 에러

**원인**: Toss Secret Key 누락 또는 만료

**조치**:
```bash
# 1. 환경 변수 확인
npx wrangler pages secret list --project-name ur-live

# 2. Secret Key 재설정
npx wrangler pages secret put TOSS_SECRET_KEY --project-name ur-live

# 3. 재배포
npx wrangler pages deploy dist --project-name ur-live
```

### 🔥 **JWT 인증 오류 시**

**증상**: 로그인 후 401 Unauthorized

**원인**: JWT_SECRET 누락 또는 불일치

**조치**:
```bash
# 1. JWT Secret 확인
npx wrangler pages secret list --project-name ur-live

# 2. JWT Secret 재설정 (32자 이상 랜덤 문자열)
npx wrangler pages secret put JWT_SECRET --project-name ur-live

# 3. 모든 사용자 재로그인 필요
```
