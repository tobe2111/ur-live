## 📋 사전 준비

### 필요한 것
- ✅ 브라우저 (Chrome 또는 Firefox 권장)
- ✅ Kakao 계정 (로그인용)
- ✅ NicePay 테스트 카드 정보:
  - 카드번호: `5465-7801-2345-6789`
  - 유효기간: `12/25`
  - CVV: `123`
  - 생년월일: `900101`
  - 비밀번호 앞 2자리: `00`

1. F12 키를 눌러 개발자 도구 열기
2. **Console** 탭과 **Network** 탭을 사용합니다
3. **Network** 탭에서 "Preserve log" 체크

**확인 사항:**
- [ ] Kakao 로그인 페이지 정상 로딩
- [ ] 로그인 성공 후 원래 페이지로 돌아옴
- [ ] 우측 상단에 사용자 이름 표시
- [ ] "로그아웃" 버튼 표시

**콘솔 확인:**
```javascript
// Console 탭에서 다음 메시지 확인
```

```javascript
// Console에서 실행
localStorage.getItem('userId')      // "10"

localStorage.getItem('session')     // "user_10_..."
```

### Phase 4: 장바구니 기능 (5분)

#### 4.1 상품을 장바구니에 추가
```
1. 하단 상품 목록에서 첫 번째 상품 선택
2. "장바구니에 추가" 버튼 클릭
3. 알림 메시지 확인: "장바구니에 추가되었습니다!"
4. 다른 상품도 2-3개 추가
```

**확인 사항:**
- [ ] 버튼 클릭 시 알림 표시
- [ ] 우측 상단 장바구니 아이콘에 숫자 배지 표시 (예: "3")

**Network 탭 확인:**
```
POST /api/cart
Request Body:
{
  "userId": 10,
  "productId": 1,
  "quantity": 1
}

Response:
{
  "success": true,
  "data": { "cartItemId": 123 }
}
```

#### 4.2 장바구니 페이지로 이동
```
1. "결제하기" 또는 장바구니 아이콘 클릭
2. /cart 페이지로 이동
```

**확인 사항:**
- [ ] 장바구니에 추가한 상품들이 목록에 표시
- [ ] 각 항목에 상품명, 가격, 수량, 소계 표시
- [ ] 수량 변경 버튼 (+ / -) 작동
- [ ] 삭제 버튼 작동
- [ ] 하단에 소계, 배송비(3,000원), 총액 표시

**수량 변경 테스트:**
```
1. 수량 증가 버튼(+) 클릭
2. 소계 금액 업데이트 확인
3. 수량 감소 버튼(-) 클릭
4. 소계 금액 업데이트 확인
```

**삭제 테스트:**
```
1. 한 항목의 "삭제" 버튼 클릭
2. 해당 항목이 목록에서 제거됨
3. 총액 재계산 확인
```

### Phase 5: 결제 플로우 (10분)

#### 5.1 배송지 정보 입력
```
1. 장바구니 페이지 하단 "배송지 정보" 섹션
2. 다음 정보 입력:
   - 받는 사람: 홍길동
   - 전화번호: 010-1234-5678
   - 주소: 서울시 강남구 테헤란로 123
   - 상세주소: 101동 1001호
   - 우편번호: 06234
```

**확인 사항:**
- [ ] 모든 필드 입력 가능
- [ ] 전화번호 형식 검증 (숫자만 입력)

#### 5.2 결제 시작
```
1. "결제하기" 버튼 클릭
2. 주문 생성 API 호출 대기
```

**Network 탭 확인:**
```
POST /api/orders/create
Request Body:
{
  "userId": 10,
  "items": [{ "productId": 1, "quantity": 1, ... }],
  "shippingAddress": { ... },
  "totalAmount": 56400
}

Response:
{
  "success": true,
  "data": {
    "orderNo": "ORD-1738836000-ABC123",
    "totalAmount": 56400
  }
}
```

**확인 사항:**
- [ ] 주문 번호(orderNo) 생성됨
- [ ] 총액(totalAmount)이 일치함

#### 5.3 NicePay 결제창
```
1. NicePay 결제창 팝업 열림
2. 주문 정보 확인:
   - 주문번호: ORD-...
   - 상품명: 프리미엄 겨울 패딩 외 N건
   - 금액: 56,400원
```

**확인 사항:**
- [ ] 팝업이 정상적으로 열림
- [ ] 주문 정보 정확하게 표시
- [ ] "카드 결제" 선택됨

**콘솔 확인:**
```javascript
// Console에서 확인
NICEPAY_CLIENT_ID: R2_b5b7a7a3bcce40b19147228bf3f5c50e
✅ AUTHNICE 로딩 완료
```

#### 5.4 테스트 카드 입력 및 결제
```
1. 결제 수단: 카드
2. 카드 정보 입력:
   - 카드번호: 5465-7801-2345-6789
   - 유효기간: 12/25
   - CVV: 123
   - 생년월일: 900101
   - 비밀번호 앞 2자리: 00
3. "결제" 버튼 클릭
4. (필요시) 3D Secure 인증 완료
```

**확인 사항:**
- [ ] 카드 정보 입력 가능
- [ ] "결제" 버튼 활성화
- [ ] 결제 처리 중 로딩 표시

### Phase 6: 결제 결과 처리 (3분)

#### 6.1 결제 결과 페이지
```
1. 결제 완료 후 /payment-result 페이지로 리다이렉트
2. URL 파라미터 확인
```

**URL 예시:**
```
https://live.ur-team.com/payment-result?
  resultCode=0000
  &tid=nictest00m01012501061738838240082
  &orderId=ORD-1738836000-ABC123
  &amount=56400
```

**확인 사항:**
- [ ] resultCode = "0000" (성공)
- [ ] tid (거래 ID) 존재
- [ ] orderId가 주문 번호와 일치
- [ ] amount가 총액과 일치

**콘솔 확인:**
```javascript
✅ Result Code: 0000
✅ TID: nictest00m01012501061738838240082
✅ Order ID: ORD-1738836000-ABC123
✅ Amount: 56400
```

```
Network 탭에서 확인:

POST /api/payments/nicepay/callback
Request Body:
{

  "authResultMsg": "성공",
  "tid": "nictest00m...",

  "amount": 56400,
  ...
}

Response:
{
  "success": true,
  "message": "결제 승인이 완료되었습니다",
  "data": {
    "order_number": "ORD-...",
    "payment_key": "nictest00m...",
    "amount": 56400
  }
}
```

**확인 사항:**
- [ ] API 응답 성공 (success: true)
- [ ] 결제 승인 완료 메시지

#### 6.3 주문 완료 페이지


**확인 사항:**
- [ ] **주문 정보**:

  - 상품 금액

- [ ] **주문 취소 버튼**:
