## 🔄 셀러가 할 수 있는 작업

### 1. **주문 상태 변경**
**워크플로우:**
```
결제완료 → 준비중 → 배송중 → 배송완료
         ↘ 취소
```

**버튼:** "준비중으로 변경" → "배송중으로 변경" → "배송완료로 변경"

### 2. **배송 정보 입력**
**입력 필드:**
- 택배사 (예: CJ대한통운, 우체국택배)
- 송장번호 (예: 1234567890)

**효과:**
- 구매자가 배송 추적 가능
- 주문 상세에 택배사/송장번호 표시

## 🗂️ 데이터베이스 구조

### Orders 테이블
```sql
CREATE TABLE orders (
  id INTEGER PRIMARY KEY,
  order_number TEXT UNIQUE,      -- 주문번호
  user_id INTEGER,                -- 구매자 ID
  total_amount REAL,              -- 총 주문금액
  status TEXT,                    -- 주문상태
  payment_status TEXT,            -- 결제상태
  shipping_name TEXT,             -- 수령인 이름 ✅
  shipping_phone TEXT,            -- 수령인 연락처 ✅
  shipping_address TEXT,          -- 배송지 주소 ✅
  courier TEXT,                   -- 택배사
  tracking_number TEXT,           -- 송장번호
  created_at DATETIME,
  updated_at DATETIME
)
```

### Order Items 테이블
```sql
CREATE TABLE order_items (
  id INTEGER PRIMARY KEY,
  order_id INTEGER,               -- 주문 ID
  product_id INTEGER,             -- 상품 ID
  seller_id INTEGER,              -- 판매자 ID
  quantity INTEGER,               -- 수량 ✅
  price REAL,                     -- 개당 가격 ✅
  FOREIGN KEY (order_id) REFERENCES orders(id),
  FOREIGN KEY (product_id) REFERENCES products(id),
  FOREIGN KEY (seller_id) REFERENCES sellers(id)
)
```

## 🎯 PG 연동 후 완전한 플로우

```
[소비자]
1. 라이브 방송 시청
2. 상품 장바구니 담기
3. 주문하기
4. 배송 정보 입력
   - 수령인: 김영희 ✅
   - 연락처: 010-1234-5678 ✅
   - 주소: 서울시 강남구... ✅
5. 결제하기 (PG 연동)
   ↓
   [DB에 저장]
   - orders 테이블: 주문 정보, 배송 정보
   - order_items 테이블: 주문 상품 목록
   ↓
[셀러]
6. 주문 관리 페이지 접속
7. 신규 주문 확인
   ✅ 주문 상품 목록 (이미지, 수량, 가격)
   ✅ 수령인 정보
   ✅ 배송지 주소
   ✅ 연락처
   ✅ 총 주문금액
8. 준비중으로 변경
9. 배송 정보 입력
10. 배송중으로 변경
11. 배송완료로 변경
```
