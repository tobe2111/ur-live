## 🎯 시스템 개요

### 플랫폼 아키텍처
- **프론트엔드**: React 18 + TypeScript + Vite
- **백엔드**: Cloudflare Workers (Hono Framework)
- **데이터베이스**: Cloudflare D1 (SQLite)
- **인증**: Firebase Auth (구매자), JWT (판매자/관리자)
- **결제**: PortOne (한국), Stripe (글로벌)
- **라이브**: YouTube 임베드 + Cloudflare KV (채팅)
- **알림**: Kakao Alimtalk, Resend Email API

### 주요 기능
1. **구매자**: 회원가입/로그인, 상품 검색, 장바구니, 주문/결제, 라이브 시청
2. **판매자**: 상품 등록/관리, 주문 관리, 라이브 방송, 정산 조회
3. **관리자**: 판매자 승인, 주문 모니터링, 배너 관리, 정산 대시보드

## 👨‍💼 관리자 가이드

### 1️⃣ 관리자 로그인
```
URL: https://live.ur-team.com/admin/login
계정: admin@ur-team.com
비밀번호: admin123
```

### 2️⃣ 판매자 승인 프로세스

#### 승인 대기 중인 판매자 확인
1. **관리자 대시보드** 접속
2. **"판매자 승인 대기"** 섹션 확인
3. 각 판매자의 사업자 정보 검토:
   - 이메일
   - 사업자명
   - 사업자등록번호
   - 연락처

#### 판매자 승인하기
1. 승인 대기 목록에서 **"승인"** 버튼 클릭
2. 자동으로 상태가 `approved`로 변경
3. 판매자에게 승인 완료 이메일 자동 발송
4. 판매자는 즉시 로그인하여 상품 등록 가능

#### 판매자 거부하기
1. 승인 대기 목록에서 **"거부"** 버튼 클릭
2. 거부 사유 입력 (필수)
3. 확인 후 거부 완료
4. 판매자에게 거부 사유 포함 이메일 발송

### 3️⃣ 주문 관리
```
URL: https://live.ur-team.com/admin/orders
```

## 🏪 판매자 가이드

### 1️⃣ 판매자 회원가입
```
URL: https://live.ur-team.com/seller/register
```

#### 가입 절차
1. **필수 정보 입력**:
   - 이메일
   - 비밀번호 (최소 6자)
   - 이름
   - 연락처
   - 사업자명
   - 사업자등록번호 (선택)
2. **가입 완료** → 관리자 승인 대기 상태
3. **승인 완료 이메일 수신** → 로그인 가능

### 2️⃣ 판매자 로그인
```
URL: https://live.ur-team.com/seller/login
계정: tobe2111@naver.com
비밀번호: 358533aa!!
```

### 3️⃣ 상품 등록하기
```
URL: https://live.ur-team.com/seller/products/new
```

#### 기본 정보
- **상품명**: 고객에게 보이는 상품 이름
- **가격**: 판매 가격 (원)
- **재고**: 초기 재고 수량
- **카테고리**: 의류, 전자기기, 식품 등
- **상품 설명**: 상세 설명 (HTML 지원)

#### 이미지 등록
- **메인 이미지**: 썸네일로 사용될 대표 이미지
- **추가 이미지**: 상세 페이지 이미지 (최대 10개)

#### 옵션 설정 (선택)
예시: 사이즈(S, M, L), 색상(빨강, 파랑, 검정)
1. **옵션명** 입력 (예: "사이즈")
2. **옵션값** 추가 (예: "S", "M", "L")
3. 각 옵션별 **추가 금액** 및 **재고** 설정

### 4️⃣ 주문 관리
```
URL: https://live.ur-team.com/seller/orders
```

#### 주문 상태 관리
1. **결제완료** → 새 주문 확인
2. **배송준비중** → 상품 포장 및 송장 등록
3. **배송중** → 택배사 + 송장번호 입력
4. **배송완료** → 고객 수령 확인
5. **취소/환불** → 관리자 문의

#### 송장 등록하기
1. 주문 목록에서 주문 클릭
2. **"송장 등록"** 버튼
3. 택배사 선택
4. 송장번호 입력
5. 저장 → 고객에게 자동 알림 발송

2. **라이브 상품 선택**:

## 🚨 트러블슈팅

### 1. 로그인이 안 됩니다

#### 판매자/관리자 로그인 실패
**증상**: "이메일 또는 비밀번호가 일치하지 않습니다"

**원인 및 해결**:
1. **계정이 승인되지 않음**
   - 판매자: 관리자에게 승인 요청
   - 관리자: DB에서 `admins` 테이블 확인
   
2. **비밀번호 오류**
   - 비밀번호 재설정 기능 이용
   - 또는 관리자가 DB에서 직접 재설정

3. **JWT_SECRET 미설정**
   ```bash
   # Cloudflare Pages 환경 변수 확인
   JWT_SECRET이 설정되어 있는지 확인
   ```

#### 구매자 Firebase 로그인 실패
**증상**: "로그인 중 오류가 발생했습니다"

**해결**:
1. Firebase Console에서 Authentication 활성화 확인
2. 도메인 허용 목록에 `live.ur-team.com` 추가
3. 브라우저 쿠키/캐시 삭제

### 2. 결제가 안 됩니다

#### PortOne 결제 실패
**증상**: 결제 버튼 클릭 후 오류 발생

**확인 사항**:
1. **PortOne API 키 확인**
   - 환경 변수 `PORTONE_API_KEY` 설정 확인
   
2. **결제 금액 확인**
   - 최소 결제 금액: 100원
   - 최대 결제 금액: 10,000,000원

3. **상품 재고 확인**
   - 재고 부족 시 결제 불가

### 3. 이미지가 표시되지 않습니다

**원인**:
- 이미지 URL이 HTTPS가 아님
- 이미지 URL이 만료됨
- CORS 문제

**해결**:
```bash
# Cloudflare Images 사용 권장
1. 이미지를 Cloudflare Images에 업로드
2. 반환된 URL을 상품 이미지로 사용
```

### 4. 라이브 방송이 재생되지 않습니다

**원인**:
- YouTube 동영상 ID가 잘못됨
- YouTube 동영상이 비공개 상태
- 임베드 허용이 꺼져 있음

**해결**:
1. YouTube 동영상 설정 → **공개** 또는 **일부 공개**
2. YouTube 동영상 설정 → **임베드 허용** 체크
3. 올바른 동영상 ID 확인 (예: `dQw4w9WgXcQ`)

### 5. 알림톡이 발송되지 않습니다

**확인 사항**:
1. **Kakao API 키 확인**
   - 환경 변수 `KAKAO_API_KEY` 설정 확인
   
2. **발신 프로필 승인 확인**
   - 카카오비즈니스 채널 승인 완료 확인

3. **템플릿 승인 확인**
   - 사용하려는 템플릿이 승인되었는지 확인

## 🔒 보안 정책

### 비밀번호 정책
- **최소 길이**: 6자 (권장 8자 이상)
- **비밀번호 해싱**: bcrypt (강도 10)
- **비밀번호 변경**: 3개월마다 권장
- **재사용 금지**: 이전 3개 비밀번호 재사용 불가

### 세션 관리
- **Access Token 유효기간**: 15분
- **Refresh Token 유효기간**: 7일
- **자동 로그아웃**: 비활동 30분 후

### API 보안
- **Rate Limiting**: IP당 분당 100 요청
- **CORS**: 허용된 도메인만 접근 가능
- **CSRF Protection**: 모든 POST 요청에 토큰 필요
- **SQL Injection 방어**: 파라미터화된 쿼리 사용

### 데이터 보호
- **개인정보 암호화**: AES-256
- **카드 정보**: PCI-DSS 준수 (PortOne 처리)
- **로그 보관**: 90일간 보관 후 자동 삭제
- **백업**: 주간 자동 백업
