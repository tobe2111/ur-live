# 오프라인 공구(동네딜) 소개서 — 콘텐츠 브리프 (디자인 AI 입력용)

> **이 문서의 역할**: 디자인 AI가 "오프라인 공구(동네딜/맛집 공동구매) 소개서"를 레이아웃할 때 쓸 **정확한 소스 원고**.
> 디자인(시각·컬러·폰트)이 아니라 **사실/카피/구조**가 핵심. 모든 수치는 실제 코드/설정에서 추출했고, 불확실한 값은 `[확인 필요]`로 표시.
> 작성일: 2026-06-07 · 근거: 코드 audit (group-buy / community-group-buy / seller-roles / tax-withholding / CLAUDE.md)

`[디자인 지시]` 블록 = 디자이너에게 주는 레이아웃 힌트. 본문 카피는 그대로 쓸 수 있는 한국어 완성문.

## ⚠️ 디자인 AI가 먼저 알아야 할 핵심 사실 2가지 (오해 방지)

이 플랫폼에는 **이름이 "공동구매"인 흐름이 2개** 있고, 경제 구조가 완전히 다르다. 소개서에서 섞으면 안 된다.

| 구분 | (A) 셀러/매장 공구 = **즉시판매형 동네딜** | (B) 커뮤니티(유저) 공구 = **맛집 공동구매 제안** |
|---|---|---|
| 누가 시작 | 매장 사장님(store_owner) 또는 크리에이터(influencer) | 일반 소비자(검증된 활동 인플루언서/셀러만 등록 가능) |
| 결제 | 즉시 전액 결제 → **교환권 즉시 발급** | 1인당 **딜 보증금 예치**(최소 1,000딜) |
| 가격 | 인원 무관 **단일 공구가 즉시 적용**(`maxTierDiscount`) | 희망가 제안 → 목표 인원 모이면 매장과 가격 확정 |
| 목표 미달 시 | **패널티 없음**(이미 유효한 교환권) | **보증금 자동 전액 환불** |
| 코드 근거 | `helpers.maxTierDiscount`, `docs/design/groupbuy-instant-sale.md` | `community-group-buy.routes.ts`, `UserGroupBuyCreatePage.tsx` |

> **이 소개서의 주인공은 (A) 즉시판매형 동네딜**(매장 사장님 가치제안의 중심). (B)는 §7에서 "소비자가 직접 맛집을 제안하는 보조 흐름"으로 짧게 다룬다.
> `[디자인 지시]` (A)와 (B)를 한 슬라이드에 섞지 말 것. (B)는 부록/보조 카드로.

## 1. 문서 목적 & 타깃

**목적**: 오프라인 매장 사장님에게 "유어딜 동네딜(오프라인 공동구매)에 입점하면 왜 이득인지"를 설득하고, 소비자에게 "왜 동네딜로 사면 싸고 안전한지"를 알린다. 영업자료 + 온보딩 안내서 겸용.

| 타깃 | 정의(코드상 role) | 핵심 니즈 |
|---|---|---|
| **주 타깃 — 매장 사장님** | `seller_type = 'store_owner'` (또는 `'both'`) | 신규 고객 확보, 빈 시간·공실 채우기, 광고비 절감, 선결제로 노쇼 줄이기 |
| 부 타깃 — 크리에이터 | `seller_type = 'influencer'` | 동네 맛집 홍보로 커미션 적립 |
| 부 타깃 — 소비자 | 일반 user | 동네 맛집/뷰티/숙소를 검증된 최저가로 |

**업종(카테고리) — 코드 확정 4종** (`voucher-categories.ts`):

| 카테고리 코드 | 라벨 | 이모지 | 포함 업종 |
|---|---|---|---|
| `meal_voucher` | 식사권 (음식점/카페) | 🍽️ | 맛집·카페·디저트 |
| `beauty_voucher` | 미용 (헬스/뷰티) | 💇 | 미용실·네일·피부·헬스·필라테스 |
| `stay_voucher` | 숙소 | 🏨 | 호텔·펜션·게스트하우스 |
| `etc_voucher` | 기타 | 🎯 | 펫·액티비티·체험 등 |

> 과거 6종(meal/beauty/health/pet/stay/activity) → 현재 4종으로 통합됨. 소개서엔 **4종으로만** 노출. "헬스/펫/액티비티"는 설명 문구에서 예시로만 언급(별도 탭 만들지 말 것).

## 3. 문제 정의

### 3-1. 매장 사장님의 문제
| 문제 | 구체 설명 |
|---|---|
| 신규 고객 확보 비용 | 배달앱/포털 광고는 클릭당 과금 + 수수료 누적. 실제 방문 보장 없음. |
| 공실/유휴 시간 | 평일 낮, 브레이크타임, 비수기 객실 — 고정비는 나가는데 매출 0. |
| 광고 효율 불투명 | 노출은 됐는데 방문·매출로 이어졌는지 추적 불가. |
| **노쇼** | 예약만 받으면 안 오는 손님 — 준비한 재료·시간 손실. |

### 3-2. 소비자의 문제
| 문제 | 구체 설명 |
|---|---|
| 가격 부담 | 정가로 사기엔 부담, 할인은 어디서 받을지 모름. |
| 검증 안 됨 | 처음 가는 동네 가게, 믿을 수 있는지 불확실. |
| 쿠폰 사용 마찰 | 종이쿠폰·복잡한 조건·현장 결제 실랑이. |

`[디자인 지시]` 좌우 2분할(사장님 고민 / 소비자 고민) 또는 "Before(문제) → After(솔루션)" 다이어그램.

## 4. 솔루션 = 동네딜(공동구매) 작동 방식

### 4-1. 핵심 모델 — "즉시판매형 단일가" (가장 중요, 정확히 표현할 것)

이 플랫폼의 동네딜은 **이름은 "공동구매"지만 경제 구조는 즉시판매**다. (근거: `docs/design/groupbuy-instant-sale.md`, `helpers.maxTierDiscount`)

- **가격은 인원과 무관하게 처음부터 단일 공구가**로 고정. (`maxTierDiscount` = 셀러가 설정한 최대 할인을 처음부터 모두에게 적용 — "A2 모델")
- 결제하면 **즉시 교환권이 확정 발급**된다. 목표 인원에 도달하지 못해도 교환권은 **그대로 유효**(자동 취소·환불 없음).
- "먼저 산 사람이 더 비싸지는" 동적 할인은 **제거됨**. 모두 같은 가격.

> ⚠️ **절대 쓰지 말 카피**: "N명 더 모이면 추가 할인", "인원 늘수록 더 싸짐". 실제로 가격이 인원 따라 내려가지 않으므로 **표시광고법 위반 소지**. (근거: `groupbuy-instant-sale.md §8`)
> ✅ **정직한 프레이밍**: "공동의 힘으로 떼온 대량 단가를 **지금 바로 모두에게** 동일 적용."

### 4-2. 인원/목표/마감의 역할 = "가격 게이트"가 아니라 "연출"

`group_buy_current`(현재 참여), `group_buy_target`(목표), `group_buy_deadline`(마감)은 **가격·환불을 좌우하지 않고**, 다음 용도로만 쓴다:

| 필드 | 의미 | UI 연출 |
|---|---|---|
| `group_buy_current` | 함께 산 사람 수 | "👥 237명이 함께 구매 중" (소셜 증거) |
| `group_buy_target` | 목표 인원(선택) | "🎉 목표 달성" 배지 — 장식용, 미달 패널티 없음 |
| `group_buy_deadline` | 판매 마감 시각 | "⏰ D-2 한정 공구" 긴장감 + 마감 후 판매 종료 |

`[디자인 지시]` 진행 게이지 바 + "👥 N명 참여 / 🎉 목표 M명" + 카운트다운 링. 단 "더 모이면 싸짐" 텍스트 금지.

### 4-3. 상태 흐름 (status)
`active`(판매중) → `achieved`(목표 달성·장식) / `expired`(마감) / `cancelled`(취소). 마감(`deadline < now`)되면 신규 참여 거부.

- 사장님 전용 **Magic Link 알림톡**: 공구 등록·첫 손님·사용 완료 시 자동 발송 (`sendStoreOwnerAlimtalk` 등)

## 7. 공구 유형 — 셀러 공구 vs 커뮤니티 공구

### 7-1. 셀러/매장 공구 (메인 = 즉시판매형 동네딜)
- 매장 사장님 또는 크리에이터가 등록. 즉시판매·단일가·교환권 즉시 발급(위 §4·§5).
- 카테고리: 식사권 / 미용 / 숙소 / 기타 4종.

### 7-2. 커뮤니티(유저) 공구 = "맛집 공동구매 제안" (보조 흐름)
소비자가 **"이 맛집 같이 싸게 가자"** 며 직접 제안하는 흐름. (`UserGroupBuyCreatePage.tsx`, `community-group-buy.routes.ts`)

| 항목 | 값 (코드 확정) |
|---|---|
| 등록 자격 | **점주(셀러) 또는 활동 인플루언서만** (일반 user 차단 — "아무나 못 올림") |
| 1인 보증금 | **최소 1,000딜** |
| 목표 인원 | **최소 3명** |
| 흐름(status) | `proposed`(제안) → `negotiating`(매장 협의) → `confirmed`(가격 확정) → `achieved`(달성) / `failed`(실패) / `refunded`(환불) |
| 미달성 시 | **보증금 전액 자동 환불** (만료 후 약 5분 내) |
| 입력값 | 카카오맵으로 맛집 선택 + 희망가 + 1인 보증금 + 목표 인원 |
| 초대 | `invite_code`로 친구 초대 → `/community-group-buy/:code` |

> **(A) 즉시판매형과의 결정적 차이**: (B)는 **보증금 예치 + 목표 미달 시 환불**이라 "진짜 조건부 공동구매". (A)는 결제 즉시 확정. 소개서에서 명확히 구분.

`[디자인 지시]` (B)는 "소비자가 직접 제안하는 동네 공구" 부록 카드 1개로. 보증금→협의→확정 3단계 미니 다이어그램.

## 8. 스텝 바이 스텝

### 8-1. 사장님 여정 (즉시판매형 동네딜)
1. **가입/등록** — `/seller/register`에서 점주(store_owner)로 가입. 매장 정보(상호·주소·전화) 등록.
2. **공구 등록** — 식사권/미용/숙소/기타 중 선택, **단일 공구가** 입력(과거 동적 tier UI는 단일가 입력으로 변경됨).
3. **노출/모집** — 동네딜 피드(`/group-buy`) + 크리에이터 홍보로 노출. 참여자 수가 소셜 증거로 누적.
4. **달성/판매** — 손님이 결제할 때마다 교환권 즉시 발급. (목표 도달은 장식 배지)
5. **방문/사용** — 손님이 매장에서 QR 제시 → 사장님 Magic Link로 스캔 → 사용 완료 처리.
6. **정산** — 사용 + **T+7** 후 등록 계좌로 자동 송금 (`restaurant_settlements`).

### 8-2. 소비자 여정
1. **발견** — 동네딜 피드/지도/검색에서 가게 발견.
2. **참여(결제)** — 단일 공구가로 즉시 결제(딜 또는 카드/토스).
3. **교환권/예약** — 결제 즉시 `UR-XXXX-XXXX` 발급(`/my-vouchers`). 예약형(뷰티/숙소)은 예약 잡기.
4. **사용** — 방문해서 QR 제시. 사용 완료 알림톡 수신. 후기 작성 시 보너스 딜.

`[디자인 지시]` 가로 6단계(사장님) / 4단계(소비자) 타임라인 2줄. 아이콘 + 한 단어 라벨.

## 9. 수수료 / 정산 / 포인트

> 모든 비율은 코드/설정에서 추출. 셀러별·GMV별로 달라지는 부분은 명시.

### 9-1. 플랫폼 수수료 (매장 사장님 부담)
| 항목 | 값 | 근거 |
|---|---|---|
| 동네딜(식사권 등) 기본 수수료 | **5%** | `helpers.DEFAULT_MEAL_VOUCHER_COMMISSION_RATE = 0.05` / `platform_settings.commission_rate_default` |
| 월 GMV 1,000만원+ 셀러 | **4%** | `TIER_COMMISSION` 자동 적용 |
| 월 GMV 1억원+ 셀러 | **3%** | `TIER_COMMISSION` 자동 적용 |
| 어드민 셀러별 조정 | `sellers.commission_rate` override 우선 | 위 tier보다 우선 |
| 후원(라이브) 수수료 | 15% (동네딜과 별개) | CLAUDE.md |

### 9-2. 딜 포인트
| 항목 | 값 |
|---|---|
| 충전 환율 | **1원 = 1딜** (수수료 없음) |
| 후원/상품 결제 | 딜 즉시 차감 |
| 최소 후원 | 500딜 |
| 커뮤니티 공구 보증금 | 최소 1,000딜 (목표 미달 시 환불) |

### 9-3. 정산 타이밍 & 원천징수
| 항목 | 값 | 근거 |
|---|---|---|
| 매장 정산 시점 | voucher **사용 + T+7** 자동 송금 | `auto-settlement.ts` |
| 원천징수 (사업자) | 면제(verified/exempt) — 세금계산서 발행 | `tax-withholding.ts` |
| 원천징수 (미등록·사업소득) | **3.3%** (default, 반복 활동) | `WITHHOLDING_RATES.business_income` |
| 원천징수 (기타소득·단발) | **8.8%** | `WITHHOLDING_RATES.other_income` |

### 9-4. 크리에이터(영입 인플루언서) 매장 입점 커미션
| 항목 | 값 | 근거 |
|---|---|---|
| 매장 입점 유치(intro) 커미션 | 매출의 **1.5%** (default) | `platform_settings.influencer_store_intro_pct` (CLAUDE.md 2026-06-01) |
| 대상 | `introduced_by_influencer_id`가 있는 매장 | 영구 적립(source='store_intro') |
| 지급 | T+7 성숙 + 원천징수(3.3%/8.8%) 후 송금 | influencer-payout cron |

`[디자인 지시]` 수수료는 표로. "5% (대형 매장 최저 3%)" + "정산 = 손님 방문 후 7일 자동 입금"을 강조 박스로.
`[확인 필요]` 카드/토스 결제 시 PG 수수료가 위 5%에 포함인지 별도인지 — 소개서 노출 전 사업팀 확인.

## 15. FAQ

### 사장님 FAQ
1. **입점에 비용이 드나요?** — 등록 무료입니다. 손님이 실제로 결제·방문할 때만 수수료(기본 5%)가 부과됩니다.
2. **언제 정산되나요?** — 손님이 교환권을 사용한 뒤 **7일 후** 등록 계좌로 자동 입금됩니다.
3. **인원이 안 모이면 손해인가요?** — 아닙니다. 즉시판매형이라 1명이 사도 그 즉시 확정 매출입니다. 목표 인원은 표시용입니다.
4. **손님이 안 오면(노쇼)?** — 손님이 **먼저 결제**하므로 미사용 교환권은 매출로 남고, 노쇼 손실이 구조적으로 줄어듭니다.
5. **사용 처리는 어떻게 하나요?** — 손님 QR을 전용 Magic Link(PIN 없는 영구 링크)로 스캔 1번이면 끝. POS 결제 안 하셔도 됩니다.
6. **세금계산서/원천징수는?** — 사업자 등록 인증 시 원천징수 면제(세금계산서 발행), 미등록 시 3.3%(사업소득) 원천징수 후 송금. `[확인 필요]` 세부 정산명세 양식.

### 소비자 FAQ
1. **인원이 안 모이면 환불되나요?** — 셀러 동네딜은 결제 즉시 교환권이 확정 발급되어 인원과 무관합니다. (소비자가 제안하는 '커뮤니티 공구'만 미달 시 보증금 자동 환불.)
2. **교환권은 어디서 보나요?** — `/my-vouchers`에서 QR로 확인합니다.
3. **환불 되나요?** — 발급 7일 이내·미사용이면 셀프 취소 가능. 딜은 즉시, 카드는 3~5영업일 환급.
4. **어떻게 사용하나요?** — 매장에서 QR을 보여주면 사장님이 스캔해 처리합니다.
5. **딜이 뭔가요?** — 1원=1딜로 충전하는 포인트입니다. 결제에 쓰고, 추천·후기 시 보너스로 적립됩니다.
6. **유효기간은?** — 교환권마다 `voucher_expiry`로 표시됩니다(발급 알림톡에 명시).
