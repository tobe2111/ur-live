## 1. 문서 목적 & 타깃

### 1.1 목적
유어딜 "링크샵"이 무엇인지, 크리에이터가 왜 개설해야 하는지, 어떻게 시작하고 수익을 내는지를 **한 장의 소개서**로 전달한다. 디자인 AI가 이 브리프를 받아 슬라이드/랜딩 형태로 시각화한다.

### 1.2 주 타깃 (Primary)
- **인플루언서 / 크리에이터 / 셀러** — 인스타·유튜브·틱톡 팔로워를 보유하고, 자기 링크 하나로 상품을 팔고 소통하며 수익화하려는 사람.
- 핵심 동기: "인스타 bio 링크 1개"의 한계를 넘어, 상품·라이브·공구·후원을 한 페이지에 모으고 싶다.

### 1.3 부 타깃 (Secondary)
- **매장 사장님(store_owner)** — 오프라인 매장을 운영하며 식사권/공구로 단골을 모으고 싶은 자영업자.
- 코드 근거: `SellerPublicPage`의 `seller.business_name` 표시, `식사권(meal_voucher)` 탭, `category === 'meal_voucher'` 필터.

> [디자인 지시] 표지/도입부는 주 타깃(크리에이터)을 전면에, 매장 사장님은 "이런 분도 좋아요" 보조 카드로.

### 1.4 이 소개서의 위상 — **크리에이터 수익화 허브 (Creator Monetization Hub)**
이 소개서는 단순한 "링크 모음 페이지" 소개가 아니라, **유어딜의 크리에이터(link-in-bio) 수익화 전체를 담는 1차 거점(primary home)**이다. 마스터 커버리지 문서(`docs/proposals/00-service-overview-and-coverage.md` §4-4)의 권고에 따라 다음 수익원의 **대표 소유 소개서**로 확정한다.

| 흡수한 수익/기능 라인 | 인벤토리 # | 이 deck에서의 위상 |
|---|---|---|
| **후원 (Donations)** | B3 | **primary home** — 라이브/링크샵 후원의 정식 수익 섹션 (§7-A) |
| **추천/어필리에이트 (Referral)** | B4 | **primary home** — 친구초대·핀·매장영입 보상의 중심 (§7-B) |
| **큐레이터 핀 (Curator Pin)** | B2 | 추천/어필리에이트 안에 포함 (§7-B) |
| **매장 영입 (Store Intro)** | B5 | 크리에이터 영입 관점 primary (§7-B), 에이전시 관점은 에이전시 소개서 |
| **호스팅 공구 (누구나 호스트)** | A10 | 링크샵 연결 동선으로 흡수 (§7-C) |
| **딜 포인트/지갑/원장** | B6 | 공통 인프라 — 결제·적립·출금 1줄 명시 (§7-D) |
| **로열티/팔로우/단골** | B7/G8 | 팬덤 전환 카피로 흡수 (§5-A) |
| **링크샵 내 라이브/쇼츠 (크리에이터 관점)** | A1/A2 | 크리에이터 송출·연결 동선으로 흡수 (§4-A) |

> [디자인 지시: 그라데이션] (이 1.4 위상 표) 표 헤더 행 = Primary 그라데이션, 본문은 다크(#121212 줄무늬). 밀도 '약'(16-G.6). "primary home" 텍스트만 Primary 그라데이션 텍스트로 강조(다크 면 위).

## 2-B. 범위 & 경계 (Scope & Boundary) — 다른 소개서와 안 겹치게

> **이 소개서가 다루는 것 vs 안 다루는 것을 명확히** 한다. 유어딜은 4개 소개서로 서비스 전체를 빠짐없이(MECE) 나누므로, 경계를 명시해 중복/혼동을 차단한다.

### 2-B.1 이 소개서의 소관 (In Scope)
- **링크샵(공개 link-in-bio 샵)** 그 자체 — 프로필/탭/공유/SEO
- **크리에이터 수익화 4종**: ① 후원, ② 추천/어필리에이트(친구초대·핀·매장영입), ③ 호스팅 공구, ④ 링크샵 상품 판매 정산
- **크리에이터 관점의 라이브/쇼츠** — 내 링크샵에서 라이브를 켜고·예약하고·쇼츠를 올려·시청자를 상품/공구/후원으로 전환하는 동선
- **딜 포인트/지갑/원장/출금** — 위 수익이 쌓이고 빠져나가는 공통 인프라(1줄 명시)

### 2-B.2 이 소개서가 **안** 다루는 것 (Out of Scope → 어느 소개서로)

| 주제 | 이 deck에서 | 담당 소개서 |
|---|---|---|
| **셀러 상품 입점/등록/대량등록/Cafe24/주문·배송 관리** | ❌ (판매 채널 노출만) | **온라인 입점** (`online-listing-proposal-brief`) |
| **라이브커머스의 "송출 인프라" (WHIP/OBS/멀티플랫폼·경매·타임딜)** | 🔸 크리에이터 관점만 언급, 입점·판매도구 측면은 ❌ | **온라인 입점** (라이브 판매채널) |
| **오프라인 매장 공구 / 동네딜 / 식사권 매장 발급·QR 정산** | ❌ (매장 사장님은 보조 타깃, 발급/정산은 안 다룸) | **오프라인 공구** (`offline-groupbuy-brief`) |
| **B2B 도매·제조·유통 등급제·OEM** | ❌ | **도매몰/유통스타트** (`wholesale-mall-brief`) |
| **에이전시의 매장 영입·정산 분배·캐스팅** | 🔸 크리에이터 영입만 다룸, 에이전시 조직 관점은 ❌ | 에이전시(별도/부록) |

### 2-B.3 "공동구매(공구)" 용어 충돌 정리 (중요)
유어딜에는 이름이 같은 "공구"가 4종 있다. **이 소개서가 다루는 것은 ③ 호스팅 공구(크리에이터/누구나 호스트)**다.

| 공구 종류 | 정의 | 담당 소개서 |
|---|---|---|
| ① 셀러 온라인 공구 (A6) | 즉시판매형 단일 할인가(이름만 공구) | 온라인 입점 |
| ② 오프라인 동네딜 (A7) | 매장 선결제 → 교환권 즉시발급 | 오프라인 공구 |
| ③ **호스팅 공구 (A10)** | **누구나 호스트가 voucher 공구를 모집(HOST 인센티브 1%)** | **링크샵 (이 deck)** ✅ |
| ④ 커뮤니티 맛집 제안 (A8) | 소비자가 맛집 공구를 제안·보증금 | 오프라인 공구(보조) |

> **이 deck의 "공동구매" = 크리에이터/지인이 링크샵에서 직접 여는 호스팅 공구(③)**. 셀러 즉시판매형(①)·오프라인 동네딜(②)과 혼동 금지.

> [디자인 지시] 2-B는 "경계 지도" 한 장으로. 가운데 링크샵 원(밝게), 주변에 온라인입점/오프라인공구/도매몰 원(차분한 다크)으로 배치하고 화살표로 "이건 저쪽" 연결. 공구 4종은 작은 4칸 칩 매트릭스.
>
> [디자인 지시: 그라데이션] (경계 지도) **중앙 링크샵 노드 = Creator mesh(16-G.2)**로 밝게(이 deck 주인공). 주변 타 소개서 노드는 채도 죽은 Dark card(#121212, glow X) — "여긴 저 소개서"임을 명도로 구분. 연결 화살표는 얇은 Primary→Warm. 공구 4종 칩 중 ③(호스팅)만 Primary 필 + ✓, 나머지 3개는 다크 단색 + 화살표 라벨(담당 소개서명). 밀도 '약', 글로우는 중앙 노드에만 미세하게.

## 4. 솔루션 = 링크샵이란

### 4.1 정의
**링크샵**은 크리에이터/셀러 1인당 하나씩 갖는 **공개 link-in-bio 샵 페이지**다. 인스타·유튜브 bio의 단 하나의 링크에 걸어두면, 팔로워가 그 안에서 상품 구매·라이브 시청·공동구매 참여·문의·후원까지 모두 할 수 있다.

### 4.2 실제 URL 형태 (코드 확인됨)
- `live.ur-team.com/s/{핸들}` — 짧은 형태 (`src/routes/seller.routes.tsx:92`)
- `live.ur-team.com/profile/{핸들}` — 전체 형태 (`src/routes/seller.routes.tsx:93`)
- `live.ur-team.com/u/{핸들}` — 큐레이터 페이지. 연결된 셀러가 있으면 동일 SellerPublicPage를 그대로 렌더 (`CuratorPage.tsx:146`, URL은 `/u/` 유지)
- 핸들에는 username 또는 숫자 ID 모두 가능 (`SellerPublicPage`의 `sellerId = rawParam` 처리)

> [확인 필요] 외부 노출용 "정식 1순위 URL"이 `/s/`인지 `/profile/`인지 마케팅 표기 통일은 사용자 확정 필요. (SEO canonical은 코드상 `/profile/{username}` 사용 — `SellerPublicPage.tsx:306,314`)

### 4.3 링크샵에 실제로 표시되는 것 (코드 1:1)
`SellerPublicPage.tsx`의 탭 구조 (`TABS` 배열, 라인 278~285):

| 탭 | 내용 | 근거 |
|---|---|---|
| **홈(home)** | 식사권·쇼츠·최근 라이브를 한 화면에 프리뷰 (HomeTab) | `tab === 'home'` |
| **상품(shop)** | 일반 상품 그리드. 카드에 할인율 배지·정상가 취소선·판매가 | `shopProducts` (식사권·교환권 제외), grid 2~5열 반응형 |
| **교환권/식사권(vouchers)** | `category === 'meal_voucher'` 상품 (맛집 식사권 등) | `mealVouchers` 필터 |
| **영상/쇼츠(shorts)** | 셀러의 쇼츠 피드 | `/api/shorts/feed?seller_id=` |
| **라이브(live)** | 라이브 방송 카드 그리드 (지난 방송 포함) | `streams`, StreamCard |
| **정보(info)** | 소개·SNS·카카오 연락처 등 (InfoTab) | `tab === 'info'` |

### 4.4 프로필 헤더에 표시되는 것 (`ProfileHeader.tsx`)
- **커버 배너**(`banner_url`) 또는 브랜드 컬러 그라디언트(`brand_color`) 또는 기본 그라디언트
- **프로필 아바타**(`profile_image`) + 라이브 중이면 `LIVE` 배지
- **이름 + 소개(bio) + 매장명**(`business_name`, MapPin 아이콘)
- **3분할 통계**: 상품 수 / 라이브 수 / 평점(`average_rating`, 별점)
- **CTA**: 팔로우 버튼 / 단골 등록 / 카톡 1:1 문의(`kakao_chat_link`) / 후원 버튼
- **공유 버튼**(우상단): `navigator.share` 또는 링크 복사

## 6. 개설 / 운영 절차 (스텝 바이 스텝)

> 코드 근거: `BottomNav.tsx` linkshopPath 로직, `/host/new`, `KakaoCallbackPage`, `handle-generator.ts`, `SellerPublicPage` 인라인 편집/업로드.

1. **카카오로 로그인/시작** — 별도 회원가입 없이 카카오 계정 한 번. (셀러 권한 추가도 같은 계정에 1탭, `SellerUpgradePanel`)
2. **핸들·프로필 자동/수동 설정** — 첫 진입 시 카카오 닉네임 기반으로 핸들 자동 생성(`generateUniqueHandle`). 미로그인/미설정 시 `/host/new` 카탈로그로 안내.
3. **커버 배너 / 프로필 이미지 업로드** — 본인 페이지에서 아바타·배너 클릭 → 이미지 업로드(최대 5MB, JPEG/PNG/WebP/GIF). 클라이언트 압축 후 `/api/seller/upload-image`로 URL 저장.
4. **이름·소개·SNS·카톡 링크 인라인 편집** — 텍스트를 클릭하면 그 자리에서 수정(이름/bio/Instagram/YouTube/카카오 채팅 링크).
5. **상품 연결** — 셀러 대시보드(`/seller`)에서 상품·식사권 등록 → 링크샵 상품/식사권 탭에 자동 노출.
6. **라이브·쇼츠·공구 연결** — 라이브 방송 시작/쇼츠 업로드/공동구매 개설 시 해당 탭에 자동 반영.
7. **링크 공유** — 우상단 공유 버튼으로 카톡/링크복사 → 인스타·유튜브 bio에 붙여넣기.
8. **정산** — 판매·후원·추천 보상이 누적, 출금 UI에서 출금(원천징수 자동, §7).

> [디자인 지시] 8단계를 가로 타임라인 또는 세로 번호 스텝으로. 1~4는 "개설", 5~8은 "운영·수익".
>
> [디자인 지시: 그라데이션] (개설 절차 타임라인) **번호 배지 ①~⑧ = Primary 원형**(`linear-gradient(135deg, #FF0033, #FF3D6E)`)에 흰 숫자. 타임라인 연결선은 Primary→Warm 진행 그라데이션으로 "개설(1~4)→수익(5~8)" 단계가 색으로 흐르게. 각 스텝 카드는 Dark card + 좌측 Primary edge. 밝은 배지 면(밝은 #FF3D6E 쪽) 위 숫자는 흰색 700+ 고정(16-G.4).

## 7-A. 후원 (Donations) — primary home 🟥

> **이 소개서가 후원의 1차 거점.** 라이브 후원과 링크샵 후원은 같은 기능이며, 여기서 전모를 설명한다.

### 7-A.1 무엇 / 크리에이터 가치
팬이 크리에이터에게 직접 금액을 보내는 **순수 응원 수익**. 상품을 팔지 않아도, 라이브를 보며 또는 링크샵에서 바로 후원이 들어온다. 콘텐츠 자체가 수익이 되는 채널.

### 7-A.2 작동 방식 (코드 1:1)
- **동선**: 라이브 방송 중 후원 위젯(`LiveDonation`) / 링크샵 프로필 헤더 "후원" CTA(`ProfileHeader`).
- **금액**: 최소 **1,000원**, **100원 단위**, 단건 상한 1,000만원, **일 5,000만원 cap**(`donations.routes.ts`).
- **수수료**: 기본 **15%**(`donation_commission_rate` 컬럼 default 15.0 — **셀러별 어드민 조정 가능**, `/api/admin/sellers/:id/donation-commission`). 후원금에서 수수료 차감 후 `credit_amount`가 셀러 적립.
- **정산**: 후원 발생 후 **10일 경과(T+10)** 분부터 정산 신청 가능(`seller-donations.routes.ts` `donation_settlements`).
- **원천징수**: 출금 시 사업소득 3.3% / 기타소득 8.8% 자동(§7-D).
- **결제**: 토스페이먼츠 V2 표준(잠긴 SSOT 게이트웨이), 금액 서버 재검증(클라 amount 위변조 차단).

### 7-A.3 수치 요약
- 최소 **1,000원** · 수수료 **15%(조정 가능)** · 정산 **T+10** · 일 cap **5,000만원**.

> [디자인 지시] 7-A는 "후원 → 차감(15%) → T+10 정산 → 출금" 단순 흐름도 + 라이브/링크샵 두 진입점 아이콘.
>
> [디자인 지시: 그라데이션] (후원 흐름) 전체 화살표 Warm(#FF0033→#FF7A00, 후원=수익 온기). "15%" 차감 노드는 Primary 칩(다크 면), 출금 노드 Deep 마무리. 밝은 구간 라벨엔 스크림(16-G.4). 밀도 '중'.

## 7-B. 추천 / 어필리에이트 (Referral & Affiliate) — primary home 🟥

> **이 소개서가 추천/어필리에이트의 1차 거점.** 친구 초대 · 큐레이터 핀 · 매장 영입 세 갈래를 한곳에 묶는다. ref 쿠키 TTL **24시간**(`CURATOR_DEFAULTS.REF_COOKIE_TTL_HOURS = 24`).

### 7-B.1 세 갈래 한눈에

| 갈래 | 보상 | 대상 | 작동 방식 | 코드 근거 |
|---|---|---|---|---|
| **친구 초대 (양쪽 보너스)** | 각 **0.5%** (추천인+피추천인, 첫 1회·self-refer 차단) | 일반 user / 크리에이터 | 피추천인 **첫 주문**에 양쪽 적립 | `REFERRAL_BONUS_BOTHSIDES_PCT = 0.5`, `invite-reward.routes.ts` (첫 주문 `cnt===1` 체크) |
| **친구 초대 (딜 포인트형)** | 기본 **1,000딜** 정액(어드민 조정) | 추천인 | 초대한 친구 첫 주문 시 추천인에 딜 적립 | `invite-reward.routes.ts` `platform_settings.invite_reward_amount` default 1000 [확인 필요: 0.5%와 동시 적용 여부] |
| **큐레이터 핀 어필리에이트** | **1.0%** | 핀한 큐레이터에게만 (피구매자 X) | 큐레이터가 상품 핀(추천) → 그 링크로 구매 시 적립. `products.referral_commission_rate` 있으면 우선 | `CURATOR_AFFILIATE_PCT = 1.0` |
| **매장 영입 (Store Intro)** | **1.5%** | 영입자(크리에이터) | 영입한 매장의 **매 결제마다** 매출의 1.5% 적립(멱등) | `influencer-store-intro-commission.ts` `DEFAULT_STORE_INTRO_PCT = 1.5`, `platform_settings.influencer_store_intro_pct` |

> 일반 제휴(affiliate) 추천인 보상 fallback은 **5%**(`AFFILIATE_COMMISSION_PCT = 5`, `platform_settings.affiliate_commission_rate`로 조정). 핀(1%)·친구초대(0.5%)와는 별개 경로.

### 7-B.2 큐레이터 핀 (Curator Pin) — 내 샵 + 추천 동시
- **무엇**: 일반 유저도 상품을 "핀(추천)"해 자기 큐레이터 페이지(`/u/{handle}`)에 진열 → 그 링크로 팔리면 **1%** 적립.
- **크리에이터 가치**: 내 상품이 없어도(재고·정산 없이) 추천만으로 수익. 링크샵과 동일 화면(연결된 셀러면 `SellerPublicPage` inline render).
- **수치/규칙**: 핀 최대 **200개/인**(`PIN_MAX_PER_USER`), 핸들 `[a-z0-9_]{3,30}`, ref 쿠키 **24h**.

### 7-B.3 매장 영입 (Store Intro) — 크리에이터가 매장을 데려오면
- **무엇**: 크리에이터가 매장(사장님)을 유어딜로 영입 → 그 매장의 **모든 결제**에서 매출의 **1.5%**를 영입자에게 적립.
- **작동**: 매장에 `introduced_by_influencer_id` 기록 → 결제 confirm마다 `influencer_attributions(source='store_intro')` 1행 적립(멱등). **T+7 환불창** 성숙(pending→available) 후 influencer-payout cron이 원천징수·송금. 환불 시 자동 역전.
- **경계**: 같은 매장영입을 **에이전시**가 하면 별도 경로(`agency_store_intro_commissions`, default 2%) — 에이전시 관점은 에이전시 소개서. 이 deck은 **크리에이터 영입(1.5%)**만.

### 7-B.4 추천 트리 / 정산 / 출금
- **추천 트리**: `referral_tree`(parent / grandparent / great_grandparent, depth) — 다단 구조 추적.
- **정산**: 적립은 `influencer_attributions`/`user_points`에 쌓이고, 출금 UI에서 최소 1만원부터 출금(원천징수 자동, §7-D).

> [디자인 지시] 7-B는 "세 갈래 보상" 3분할 + 가운데 내 링크샵. 0.5%/1%/1.5% 수치를 크게.
>
> [디자인 지시: 그라데이션] (추천 3갈래) 세 갈래 카드 = Dark card + Primary edge, 수치(0.5%/1%/1.5%)는 Primary 그라데이션 텍스트(다크 면). 가운데 링크샵 노드 Creator mesh. 화살표 Primary→Warm. 밀도 '중', 카드당 accent 5~15%.

## 7-C. 호스팅 공구 (누구나 호스트) — 링크샵에서 직접 모집 🟥

> **2-B.3의 ③ 공구.** 셀러 즉시판매형(①)·오프라인 동네딜(②)과 다른, **크리에이터/지인이 직접 여는 voucher 공구**.

### 7-C.1 무엇 / 크리에이터 가치
크리에이터(또는 누구나)가 카탈로그에서 상품을 골라 **"N명 모이면" 공동구매를 호스팅**하고, 링크샵/초대 링크로 친구를 모집. 모집·전환이 곧 내 인센티브.

### 7-C.2 작동 방식 & 수치 (`HOSTING_DEFAULTS`)
- **호스트 인센티브**: 친구가 내 공구로 참여 시 **1.0%** 적립(`HOST_INCENTIVE_PCT = 1.0`).
- **모집 한도**: 동시 활성 공구 **최대 10개/인**(`MAX_ACTIVE_HOSTINGS`), 목표 인원 **2~100명**(`MIN_TARGET`~`MAX_TARGET`).
- **기간**: 기본 모집 **7일**(`DEFAULT_DEADLINE_DAYS`), 호스트 노트 200자.
- **동선**: `/host/new` 카탈로그 → 호스트 세션 생성 → `invite_code`(8자 hex, `/g/:invite_code`)로 공유 → 링크샵에 노출. 미로그인 링크샵 탭 진입 시 `/host/new`로 안내.
- **결제 안전**: 공구·숙소 reserve-before-charge + CAS 가드(오버부킹/이중결제 방어).

> [디자인 지시] 7-C는 "상품 고르기 → 친구 모집(N명) → 단일 할인가 → 호스트 1%" 4스텝. 진행바(현재/목표) 비주얼.
>
> [디자인 지시: 그라데이션] (호스팅 공구) 공구 카드 = Dark card + 좌 Primary edge. "단일 할인가 적용" 배지 = Warm(전환). 진행바 채움 = Primary→Warm 그라데이션. 호스트 1% 칩 Primary(다크 면). 밀도 '중'.

## 7-D. 딜 포인트 / 지갑 / 원장 / 출금 (공통 인프라)

> 위 모든 수익(판매·후원·추천·핀·영입·호스트)이 **쌓이고 빠져나가는 공통 결제·정산 인프라**. 모든 소개서가 공유하지만, 크리에이터 수익화의 종착점이라 여기 명시.

| 항목 | 내용 | 코드/수치 근거 |
|---|---|---|
| **충전** | 1원 = 1딜, **수수료 0** (토스 결제) | CLAUDE.md "충전 1원=1딜(수수료 없음)" |
| **적립/차감** | 후원·추천·핀·영입·호스트 보상 적립 / 후원·구매 시 즉시 차감 | `user_points` / `influencer_balances` / `influencer_attributions` |
| **원장(ledger)** | 충전·적립·차감·출금 내역 추적 | `/my-deal-history`, `/my-ledger`, B6 `ledger/` |
| **출금** | 최소 **10,000원**(`WITHDRAWAL_DEFAULTS.MIN_AMOUNT`) | `policy.ts` |
| **원천징수** | 출금 시 사업소득 **3.3%** / 기타소득 **8.8%** 자동(`sellers.tax_type`) | `tax-withholding.ts` `WITHHOLDING_RATES` |
| **셀러 승급 안내** | 누적 정산 **50만원** 도달 시 큐레이터→셀러(사업자) 자동 안내 | `SELLER_UPGRADE_THRESHOLD = 500_000` |

> [디자인 지시] 7-D는 "여러 수익원 → 지갑(원장) → 출금(원천징수 3.3%) → 통장" 합류 도식. §13-③ 수익 흐름도와 통합 가능.
>
> [디자인 지시: 그라데이션] (지갑/출금) 여러 입력 화살표 Warm → 지갑 노드(Dark card + Primary glow) → 출금 노드 Deep. 원천징수 칩 Primary(다크 면). 밀도 '중'.

## 9. 차별점

| 비교 대상 | 그들 | 유어딜 링크샵 |
|---|---|---|
| **링크트리** | 링크 모음만. 결제·라이브·공구 없음 | 링크 + **실제 판매·라이브·공구·후원** 올인원 |
| **인스타샵** | 인스타 내부, 외부 동선·라이브커머스 약함 | 독립 URL + 라이브 + 공동구매 + 후원 |
| **스마트스토어/자사몰** | 입점·운영·디자인 비용·시간 | **무료·즉시 개설**, 1인 크리에이터 친화 |
| **쿠팡파트너스(어필리에이트)** | 링크 수수료만, 내 페이지 없음 | 내 샵 + 추천/영입/핀 보상 동시 |

**핵심 차별**: "라이브 + 공동구매 + 후원 + 어필리에이트를 한 링크에 묶은 국내 통합 플레이어" — `linkshop-pivot.md`는 "한국에 공구+어필리에이트+큐레이션 통합 플레이어 없음"이라 명시. [가정] 경쟁사 단점 서술은 일반적 시장 인식 기준이며 정밀 벤치마크 아님.

> [디자인 지시] 4열 비교 매트릭스. 유어딜 열만 빨강(#FF0033) 강조 + 체크 아이콘 풀스택.
>
> [디자인 지시: 그라데이션] (차별점) **유어딜 열 전체 = Deep 그라데이션 컬럼 배경**(`linear-gradient(160deg, #C9184A, #FF0033 60%, #FF4D6D)`) + 컬럼 외곽 glow. 경쟁사 열은 채도 죽은 다크 단색(#121212, glow X). 체크 아이콘은 Primary, 흰색 V. **색만으로 구분하지 말고 체크 아이콘도 함께**(16-G.4 색맹 대비). 유어딜 컬럼의 밝은 #FF4D6D 구간에 텍스트가 닿으면 셀별 다크 스크림 칩 위에 텍스트 배치.

## 12. 섹션별 슬라이드 카피 초안

> 실제 사용 가능한 한국어 카피. 디자인 AI는 이 텍스트를 그대로 배치하거나 길이만 조정.

### 표지
- 제목: **"링크 하나로, 나만의 샵."**
- 부제: 유어딜 링크샵 — 상품·라이브·공동구매·후원을 한 페이지에
- 캡션: 무료 개설 · 카카오 로그인 한 번

> [디자인 지시: 그라데이션] (표지/다크 hero) 배경 = **Dark hero 배경**(`linear-gradient(180deg, #1A0008 0%, #020202 72%)`) + 그 위에 겹쳐서 **Dark hero 글로우**(`radial-gradient(at 50% 0%, rgba(255,0,51,.30), transparent 55%)`) — 상단 중앙에서 레드가 번지고 아래로 검정 수렴. 제목 "링크 하나로, 나만의 샵."은 흰색(extrabold), 다크 면 위라 대비 OK(16-G.4). 강조하려면 "샵" 한 단어만 Primary 그라데이션 텍스트(`background-clip:text`). 표지에 모바일 목업이 같이 들어가면 폰 안 배너는 Creator mesh(16-G.2). 1차 CTA 버튼은 시그니처 Primary(`linear-gradient(135deg, #FF0033, #FF3D6E)`) + glow(`box-shadow: 0 8px 32px rgba(255,0,51,.35)`). 글로우 피크가 제목 텍스트와 정면으로 겹치면 글로우 중심을 폰 목업/여백 쪽으로 살짝 이동(16-G.4). 글로우는 표지·CTA에만 강하게(16-G.6).

- 헤드: **"인스타 bio엔 링크가 딱 하나뿐."**

## 14. FAQ (크리에이터)

1. **Q. 개설 비용이 있나요?**
   A. 없습니다. 카카오 로그인 한 번으로 무료 개설됩니다.
2. **Q. 사업자 등록이 꼭 필요한가요?**
   A. 시작은 필요 없습니다. 누적 정산이 일정 수준(50만원)에 이르면 사업자 전환을 자동 안내합니다.
3. **Q. 내 링크 주소는 어떻게 정해지나요?**
   A. 카카오 닉네임 기반으로 핸들이 자동 생성됩니다(영문 소문자·숫자·언더스코어 3~30자). 형태는 `live.ur-team.com/s/내핸들`.
4. **Q. 어떤 걸 팔 수 있나요?**
   A. 일반 상품, 맛집 식사권, 라이브 커머스 상품, 공동구매 등을 한 페이지에 올릴 수 있습니다.
5. **Q. 라이브 방송은 어떻게 연결되나요?**
   A. 셀러 대시보드에서 방송을 시작하면 링크샵에 자동으로 LIVE 배지와 라이브 탭에 표시됩니다. 방문자에게 실시간으로 "라이브 시작" 알림이 갑니다.
6. **Q. 후원은 어떻게 받나요? 최소 금액이 있나요?**
   A. 라이브 방송 중이나 링크샵에서 팬이 직접 후원할 수 있습니다. 최소 후원은 **1,000원**(100원 단위)이고, 별도 수수료(기본 15%, 셀러별 조정 가능)가 적용됩니다. 후원금은 발생 10일 후(T+10)부터 정산 신청할 수 있습니다.
7. **Q. 수수료와 정산은 어떻게 되나요?**
   A. 상품 판매는 기본 5% 플랫폼 수수료가 차감된 뒤 정산됩니다(셀러별 조정 가능). 출금은 최소 1만원부터, 원천징수(사업소득 3.3% / 기타소득 8.8%)가 자동 적용됩니다.
8. **Q. 친구를 초대하거나 매장을 데려오면 보상이 있나요?**
   A. 친구 초대는 양쪽 각 0.5%(피추천인 첫 주문, 1회), 큐레이터 핀(상품 추천)은 1%, 매장(사장님) 영입은 그 매장의 매 결제마다 매출의 1.5%가 적립됩니다. 추천 링크(ref)는 24시간 유효합니다.
13. **Q. 상품이 없어도 추천만으로 수익을 낼 수 있나요?**
    A. 네. 마음에 드는 상품을 "핀(추천)"해 큐레이터 페이지(`/u/내핸들`)에 진열하면, 그 링크로 팔릴 때마다 1%가 적립됩니다(재고·정산 부담 0, 핀 최대 200개).
14. **Q. 직접 공동구매를 열 수 있나요?**
    A. 네. `/host/new`에서 상품을 골라 "N명 모이면" 공동구매를 호스팅하고 링크로 친구를 모집할 수 있습니다(동시 최대 10개, 모집 기본 7일). 친구가 참여하면 호스트 인센티브 1%가 적립됩니다.
15. **Q. 적립된 돈은 어떻게 쌓이고 출금하나요?**
    A. 판매·후원·추천·핀·영입·호스트 보상이 지갑(딜 포인트/원장)에 적립되고, 최소 1만원부터 출금합니다. 출금 시 원천징수(사업소득 3.3% / 기타소득 8.8%)가 자동 적용됩니다.
16. **Q. 라이브 방송은 내 링크샵과 어떻게 연결되나요?**
    A. 방송을 시작하면 링크샵에 LIVE 배지와 라이브 탭 카드가 자동 표시되고, 방문자에게 실시간 "라이브 시작" 알림이 갑니다. 라이브 중 상품·공동구매·후원을 같은 페이지에서 동시에 전환할 수 있습니다.
9. **Q. 프로필/배너 이미지는 어떻게 바꾸나요?**
   A. 본인 페이지에서 아바타·배너를 클릭해 바로 업로드합니다(최대 5MB, JPEG/PNG/WebP/GIF).
10. **Q. 결제는 안전한가요?**
    A. 토스페이먼츠 표준 결제를 사용하며, 결제 금액은 서버에서 재계산합니다.
11. **Q. 매장 사장님도 쓸 수 있나요?**
    A. 네. 식사권/공동구매로 단골을 모으는 데 특히 적합합니다.
12. **Q. 인스타 외에 어디에 링크를 걸 수 있나요?**
    A. 유튜브·틱톡·블로그 bio 등 외부 링크를 받는 어디든 붙일 수 있습니다.

> [확인 필요] FAQ 6의 후원 수치는 **코드 기준으로 정정**(15% 기본·최소 1,000원·T+10 정산). 친구초대 0.5%(`policy.ts`)와 딜포인트 정액 보상(`invite_reward_amount` 1,000딜)이 **동시 적용되는지/택일인지**는 코드상 별도 경로로 보여 운영 확정 필요. 승급 50만원은 `SELLER_UPGRADE_THRESHOLD` 확인됨.

## 16-G. 🎨 그라데이션 디자인 시스템 (이 덱의 비주얼 코어)

> **원칙**: 이 덱은 **다크 테마 위 #FF0033 글로우·메쉬**가 핵심 무드다. 그라데이션은 장식이 아니라 브랜드 시그니처 — 모든 슬라이드에 일관된 "패밀리"로 등장해야 한다. **무지개 남발 금지**, 빨강~핑크~오렌지 좁은 온난 스펙트럼 안에서 세련되게.

### 16-G.1 공유 브랜드 그라데이션 토큰 (verbatim — 패밀리 일관성, 그대로 사용)

| 토큰 | CSS | 용도 |
|---|---|---|
| **Primary** | `linear-gradient(135deg, #FF0033 0%, #FF3D6E 100%)` | 기본 강조 — 버튼·배지·번호 배지·표 헤더·체크 아이콘 |
| **Warm** | `linear-gradient(135deg, #FF0033 0%, #FF7A00 100%)` | 수익/전환 강조 — 수익 흐름도·전환 화살표·"매출이 되는 순간" |
| **Deep** | `linear-gradient(160deg, #C9184A 0%, #FF0033 60%, #FF4D6D 100%)` | 깊이감 강조 — hero 보조·프리미엄 카드·크리에이터 프로필 |

### 16-G.2 다크 전용 그라데이션 토큰 (이 덱 = 다크 `#020202` / 카드 `#121212`)

| 토큰 | CSS | 용도 |
|---|---|---|
| **Dark hero 배경** | `linear-gradient(180deg, #1A0008 0%, #020202 72%)` | 표지·CTA 등 전면 hero 배경(아래로 검정 수렴) |
| **Dark hero 글로우** | `radial-gradient(at 50% 0%, rgba(255,0,51,.30), transparent 55%)` | hero 배경 위에 **겹쳐서** 상단 중앙 레드 글로우 |
| **Dark card** | `linear-gradient(160deg, #1B1B1F 0%, #121212 100%)` | 모든 카드(상품/라이브/공구/혜택/구성요소) 기본 면 |
| **Card accent edge** | 좌측/상단 1~2px `linear-gradient(180deg, #FF0033, #FF3D6E)` 또는 `box-shadow: 0 0 24px rgba(255,0,51,.18)` glow | dark card 의 레드 accent edge / glow |
| **Creator auroral mesh** | Deep 베이스 + `radial-gradient(at 80% 20%, rgba(255,122,0,.22), transparent 50%)` 를 살짝 섞은 오로라 메쉬 | 프로필/링크 hero·크리에이터 강조 |

> 메쉬 작성 예 (creator 강조):
> ```css
> background:
>   radial-gradient(at 80% 20%, rgba(255,122,0,.22), transparent 50%),
>   radial-gradient(at 15% 85%, rgba(255,77,109,.20), transparent 55%),
>   linear-gradient(160deg, #C9184A 0%, #FF0033 60%, #FF4D6D 100%);
> ```

### 16-G.3 용도 규칙 (어디에 어떤 토큰)

- **전환·수익 = Warm** (#FF0033→#FF7A00). "팔로워가 매출이 되는" 흐름, 수익 흐름도, bio→올인원 전환 화살표.
- **기본 강조·인터랙션 = Primary** (#FF0033→#FF3D6E). 버튼·배지·번호 배지·표 헤더·활성 탭 언더라인.
- **깊이·프리미엄·크리에이터 = Deep / Creator mesh**. 프로필/배너 hero, 차별점의 유어딜 열.
- **카드 면 = Dark card** + accent edge / glow. 절대 단색 `#121212` 으로 끝내지 말고 160deg 미세 그라데이션 + 레드 edge.
- **배경 = Dark hero 배경 + 글로우** 조합. 글로우는 hero·CTA·표지에만(전 슬라이드 남발 X).

### 16-G.4 접근성 (필수)

- **다크 면(글로우 약하게 깔린 #020202~#1A0008) 위 흰 텍스트 = OK** — 대비 충분.
- **밝은 그라데이션 부분 위 텍스트 금지** — Warm 의 #FF7A00 쪽, Primary 의 #FF3D6E 쪽처럼 밝아지는 영역 위에 본문/숫자 직접 올리지 말 것.
  - 불가피하면 **스크림(scrim)**: 텍스트 뒤 `linear-gradient(0deg, rgba(0,0,0,.55), rgba(0,0,0,.25))` 오버레이 또는 텍스트 블록만 `#121212` 칩 위에.
- 그라데이션 **버튼 위 텍스트는 항상 `#FFFFFF` (700+)** — 회색 텍스트 금지.
- 글로우/메쉬는 **콘텐츠 가독성을 해치지 않는 투명도(≤.30)**. 텍스트 영역과 글로우 피크가 겹치면 글로우 위치를 비텍스트 쪽으로.
- 색만으로 정보 전달 금지(차별점 "유어딜 열" 등은 그라데이션 + 체크 아이콘 **둘 다**).
