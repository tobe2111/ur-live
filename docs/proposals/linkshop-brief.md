## 1. 문서 목적 & 타깃

### 1.1 목적
유어딜 "링크샵"이 무엇인지, 크리에이터가 왜 개설해야 하는지, 어떻게 시작하고 수익을 내는지를 **한 장의 소개서**로 전달한다. 디자인 AI가 이 브리프를 받아 슬라이드/랜딩 형태로 시각화한다.

### 1.2 주 타깃 (Primary)
- **인플루언서 / 크리에이터 / 셀러** — 인스타·유튜브·틱톡 팔로워를 보유하고, 자기 링크 하나로 상품을 팔고 소통하며 수익화하려는 사람.
- 핵심 동기: "인스타 bio 링크 1개"의 한계를 넘어, 상품·라이브·공구·후원을 한 페이지에 모으고 싶다.

### 1.3 부 타깃 (Secondary)
- **매장 사장님(store_owner)** — 오프라인 매장을 운영하며 식사권/공구로 단골을 모으고 싶은 자영업자.
- 코드 근거: `SellerPublicPage`의 `seller.business_name` 표시, `식사권(meal_voucher)` 탭, `category === 'meal_voucher'` 필터.

> [디자인 지시] 표지/도입부는 주 타깃(크리에이터)을 전면에, 매장 사장님은 "이런 분도 좋아요" 보조 카드로.

## 2. 한 줄 요약 & 엘리베이터 피치

### 2.1 한 줄 요약
> **"링크 하나로 상품·라이브·공동구매·소통·후원을 모두 — 크리에이터 전용 무료 샵, 유어딜 링크샵."**

### 2.2 엘리베이터 피치 (30초)
> "인스타 bio에는 링크를 딱 하나밖에 못 넣죠. 그 한 칸을 '유어딜 링크샵'으로 바꾸세요. 카카오 로그인 한 번이면 끝, 무료로 개설됩니다. 내가 고른 상품 그리드, 진행 중인 라이브, 친구와 함께 사는 공동구매, 쇼츠 영상, 카톡 1:1 문의, 팬의 후원까지 한 페이지에 담깁니다. 팔로워가 그 링크를 누르는 순간 매출과 보상이 시작됩니다. 별도 쇼핑몰 구축 비용 0원, 결제는 토스로 안전하게."

### 2.3 핵심 키워드
`무료 개설` · `링크 하나` · `올인원` · `라이브+공구+후원` · `즉시 수익화` · `다크 감성`

| # | 문제 | 크리에이터가 겪는 현실 |
|---|---|---|
| P1 | **흩어진 링크** | 인스타 bio는 외부 링크 1개 제한. 쇼핑몰·라이브·공구·연락처를 한 번에 못 건넴 → 팔로워가 이탈 |
| P2 | **팔로워 → 매출 전환 실패** | 콘텐츠로 모은 팔로워를 "어디서 사세요?"에 답할 판매 동선이 없음 |

| P5 | **공동구매를 돌릴 도구 부재** | "N명 모이면 할인" 같은 공구를 팔로워와 진행할 마땅한 인프라가 없음 |

## 4. 솔루션 = 링크샵이란

### 4.1 정의
**링크샵**은 크리에이터/셀러 1인당 하나씩 갖는 **공개 link-in-bio 샵 페이지**다. 인스타·유튜브 bio의 단 하나의 링크에 걸어두면, 팔로워가 그 안에서 상품 구매·라이브 시청·공동구매 참여·문의·후원까지 모두 할 수 있다.

### 4.2 실제 URL 형태 (코드 확인됨)
- `live.ur-team.com/s/{핸들}` — 짧은 형태 (`src/routes/seller.routes.tsx:92`)
- `live.ur-team.com/profile/{핸들}` — 전체 형태 (`src/routes/seller.routes.tsx:93`)
- `live.ur-team.com/u/{핸들}` — 큐레이터 페이지. 연결된 셀러가 있으면 동일 SellerPublicPage를 그대로 렌더 (`CuratorPage.tsx:146`, URL은 `/u/` 유지)
- 핸들에는 username 또는 숫자 ID 모두 가능 (`SellerPublicPage`의 `sellerId = rawParam` 처리)

> [확인 필요] 외부 노출용 "정식 1순위 URL"이 `/s/`인지 `/profile/`인지 마케팅 표기 통일은 사용자 확정 필요. (SEO canonical은 코드상 `/profile/{username}` 사용 — `SellerPublicPage.tsx:306,314`)

### 4.3 링크샵에 실제로 표시되는 것 (코드 1:1)
`SellerPublicPage.tsx`의 탭 구조 (`TABS` 배열, 라인 278~285):

| 탭 | 내용 | 근거 |
|---|---|---|
| **홈(home)** | 식사권·쇼츠·최근 라이브를 한 화면에 프리뷰 (HomeTab) | `tab === 'home'` |
| **상품(shop)** | 일반 상품 그리드. 카드에 할인율 배지·정상가 취소선·판매가 | `shopProducts` (식사권·교환권 제외), grid 2~5열 반응형 |
| **교환권/식사권(vouchers)** | `category === 'meal_voucher'` 상품 (맛집 식사권 등) | `mealVouchers` 필터 |
| **영상/쇼츠(shorts)** | 셀러의 쇼츠 피드 | `/api/shorts/feed?seller_id=` |
| **라이브(live)** | 라이브 방송 카드 그리드 (지난 방송 포함) | `streams`, StreamCard |
| **정보(info)** | 소개·SNS·카카오 연락처 등 (InfoTab) | `tab === 'info'` |

### 4.4 프로필 헤더에 표시되는 것 (`ProfileHeader.tsx`)
- **커버 배너**(`banner_url`) 또는 브랜드 컬러 그라디언트(`brand_color`) 또는 기본 그라디언트
- **프로필 아바타**(`profile_image`) + 라이브 중이면 `LIVE` 배지
- **이름 + 소개(bio) + 매장명**(`business_name`, MapPin 아이콘)
- **3분할 통계**: 상품 수 / 라이브 수 / 평점(`average_rating`, 별점)
- **CTA**: 팔로우 버튼 / 단골 등록 / 카톡 1:1 문의(`kakao_chat_link`) / 후원 버튼
- **공유 버튼**(우상단): `navigator.share` 또는 링크 복사
- **외부 라이브 배지**: 라이브 중일 때 TikTok/Instagram/Facebook 외부 라이브 링크 표시

> [디자인 지시] 솔루션 섹션은 위 4.3 탭과 4.4 헤더를 "실제 모바일 목업"으로. 다크(#020202) 배경 필수.

## 6. 개설 / 운영 절차 (스텝 바이 스텝)

> 코드 근거: `BottomNav.tsx` linkshopPath 로직, `/host/new`, `KakaoCallbackPage`, `handle-generator.ts`, `SellerPublicPage` 인라인 편집/업로드.

1. **카카오로 로그인/시작** — 별도 회원가입 없이 카카오 계정 한 번. (셀러 권한 추가도 같은 계정에 1탭, `SellerUpgradePanel`)
2. **핸들·프로필 자동/수동 설정** — 첫 진입 시 카카오 닉네임 기반으로 핸들 자동 생성(`generateUniqueHandle`). 미로그인/미설정 시 `/host/new` 카탈로그로 안내.
3. **커버 배너 / 프로필 이미지 업로드** — 본인 페이지에서 아바타·배너 클릭 → 이미지 업로드(최대 5MB, JPEG/PNG/WebP/GIF). 클라이언트 압축 후 `/api/seller/upload-image`로 URL 저장.
4. **이름·소개·SNS·카톡 링크 인라인 편집** — 텍스트를 클릭하면 그 자리에서 수정(이름/bio/Instagram/YouTube/카카오 채팅 링크).
5. **상품 연결** — 셀러 대시보드(`/seller`)에서 상품·식사권 등록 → 링크샵 상품/식사권 탭에 자동 노출.
6. **라이브·쇼츠·공구 연결** — 라이브 방송 시작/쇼츠 업로드/공동구매 개설 시 해당 탭에 자동 반영.
7. **링크 공유** — 우상단 공유 버튼으로 카톡/링크복사 → 인스타·유튜브 bio에 붙여넣기.
8. **정산** — 판매·후원·추천 보상이 누적, 출금 UI에서 출금(원천징수 자동, §7).

> [디자인 지시] 8단계를 가로 타임라인 또는 세로 번호 스텝으로. 1~4는 "개설", 5~8은 "운영·수익".

## 7. 수익 구조

> ⚠️ 모든 수치는 `src/shared/constants/policy.ts` (SSOT) 및 CLAUDE.md에서 추출. 어드민이 셀러별로 조정 가능한 값은 그 점을 명시.

| 항목 | 비율/금액 | 정산 대상 | 코드 근거 |
|---|---|---|---|
| **상품 판매 플랫폼 수수료** | 기본 **5%** (셀러별 어드민 조정 가능) | 플랫폼이 차감, 나머지 셀러 정산 | `COMMISSION_DEFAULTS.PLATFORM_FEE_PCT = 5`; CLAUDE.md "기본 5% 플랫폼 수수료" |
| **위탁 판매 셀러 commission** | **10%** | 위탁 상품 셀러 | `SELLER_COMMISSION_PCT = 10` |
| **후원(팬 후원)** | 수수료 **15%**, 최소 후원 **500딜** | 셀러가 후원금 수령(수수료 차감) | CLAUDE.md "후원 수수료 별도 15%", "최소 후원 500딜" |
| **친구 초대 보상** | 양쪽 각 **0.5%** | 추천인 + 피추천인 (first-time, self-refer 차단) | `REFERRAL_BONUS_BOTHSIDES_PCT = 0.5` |
| **매장 영입(크리에이터→매장) 보상** | **1.5%** | 영입자(크리에이터)에게 매출의 1.5% 적립 | `influencer-store-intro-commission.ts` `DEFAULT_STORE_INTRO_PCT = 1.5`; `platform_settings.influencer_store_intro_pct` |
| **큐레이터 핀 어필리에이트** | **1.0%** | 핀한 큐레이터에게만 적립 | `CURATOR_AFFILIATE_PCT = 1.0` |
| **공구 호스트 인센티브** | **1.0%** | 공구 호스트 | `HOSTING_DEFAULTS.HOST_INCENTIVE_PCT = 1.0` |
| **충전(딜)** | 1원 = 1딜, 수수료 0 | — | CLAUDE.md "충전: 1원=1딜 (수수료 없음)" |
| **원천징수** | 사업소득 **3.3%** / 기타소득 **8.8%** | 출금 시 자동 원천징수 | CLAUDE.md `WITHHOLDING_RATES`, `sellers.tax_type` |
| **최소 출금액** | **10,000원** | — | `WITHDRAWAL_DEFAULTS.MIN_AMOUNT = 10_000` |
| **셀러 승급 안내 임계** | 누적 정산 **500,000원** | 큐레이터→셀러(사업자) 자동 안내 | `SELLER_UPGRADE_THRESHOLD = 500_000` |

> [디자인 지시] 두 그룹으로: ① "내가 버는 법"(판매 5% 차감 후 정산, 후원 500딜~, 추천 0.5%, 영입 1.5%, 핀 1%) ② "출금"(최소 1만원, 원천징수 3.3%). 단순 도식 권장.

> [확인 필요] 후원 15% / 500딜 두 수치는 CLAUDE.md 정책 문구에서 확인됨. 코드 상수 파일(`policy.ts`)에는 별도 DONATION 상수가 grep으로 잡히지 않음 — 외부 소개서에 박기 전 최종 확정 권장.

  - 카카오 닉네임 기반 자동 생성, 충돌 시 숫자 suffix(base2…base99) → 랜덤 hex
  - 예약어(admin/api/me/login/user 등) 차단
  - 한글 등 비라틴 닉네임은 빈 슬러그 → `user{id}` 폴백 (generic `@user` 점유 방지)
- **운영 가이드**: 셀러 가이드 `/seller/guide` (DB `operation_guides`)
- **URL 공유 형태**: `live.ur-team.com/s/{핸들}` (짧음) / `/profile/{핸들}` (전체) / `/u/{핸들}` (큐레이터)

> [디자인 지시] 핸들 규칙은 "좋은 예 / 나쁜 예" 칩으로 (예: `@coffeelover` good / `@admin` 예약어 reject).

## 12. 섹션별 슬라이드 카피 초안

> 실제 사용 가능한 한국어 카피. 디자인 AI는 이 텍스트를 그대로 배치하거나 길이만 조정.

### 표지
- 제목: **"링크 하나로, 나만의 샵."**
- 부제: 유어딜 링크샵 — 상품·라이브·공동구매·후원을 한 페이지에
- 캡션: 무료 개설 · 카카오 로그인 한 번

### 문제
- 헤드: **"인스타 bio엔 링크가 딱 하나뿐."**
- 본문: 쇼핑몰도, 라이브도, 공구도, 연락처도 — 그 한 칸에 다 담을 수 없었죠.

### 솔루션
- 헤드: **"그 한 칸을 링크샵으로 바꾸세요."**
- 본문: 상품 그리드, 진행 중인 라이브, 친구와 함께 사는 공동구매, 쇼츠, 카톡 문의, 팬 후원까지 — 전부 한 페이지에.

### 크리에이터 혜택

- 불릿: 무료·즉시 개설 / 라이브·쇼츠 자동 연동 / 공동구매로 함께 구매 / 팬 후원 500딜부터 / 추천·영입 보상

### 구성 요소
- 헤드: **"한 페이지에 다 들어 있어요."**
- 본문: 프로필·배너 · 상품 · 라이브 · 쇼츠 · 식사권 · 후원 · 카톡 문의

### 개설 절차

- 헤드: **"파는 만큼, 모으는 만큼 정산."**
- 본문: 판매 정산(기본 수수료 5%) · 후원(최소 500딜) · 친구초대 0.5% · 매장영입 1.5% · 핀 1% · 출금 최소 1만원(원천징수 자동)

### 차별점
- 헤드: **"링크트리는 링크만, 우린 판매까지."**

## 14. FAQ (크리에이터)

1. **Q. 개설 비용이 있나요?**
   A. 없습니다. 카카오 로그인 한 번으로 무료 개설됩니다.
2. **Q. 사업자 등록이 꼭 필요한가요?**
   A. 시작은 필요 없습니다. 누적 정산이 일정 수준(50만원)에 이르면 사업자 전환을 자동 안내합니다.
3. **Q. 내 링크 주소는 어떻게 정해지나요?**
   A. 카카오 닉네임 기반으로 핸들이 자동 생성됩니다(영문 소문자·숫자·언더스코어 3~30자). 형태는 `live.ur-team.com/s/내핸들`.
4. **Q. 어떤 걸 팔 수 있나요?**
   A. 일반 상품, 맛집 식사권, 라이브 커머스 상품, 공동구매 등을 한 페이지에 올릴 수 있습니다.
5. **Q. 라이브 방송은 어떻게 연결되나요?**
   A. 셀러 대시보드에서 방송을 시작하면 링크샵에 자동으로 LIVE 배지와 라이브 탭에 표시됩니다. 방문자에게 실시간으로 "라이브 시작" 알림이 갑니다.
6. **Q. 후원은 어떻게 받나요? 최소 금액이 있나요?**
   A. 팬이 직접 후원할 수 있고 최소 후원은 500딜입니다. 후원에는 별도 수수료(15%)가 적용됩니다.
7. **Q. 수수료와 정산은 어떻게 되나요?**
   A. 상품 판매는 기본 5% 플랫폼 수수료가 차감된 뒤 정산됩니다(셀러별 조정 가능). 출금은 최소 1만원부터, 원천징수(사업소득 3.3% / 기타소득 8.8%)가 자동 적용됩니다.
8. **Q. 친구를 초대하거나 매장을 데려오면 보상이 있나요?**
   A. 친구 초대는 양쪽 각 0.5%, 매장(사장님) 영입은 매출의 1.5%가 적립됩니다.
9. **Q. 프로필/배너 이미지는 어떻게 바꾸나요?**
   A. 본인 페이지에서 아바타·배너를 클릭해 바로 업로드합니다(최대 5MB, JPEG/PNG/WebP/GIF).
10. **Q. 결제는 안전한가요?**
    A. 토스페이먼츠 표준 결제를 사용하며, 결제 금액은 서버에서 재계산합니다.
11. **Q. 매장 사장님도 쓸 수 있나요?**
    A. 네. 식사권/공동구매로 단골을 모으는 데 특히 적합합니다.
12. **Q. 인스타 외에 어디에 링크를 걸 수 있나요?**
    A. 유튜브·틱톡·블로그 bio 등 외부 링크를 받는 어디든 붙일 수 있습니다.
