## 0. 🎨 그라데이션 디자인 시스템 (덱 전체 공통 — 필수 준수)

> 이 덱은 **그라데이션을 시그니처 비주얼 언어**로 사용합니다. 표지·섹션헤더·CTA·스탯·퍼널 다이어그램에 적극 적용하고, 본문 카드/표는 subtle 하게 사용합니다. 이 덱은 **라이트 테마**(화이트/`#F4F5F7` 쇼핑·셀러 베이스) + 포인트 `#FF0033` 입니다. (유어딜 제안서 family 공통 토큰 — 다른 덱과 cross-deck 일관성 유지.)

### 0-1. 그라데이션 토큰 (verbatim — 그대로 사용)

| 토큰명 | CSS | 용도 |
|---|---|---|
| **시그니처 (Primary)** | `linear-gradient(135deg, #FF0033 0%, #FF3D6E 100%)` | CTA 버튼·스탯 숫자·핵심 강조·단계 번호 배지 |
| **warm (성장·매출)** | `linear-gradient(135deg, #FF0033 0%, #FF7A00 100%)` | 라이브커머스 에너지·매출/성장 지표·타임라인 진행선 |
| **deep (표지·전환)** | `linear-gradient(160deg, #C9184A 0%, #FF0033 60%, #FF4D6D 100%)` | 표지·섹션 전환 풀폭·신뢰 섹션 풀폭 배경 |
| **mesh 배경 (옅게)** | `radial-gradient(at 20% 20%, rgba(255,0,51,.16), transparent 42%), radial-gradient(at 85% 0%, rgba(255,122,0,.12), transparent 46%)` | 큰 면적 배경에 옅게 깔아 깊이감 (텍스트는 흰 카드 위) |
| **카드 subtle** | `linear-gradient(180deg, #FFFFFF 0%, #FFF1F3 100%)` + border `#FFE4E9` | 혜택/운영도구/FAQ 카드 면 |
| **아이콘 원 (포인트)** | `linear-gradient(135deg, #FF0033 0%, #FF3D6E 100%)` (지름 40~56px 원) | 카드 좌상단 아이콘 백그라운드 |
| **표 헤더행** | `linear-gradient(90deg, #FF0033 0%, #FF3D6E 100%)` | 수수료/정산/비교표 헤더행 (텍스트 흰색) |

### 0-2. 용도 규칙 (어디에 무엇을)

- **표지 / 섹션 전환 / 신뢰 풀폭**: deep 그라데이션 + mesh 오버레이. 큰 면적·드라마틱.
- **CTA 버튼 / 스탯 대형 숫자(5%, 3채널, 3.3%) / 단계 번호 배지**: 시그니처 그라데이션. + CTA 버튼은 glow (`box-shadow: 0 8px 28px rgba(255,0,51,.38)`).
- **라이브커머스 / 매출·성장 지표 / 타임라인 진행선**: warm 그라데이션 (에너지·역동).
- **본문 카드 면 (혜택·운영도구·FAQ)**: 카드 subtle (거의 흰색, 분홍 hint) — 본문 텍스트 가독성 우선.
- **표 헤더행 (수수료·비교)**: 표 헤더행 그라데이션 + 흰 텍스트. 데이터 셀은 흰 배경.
- **큰 빈 배경**: mesh 를 옅게 (절대 진하게 X — 본문 가독성 해침).

### 0-3. 접근성 (필수)

- 그라데이션 **진한 부분(빨강~딥핑크/주황) 위에만 흰 텍스트** 허용. 밝은 부분(`#FFF1F3`, `#FF7A00` 의 밝은 끝) 위에는 흰 텍스트 금지 → 대비 부족.
- deep/mesh 위에 텍스트를 얹을 땐 **스크림**(`rgba(0,0,0,.28)` 오버레이 또는 텍스트 뒤 어두운 그라데이션 vignette) 추가로 WCAG AA(4.5:1) 확보.
- subtle 카드(`#FFF1F3`) 위 본문은 **항상 `text-gray-900`/`text-gray-700`** (흰 텍스트 금지).
- 표 헤더행 그라데이션 위 텍스트는 흰색 + `font-weight 600` 이상.
- 그라데이션은 **장식**, 정보 전달은 텍스트/숫자로. 색만으로 의미 구분 금지(색맹 대응).

### 0-4. Do / Don't

- ✅ Do: 표지·CTA·스탯·퍼널에 시그니처/딥/warm 풀강도. 카드는 subtle. 한 슬라이드 1~2개 그라데이션 면 + 1개 포인트.
- ✅ Do: 그라데이션 각도 통일(135deg 시그니처/warm, 160deg deep, 90deg 표헤더).
- ❌ Don't: 본문 텍스트 블록 전체를 진한 그라데이션 위에 배치(가독성↓).
- ❌ Don't: 한 슬라이드에 3개 이상 강한 그라데이션 면(시각 과부하).
- ❌ Don't: 그라데이션 위 흰 텍스트를 밝은 끝(분홍/주황 밝은 쪽)에 배치.
- ❌ Don't: 대시보드/표 데이터 셀 배경에 그라데이션(숫자 가독성↓ — 흰 배경 유지).

### 0-5. 적용 빈도 가이드

- **풀강도 그라데이션 면** (deep/시그니처 큰 면): 슬라이드당 최대 1개 (표지·CTA·신뢰·섹션전환).
- **subtle 카드 그라데이션**: 카드형 슬라이드(혜택·운영도구·FAQ)에 반복 OK.
- **포인트 그라데이션** (아이콘 원·배지·숫자·진행선): 슬라이드당 여러 개 OK (작은 면적).
- **mesh 배경**: 흰 베이스 슬라이드에 옅게 상시 깔기 OK (깊이감).

`[디자인 지시: 그라데이션]` 위 토큰을 SSOT 로 — 모든 `[디자인 지시]` 콜아웃의 그라데이션 지정은 이 §0 토큰명을 참조한다(임의 색 추가 금지). 각 섹션의 `[디자인 지시: 그라데이션]` 콜아웃을 함께 적용할 것.

## 1. 문서 목적 & 타깃

### 문서 목적
유어딜(live.ur-team.com)에 **상품을 온라인으로 입점·판매하려는 사업자**에게, 유어딜이 제공하는 3가지 판매 채널(쇼핑 / 라이브·쇼츠 커머스 / 링크샵)과 빠른 입점·자동 정산 구조를 설득력 있게 전달하는 **입점 제안서 덱**을 제작하기 위한 콘텐츠 브리프.

### 타깃 (Primary)
- **브랜드 / 제조사** — 자사 상품을 신규 판매 채널에 노출·판매하려는 곳
- **온라인 셀러 / 위탁판매 사업자** — 기존 오픈마켓 외 추가 채널을 찾는 사업자
- **오프라인 매장 사장님(store_owner)** — 매장 상품/교환권을 온라인으로 판매하려는 곳

### 부 타깃 (Secondary)
- **인플루언서 / 크리에이터(influencer)** — 라이브·쇼츠·링크샵으로 상품을 판매·홍보하고 성과 보상을 받으려는 셀러

> `[확인 필요]` 셀러 역할 값(`seller_type`)은 코드상 `influencer` / `store_owner` / `both` 3종 (출처: `src/shared/seller-roles.ts`). 사용자 대면 라벨은 각각 "🎤 크리에이터" / "🏪 매장 사장님" / "🎤🏪 크리에이터 + 매장".

`[디자인 지시]` 표지에 타깃 3종(브랜드/제조사 · 온라인 셀러 · 매장 사장님)을 아이콘 3개로 시각화. 부 타깃(크리에이터)은 별도 보조 배지로.

`[디자인 지시: 그라데이션]` **표지 = deep 그라데이션 풀폭** (`linear-gradient(160deg, #C9184A 0%, #FF0033 60%, #FF4D6D 100%)`) + 그 위에 mesh 오버레이로 깊이감. 헤드라인 흰 텍스트는 좌하단 `rgba(0,0,0,.28)` 스크림 위에 배치(AA 확보). 타깃 아이콘 3개는 흰 라운드 칩 안에 두고 아이콘만 시그니처 그라데이션. 부 타깃(크리에이터) 보조 배지는 warm 그라데이션 pill. 하단 얇은 띠에 warm 그라데이션 라인으로 마감.

## 1-A. 📍 이 소개서의 범위 & 경계 (먼저 읽어주세요)

> 유어딜은 하나의 코드베이스/DB 위에서 여러 사업 라인을 운영합니다. 이 소개서는 그중 **"셀러/브랜드가 온라인으로 파는 모든 것" = B2C 온라인 판매**를 다룹니다. 인접한 다른 소개서와의 경계를 먼저 명확히 합니다.

### 이 소개서가 다루는 것 (IN SCOPE)
**셀러(브랜드/제조사/온라인 셀러/매장 사장님/크리에이터) → 소비자(B2C)** 의 온라인 판매 전 영역.

- **3대 판매 채널** — 쇼핑(`/browse`) · 라이브/쇼츠 커머스(`/live`, `/shorts`) · 링크샵(크리에이터 공개샵).
- **라이브 방송 수익화 모델** — 라이브 경매(`live_auctions`) · 타임딜(`time_deals`) · 펀딩(`live_fundings`) · 선물하기(`gifts`).
- **온라인 판매 모델 다양화** — 일반 진열 상품 · 셀러 온라인 즉시판매형 공구(B2C) · 번들/묶음 · 디지털 상품 · 교환권/기프티콘.
- **입점 후 운영 도구 전체** — 상품/재고/대량등록 · 주문/배송/반품 · 정산(사업자 게이팅)/원천징수 · 분석/리포트 · 리뷰 · 알림톡/마케팅 · 광고 슬롯/부스트 · 위탁/프록시 · YouTube 성장 · 셀러 이전.
- **브랜드↔셀러 매칭** — 캐스팅 마켓플레이스(광고주↔셀러 협업 중개, `casting_requests`).
- **에이전시 매니징 옵션** — 입점 셀러가 매니징을 위임할 수 있는 경로(보조 §9-A, 전체는 에이전시 전용 자료 참조).

### 이 소개서가 다루지 **않는** 것 (OUT — 다른 소개서)
| 영역 | 무엇 | 담당 소개서 |
|---|---|---|
| **B2B 도매 유통** | 제조사↔유통사 무재고 사입, 등급제 공급가, OEM/ODM, 거래명세서 (utongstart.com) | **도매몰 소개서** |
| **오프라인 매장 공구 / 동네딜** | 매장 선결제 손님 확보 → 교환권 즉시발급(당근식), 맛집·숙소·예약 | **오프라인 공구 소개서** |
| **크리에이터 수익화 올인원** | 링크샵 자체 구조·큐레이터 핀·후원·추천 어필리에이트의 primary home | **링크샵 소개서** |

> 단, 위 3개와 **겹치는 접점**(링크샵을 판매 채널③으로, 교환권을 온라인 판매 모델로, 캐스팅을 셀러 협업으로)은 본 소개서에서도 셀러 관점으로 다룹니다.

### ⚠️ "공동구매" 명칭 충돌 정리 (필수 — 혼동 차단)
유어딜에는 이름이 "공구"인 라인이 둘 있습니다. **이 소개서가 다루는 공구는 ①번입니다.**

| 구분 | 이 소개서의 "공구" ① | 오프라인 동네딜 ② (다른 소개서) |
|---|---|---|
| 성격 | **셀러 온라인 즉시판매형 공구 (B2C)** | 오프라인 매장 선결제 손님 확보 |
| 경제 모델 | 즉시판매 — **인원 무관 단일가**(최대 tier 할인 즉시 적용). 이름만 "공구", 경제는 즉시판매 (출처 `docs/design/groupbuy-instant-sale.md`) | 매장 결제 → 교환권 발급 → 현장 사용 |
| 위치 | `/group-buy`, `/seller/group-buy` (온라인 상품) | `/group-buy` 동네딜 탭, `/meal-vouchers`, `/restaurant-map` |
| 배송 | 실물 배송 / 디지털 / 교환권 | 매장 현장 사용 (QR) |

> 즉 "여기서 말하는 공구 = **온라인에서 단일가로 즉시 팔리는 셀러 상품**"이며, 오프라인 매장이 손님을 모으는 동네딜 공구는 **별도 오프라인 공구 소개서**가 다룹니다.

`[디자인 지시]` 이 섹션은 **벤다이어그램 또는 2분할 경계도**로 시각화 — 중앙에 "본 소개서 = B2C 온라인 판매" 강조 원, 주변에 도매몰/오프라인공구/링크샵 3개 인접 원을 흐리게 배치하고 겹치는 영역(링크샵·교환권·캐스팅)만 점선으로 연결. "공구 2종 구분표"는 별도 비교 카드.

`[디자인 지시: 그라데이션]` 중앙 "B2C 온라인 판매" 원만 **시그니처 그라데이션** 풀 채움 + glow, 인접 3개 원은 무채색 그라데이션(`linear-gradient(160deg,#E9ECEF,#CED4DA)`)으로 흐리게(대비로 본 소개서 강조). 겹침 점선은 warm 그라데이션 stroke. "공구 2종 구분표"는 헤더행만 표 헤더행 그라데이션, 데이터 셀 흰 배경.

## 2. 한 줄 요약 & 엘리베이터 피치

**한 줄 요약**

## 3. 시장 기회 / 문제 정의

### 기존 오픈마켓의 한계 (문제)
| 문제 | 설명 |
|---|---|
| **노출 경쟁 심화** | 검색·랭킹 알고리즘 의존 → 신규/중소 셀러는 첫 페이지 노출이 어려움 |
| **낮은 전환율** | 진열형 상세페이지만으로는 구매 설득력 부족, 단순 스펙 비교 경쟁 |
| **광고비 부담** | 노출을 위해 키워드/디스플레이 광고비를 지속 지출해야 함 |
| **브랜드/스토리 전달 한계** | 정적 이미지·텍스트만으로 제품 가치·사용법을 충분히 전달 못 함 |
| **고객 관계 부재** | 1회성 거래 위주, 재구매·팬덤 형성 어려움 |

### 유어딜이 보는 기회
- **라이브커머스**: 실시간 방송으로 제품 시연·Q&A → 높은 전환·체류
- **크리에이터 커머스**: 인플루언서/크리에이터가 자기 팬에게 직접 추천·판매 → 신뢰 기반 구매 (성과형 보상으로 셀러가 자발적으로 밀어줌)
- **쇼츠(숏폼)**: 짧은 영상으로 상품을 바이럴 노출 (`/shorts`)
- **통합 채널**: 한 번 등록한 상품이 3채널에 동시 노출 → 채널별 중복 작업 제거

`[디자인 지시]` 좌(문제: 회색톤 오픈마켓 진열대) vs 우(기회: 라이브+크리에이터+숏폼) Before/After 대비 레이아웃.

`[디자인 지시: 그라데이션]` **문제→솔루션 대비를 그라데이션으로 표현**: 좌(문제)는 무채색 그라데이션(`linear-gradient(160deg,#E9ECEF,#CED4DA)`)으로 칙칙하게, 우(기회)는 시그니처 그라데이션(`linear-gradient(135deg,#FF0033,#FF3D6E)`)으로 생기있게. 두 면 사이 전환 화살표/구분선에 warm 그라데이션. 문제 표는 흰 카드(subtle), 각 행 좌측에 회색 dot — 우측 기회 리스트는 시그니처 아이콘 원.

## 4. 솔루션 = 유어딜 판매 채널 3종

상품 1건 등록 → 아래 3개 채널에서 동시에 판매·노출.

### 채널 ① 쇼핑 (`/browse`)
- 쇼핑 피드/그리드 형태로 상품 진열 (출처: `src/pages/BrowsePage.tsx`)
- 카테고리·검색·정렬 필터, 상품 상세(`/products/:id` — `ProductDetailPage.tsx`)
- 장바구니(`CartPage`) → 결제(`CheckoutPage`) → 주문(`orders.routes.ts`) 표준 e-commerce 플로우
- 이미지 자동 최적화(도미넌트 컬러 placeholder, lazy-load, CF 이미지 리사이즈)로 빠른 로딩
- `[디자인 지시]` 화이트 테마 쇼핑 그리드 목업 (`grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5`)
- `[디자인 지시: 그라데이션]` 채널① 헤더 칩에 시그니처 그라데이션 + "쇼핑" 라벨. 목업 프레임(브라우저/폰 베젤) 상단 바를 시그니처 그라데이션으로. 그리드 카드 자체는 흰색(데이터 가독성), 가격 뱃지만 시그니처 그라데이션 pill.

### 채널 ② 라이브 / 쇼츠 커머스 (`/live/*`, `/shorts`)
- 크리에이터가 **라이브 방송**으로 상품을 실시간 판매 (출처: `youtube-live.routes.ts` — `/live/create`, `/live/:id/start`, 채팅 `/live/:id/chat`, 상품 오버레이)
- **멀티플랫폼 송출**: YouTube Live `available`, RTMP 인제스트 지원(OBS/Prism 호환). TikTok·네이버 치지직·SOOP(아프리카TV)는 `coming_soon` (출처: `multi-platform.routes.ts` DESTINATIONS — `eta: 2026 Q3~Q4`)
- **쇼츠(숏폼)**: 9:16 세로 영상으로 상품 노출 (`/shorts`, `SellerShortsPage`)
- 방송 중 상품 오버레이/핀으로 즉시 구매 유도
- `[디자인 지시]` 9:16 모바일 라이브 화면 + 상품 카드 오버레이 목업. 송출 플랫폼 로고 라인(YouTube=지원, TikTok/치지직/SOOP=Coming Soon 배지).
- `[디자인 지시: 그라데이션]` 채널② = **warm 그라데이션**(에너지). 헤더 칩 warm, 9:16 폰 베젤 outer glow warm. 라이브 화면 하단 "구매" 오버레이 버튼은 warm 그라데이션 + glow. 9:16 화면 내부 라이브 영상 위 텍스트(상품명/가격)는 하단 vignette 스크림 위에. "Coming Soon" 배지는 회색(warm 아님 — 미지원 명확히).

### 채널 ③ 링크샵 (크리에이터 공개샵)
- 크리에이터/셀러 개인 공개 페이지(`SellerPublicPage` — `/profile/:handle`, `/s/:id`)
- 크리에이터가 상품을 "핀(추천)" 으로 큐레이션 → 자기 팬·팔로워에게 노출 (링크샵)
- 핀 추천 클릭/구매 시 크리에이터에게 어필리에이트 보상 적립 (성과형)
- `[디자인 지시]` 크리에이터 프로필 + 추천 상품 그리드(링크샵) 목업. 다크 테마(유저 대면).
- `[디자인 지시: 그라데이션]` 채널③ = **deep 그라데이션**. 헤더 칩 deep. 다크 목업이므로 프로필 배너/아바타 링에 deep 그라데이션(`linear-gradient(160deg,#C9184A,#FF4D6D)`) — 다크 위에서 색이 살아남. "핀(추천)" 뱃지는 시그니처 그라데이션 pill. 어필리에이트 보상(1.0%) 숫자는 시그니처 그라데이션 텍스트.

`[디자인 지시]` **3채널 퍼널 다이어그램** — 중앙 "상품 1건 등록" → 3갈래(쇼핑/라이브·쇼츠/링크샵) → "구매·정산". 이 슬라이드가 솔루션 섹션의 핵심.

`[디자인 지시: 그라데이션]` **3채널 퍼널 다이어그램 = 덱의 그라데이션 하이라이트.**
> - 중앙 **"상품 1건 등록" 노드**: 시그니처 그라데이션 풀 채움 원/필 (`linear-gradient(135deg,#FF0033,#FF3D6E)`) + glow `box-shadow:0 8px 28px rgba(255,0,51,.38)`, 흰 텍스트.
> - **3채널 노드**(각각 그라데이션 채움 + 흰 아이콘):
>   - ① 쇼핑 = 시그니처 `linear-gradient(135deg,#FF0033,#FF3D6E)`
>   - ② 라이브·쇼츠 = warm `linear-gradient(135deg,#FF0033,#FF7A00)` (에너지 강조)
>   - ③ 링크샵 = deep `linear-gradient(160deg,#C9184A,#FF4D6D)`
> - **연결선(엣지)**: 중앙→채널은 시그니처→해당채널색으로 흐르는 그라데이션 stroke(`stroke:url(#gradEdge)`, 두께 3~4px), 흐름 방향 화살표. 채널→"구매·정산"은 warm 그라데이션 수렴선.
> - **"구매·정산" 종착 노드**: deep 그라데이션 + mesh, 흰 텍스트(스크림).
> - 다이어그램 전체 배경은 흰색 + 옅은 mesh. 노드 라벨/설명 텍스트는 노드 아래 흰 영역에(그라데이션 위 직접 본문 텍스트 금지).

## 6. 입점 절차 (스텝 바이 스텝)

> 출처: `seller-registration.routes.ts`(가입·사업자검증), `BulkUploadModal.tsx`(CSV), `cafe24.routes.ts`(연동), `seller-onboarding.routes.ts`(부트캠프), `seller-settlements.routes.ts`(정산)

1. **가입 / 셀러 등록** — 셀러 가입(`POST /api/seller/register`). 필수: username, email, password, name, business_name, business_number, phone, youtube_email. 역할(`seller_type`) 선택: 크리에이터 / 매장 사장님 / 겸업.
2. **사업자 등록번호 검증** — 사업자번호 형식 검증(`XXX-XX-XXXXX`). 등록 직후 status=`pending` → 국세청(NTS) 비동기 진위확인 → 일치 시 `approved` 자동 승급 (대표자명·개업일 제공 시). (출처: `seller-registration.routes.ts`)
3. **상품 등록** — 3가지 방법:
   - **개별 등록** (`POST /api/products`, `SellerProductNewPage`)
   - **CSV 대량등록** (`BulkUploadModal` → `POST /api/bulk-upload/upload`). 한글 헤더 매핑 지원: `상품명*`, `판매가격*`, `재고수량*`, `카테고리*`, `상품타입*`, `이미지URL`, `라이브특가`, `옵션타입/옵션값/옵션추가금액/옵션재고` 등.
   - **Cafe24 외부몰 연동** — OAuth 인증 후 상품 자동 동기화(`POST /api/.../cafe24/sync`). `cafe24_product_map` 으로 중복 방지, 가격/공급가/소비자가 매핑. `[확인 필요]` 현재 Cafe24 연동 트리거는 **어드민 권한(requireAdmin)** 으로 보호됨 — 셀러 셀프 연동인지 어드민 대행인지 운영 정책 확인 필요.
4. **채널 노출** — 등록 상품이 쇼핑(`/browse`) 노출 + 크리에이터 라이브/쇼츠/링크샵 판매 대상으로 편입.
5. **주문 / 배송** — 구매 발생 시 주문 생성(`orders.routes.ts`), 셀러 주문 관리(`SellerOrdersPage`), 배송 추적(tracker.delivery 연동, 6시간 주기 sync).
6. **정산** — 정산 신청(`POST /api/seller/settlements/request`) → 어드민 승인 → 송금. 원천징수 자동 처리.

`[디자인 지시]` **입점 타임라인** — 6스텝 가로 타임라인. "상품 등록" 스텝에서 3갈래(개별/CSV/Cafe24) 분기 시각화. 사업자 검증은 "자동 승인" 배지로 강조.

`[디자인 지시: 그라데이션]` **타임라인 = 단계 번호 배지 + 진행선 그라데이션.**
> - 각 스텝 **번호 배지(1~6, 원형 36~44px)**: 시그니처 그라데이션 채움 + 흰 숫자 + 작은 glow.
> - 스텝들을 잇는 **진행선(가로 레일)**: warm 그라데이션(`linear-gradient(90deg,#FF0033,#FF7A00)`) 좌→우 흐름, 두께 4px. "상품 등록" 분기점에서 3갈래로 갈라질 때 각 분기선도 warm 그라데이션 가늘게.
> - **"자동 승인" 배지**: warm 그라데이션 pill + 흰 텍스트(성장/긍정 신호).
> - 스텝 설명 카드는 흰색(subtle), 본문 `text-gray-700`. 배경은 옅은 mesh.

`[디자인 지시]` **수수료 비교표** — 유어딜 기본 5% 를 큰 숫자로. 세금 표(3.3%/8.8%)는 별도 박스. "사업자 등록 시 원천징수 면제" 를 강조 콜아웃. 경쟁사 비교는 §11 참조.

`[디자인 지시: 그라데이션]` **수수료/정산/세금 표 = 표 헤더행 그라데이션.**

> - **"5%" 대형 숫자**: 시그니처 그라데이션 텍스트 클립(`background:linear-gradient(135deg,#FF0033,#FF3D6E)` + `-webkit-background-clip:text;color:transparent`). 슬라이드 한가운데 히어로 숫자.
> - **"사업자 등록 시 원천징수 면제" 강조 콜아웃 박스**: 카드 subtle 면(`linear-gradient(180deg,#FFFFFF,#FFF1F3)` + border `#FFE4E9`) + 좌측 4px warm 그라데이션 액센트 바. 본문 `text-gray-900`/`text-gray-700`.
> - 세금 표의 3.3%/8.8% 숫자는 시그니처 그라데이션 텍스트로 포인트.
> - 슬라이드 배경은 흰색 + 옅은 mesh. 풀강도 그라데이션 면은 0개(표 헤더행 + 히어로 숫자만) — 데이터 슬라이드는 가독성 우선.

`[디자인 지시: 그라데이션]` **라이브커머스 차별화 = warm 그라데이션(에너지·역동).**

## 8-A. 라이브커머스 = 유어딜의 1급 판매 채널 (대폭 강화)

> 유어딜의 **간판 사업**은 라이브커머스입니다. 입점사 입장에서 라이브는 "있으면 좋은 보조"가 아니라 **전환율을 끌어올리는 핵심 매출 채널**입니다. 출처: `youtube-live.routes.ts`, `multi-platform.routes.ts`, `auction.routes.ts`, `timedeal.routes.ts`, `funding.routes.ts`, `gifts.routes.ts`.

### 8-A-1. 왜 라이브로 팔면 더 잘 팔리나 (입점사 가치)
- **실시간 시연·Q&A**로 진열형 상세페이지가 못 하는 "설득"을 함 → 체류·전환↑.
- **크리에이터가 자기 팬에게** 직접 방송 판매 → 신뢰 기반 구매. 성과형 보상이라 셀러가 자발적으로 밀어줌.
- **상품 핀/오버레이**로 방송 보다가 한 탭에 구매 — 이탈 없는 즉시 전환.
- 입점사는 상품만 등록하면, **방송은 크리에이터가** 한다(매장 사장님은 매장 상품 등록 중심, 송출은 `canBroadcast()` 역할만).

### 8-A-2. 라이브 판매 작동 방식 (스텝)
1. 크리에이터가 라이브 생성/시작(`/live/create`, `/live/:id/start`) — YouTube Live 즉시, RTMP 인제스트로 OBS/Prism 송출.
2. 방송 화면에 입점 상품을 **핀/오버레이**로 노출(상품 오버레이 API).
3. 실시간 채팅(`/live/:id/chat`)으로 Q&A·구매 유도.
4. 시청자가 오버레이 "구매" → 표준 주문/결제 흐름으로 연결(쇼핑 채널과 동일 정산).
5. 라이브 분석(`SellerLiveAnalyticsPage`)으로 시청→구매 전환 측정.

### 8-A-3. 라이브 방송 중 판매(수익화) 모델 4종 — 같은 상품, 더 다양한 파는 법
> 입점 상품을 라이브에서 **여러 판매 방식**으로 굴릴 수 있습니다. (각 표기 [확인 필요]는 노출 route/운영 활성 여부.)

| 모델 | 무엇인지 | 입점사 가치 | 작동 방식 (1~2줄) | 근거 |
|---|---|---|---|---|
| **라이브 경매** | 방송 중 실시간 입찰로 가격이 오르는 경매 판매 | 한정/프리미엄 상품의 단가 극대화 + 시청자 몰입·체류↑ | 셀러가 경매 생성(시작가·최소 입찰단위·시간) → 시청자 입찰, **입찰 시 딜 보증금 hold** → 종료 시 최고가 낙찰(`current_price`만 조건부 갱신으로 동시입찰 race 방지) | `auction.routes.ts` (`live_auctions`, `auction_holds`, 시작가/`min_increment`/`duration_seconds`) |
| **타임딜** | 방송 한정 시간/수량 할인(1~99%) | 즉시 구매 압박(FOMO) → 짧은 시간 폭발적 전환 | 셀러가 원가 대비 할인율·최대 수량·지속시간 설정 → `deal_price = price×(100−할인)/100` 자동 계산, 시간/수량 소진 시 자동 종료(`sold_out`/`ended`) | `timedeal.routes.ts` (`time_deals`, `discount_percent` 1~99, `max_claims`) |
| **타임딜 공동구매 모드** | 타임딜에 목표 인원 도달 시 **추가 할인** 발동 | 시청자끼리 모아 사게 만드는 바이럴 + 추가 매출 | `is_group_buy=1` + `target_participants` 설정 → 목표 달성 시 `bonus_discount_percent` 적용, status `achieved` | `timedeal.routes.ts` (`is_group_buy`, `target_participants`, `bonus_discount_percent`) |
| **선물하기** | 시청자가 상품을 다른 사람에게 선물 발송 | 객단가↑ + 수신자 신규 유입(선물 알림톡으로 바이럴) | 발송자 결제 → `gifts` 생성(claim_token) → 수신자에게 카카오 알림톡 claim URL → 수신자 주소 입력·수령 | `gifts.routes.ts` (`gifts`, claim_token, status paid→claimed→shipped) |

### 8-A-4. 쇼츠(숏폼) 커머스
- 9:16 세로 영상으로 상품을 **바이럴 노출**(`/shorts`, `SellerShortsPage`) — 방송이 아니어도 상시 콘텐츠 트래픽.
- 입점 상품을 쇼츠에 태깅 → 피드에서 바로 상품 진입.

### 8-A-5. 멀티 플랫폼 송출 (현황 정확)

### 8-A-6. 라이브 일정·리마인더 / 운영 도구
- 라이브 생성/시작/종료, 실시간 채팅, 상품 오버레이/핀.
- 방송 예고·팔로워 알림(`broadcast-notify`, `live-notify-followers`) — 시청 예약·리마인더로 동접 확보. `[확인 필요]` 정확한 예약/리마인더 UI 노출.
- 좀비 세션 복구·진단 endpoint(송출 안정성).

## 8-B. 캐스팅 마켓플레이스 (브랜드 ↔ 셀러 매칭)

> 입점만 하고 끝이 아니라, **브랜드(광고주)가 어울리는 크리에이터/셀러를 직접 섭외**해 협업·광고 방송을 만드는 중개 마켓입니다. 출처: `casting.routes.ts` (`advertisers`, `casting_requests`).

### 무엇인지
광고주(브랜드)가 캠페인(상품 카테고리·예상매출·제안비·라이브 희망일)을 제시하면, 어드민 검토를 거쳐 적합 셀러에게 캐스팅 제안이 전달되고, 셀러가 수락/거절합니다.

### 입점사(브랜드) 가치
- 자사 상품을 **잘 팔 크리에이터를 직접 캐스팅** → 라이브/쇼츠 협업 광고로 확장.
- 셀러 입장에서도 **광고/협업 수익**(제안비 `proposed_fee`)이 들어오는 새 수익원.

### 작동 방식 (스텝)
1. 광고주 가입(셀프 또는 어드민 등록) → 어드민이 `active` 승인.
2. 캐스팅 신청 생성(`campaign_title`, `proposed_fee`, `expected_revenue`, `proposed_live_date`) → `admin_review` → 어드민 검토 후 `sent_to_seller`.
3. 셀러가 받은 캐스팅 확인(`/api/seller/castings`) → 수락(`accepted`)/거절(`rejected`).
4. 수락 시 외부 거래 진행(결제는 별도) → 완료 시 어드민이 `completed`.

> `[확인 필요]` 캐스팅 제안비(`proposed_fee`)에 대한 플랫폼 수수료/중개료 정책은 코드 상수에 직접 노출되지 않음 — 운영 정책 확인 필요.

`[디자인 지시]` **광고주 → (어드민 검토) → 셀러 → 협업 라이브** 4노드 매칭 흐름도. "브랜드가 셀러를 캐스팅" 헤드라인.

`[디자인 지시: 그라데이션]` 매칭 흐름도 노드 = 시그니처 그라데이션 채움 + 흰 아이콘, 어드민 검토 게이트 노드만 deep 그라데이션(중개/신뢰). 엣지 stroke 는 warm 그라데이션 흐름선. 제안비/예상매출 수치는 시그니처 그라데이션 텍스트.

## 8-D. 교환권 / 기프티콘 온라인 판매

> 실물 배송 없이 **교환권(기프티콘)을 발행·판매**하고, 구매자는 QR/코드로 사용합니다. 매장 상품·체험·디지털 혜택을 온라인에서 즉시 파는 모델입니다. 출처: `group-buy-voucher.routes.ts` (`vouchers`), `gifts.routes.ts`.

### 무엇인지 / 입점사 가치
- 입점사(특히 매장 사장님)는 **재고/배송 없이** 교환권을 온라인에 올려 즉시 판매 → 현금 선확보.
- 구매자는 모바일 교환권을 받아 **QR/코드 + PIN**으로 사용(현장 또는 온라인). 선물하기와 결합해 바이럴.

### 작동 방식 (1~2줄)
- 교환권 상품 구매 → `vouchers` 발급(코드 발행) → 사용 시 `POST /:code/use`(PIN 검증, **원자적 CAS**로 1회만 사용, 사용 로그). 부분 환불(`partial-refund`)도 지원.
- 카테고리는 동적(식사권/미용/숙소/기타 등 — 하드코딩 제거).

> `[확인 필요]` **KT alpha 등 외부 기프티콘 발행 사업자 연동은 코드 상 없음** — 현재 교환권/기프티콘은 **유어딜 자체 발행·QR 사용 체계**(`vouchers`)입니다. 외부 기프티콘몰 유통 연동은 미구현으로 보임(운영 정책/로드맵 확인 필요).

`[디자인 지시]` 모바일 교환권 카드(QR + 코드 + PIN) 목업 + "발행→판매→QR 사용" 3스텝. 매장 사장님 아이콘 강조.

`[디자인 지시: 그라데이션]` 교환권 카드 목업은 흰/다크 실제 색 유지, 상단 띠만 시그니처 그라데이션. 3스텝 번호 배지 시그니처+glow, 진행선 warm.

## 9-A. 에이전시 매니징 옵션 (보조 — 혼자 다 하기 버거운 입점사에게)

> 입점 셀러는 운영을 **직접** 할 수도, **에이전시에 매니징을 위임**할 수도 있습니다. 이 소개서(셀러 직접 입점) 관점에서 "선택지"로만 짧게 다루며, **에이전시 사업 전체(매니징 조직 대상)는 별도 에이전시 전용 자료**를 참조하세요. 출처: `src/features/agency/api/*`, `policy.ts` (`AGENCY_SHARE_PCT`, `INFLUENCER_INTRO_SHARE_PCT`).

### 무엇인지 / 셀러 가치
- 방송·상품·정산·마케팅을 혼자 다 하기 버거운 입점사는 **에이전시에 소속/위임**해 스케줄·캠페인·인센티브 관리를 맡길 수 있음.
- 에이전시는 소속 셀러의 KPI·정산·랭킹을 관리하고, 매장/셀러를 영입·매칭(캐스팅과 별개의 매니징 영입).

### 정산 분배 (참고)
- **에이전시 입점 분배 30%** (`AGENCY_SHARE_PCT = 30`, platform_fee 중) / **인플루언서 입점 분배 20%** (`INFLUENCER_INTRO_SHARE_PCT = 20`). 즉 입점 경로(에이전시/크리에이터 영입)에 따라 플랫폼 수수료 일부가 분배됨 — **입점사가 추가로 내는 비용이 아니라 플랫폼 수수료 내부 분배**.
- 매니징 위임/회수는 셀러 이전(`seller-transfer`) 흐름으로 처리(§9 운영도구 참조).

> 전체 에이전시 기능(소속 셀러 관리·캠페인·인센티브·KPI·PK 배틀·공개 페이지)은 본 소개서 범위 밖 — 에이전시 대상 별도 소개서/자료로 안내.

`[디자인 지시]` "직접 운영" vs "에이전시 위임" 2갈래 선택지 카드. 비용이 아니라 "선택지"임을 강조하는 톤.

`[디자인 지시: 그라데이션]` 2갈래 카드 = 카드 subtle 면, 아이콘 원 시그니처. "30%/20%" 분배 수치는 시그니처 그라데이션 텍스트(단, "추가 비용 아님" 캡션 회색으로 명확히). 풀강도 면 없음(보조 섹션).

## 10. 신뢰 / 안전

- **결제 안전성**: 토스페이먼츠 V2 결제 — 5개 결제 흐름(충전/주문/공구/숙소/교환권)이 단일 SSOT(`toss-gateway.ts confirmTossPayment`)를 통해 처리. circuit breaker / idempotency-key / 금액 서버 재검증 자동. 결제 금액은 클라이언트 값 절대 신뢰 안 함.
- **사업자 검증**: 가입 시 국세청(NTS) 사업자번호 진위확인 비동기 검증(`seller-registration.routes.ts`). 현금 정산은 검증된 사업자만(412 게이팅).
- **환불 / CS 흐름**: 토스 환불 자동(최대 5회 재시도, 지수 backoff), 환불 시 커미션/공급정산 역전 처리(`returns.routes.ts` reverse 로직). 분쟁 24시간 미처리 시 어드민 escalation, 재발 매장/유저 임계값 모니터링(`REFUND_POLICY`).
- **보안 규칙**: 권한 검증(IDOR 방지), rate limit, Turnstile bot 방어, 에러 메시지 generic 처리(계정 enumeration 방지).

`[디자인 지시]` "토스페이먼츠 안전결제" + "국세청 사업자 검증" + "자동 환불/CS" 3개 신뢰 배지. 자물쇠/방패 아이콘.

`[디자인 지시: 그라데이션]` **신뢰 섹션 = deep 그라데이션 풀폭(드라마틱·믿음).**
> - 슬라이드 배경 풀폭에 **deep 그라데이션**(`linear-gradient(160deg,#C9184A 0%,#FF0033 60%,#FF4D6D 100%)`) + mesh 오버레이로 깊이감. 헤드라인 흰 텍스트는 `rgba(0,0,0,.28)` 스크림(또는 vignette) 위에 배치 → WCAG AA(4.5:1) 확보.
> - **3개 신뢰 배지(토스/NTS 검증/자동환불)**: deep 배경 위에 **흰 라운드 카드** 안에 두고(본문 가독성), 카드 내 자물쇠/방패 아이콘 원만 시그니처 그라데이션 채움 + 흰 아이콘. 카드 본문 `text-gray-900`/`text-gray-700`.

> - 하단 마감 띠는 warm 그라데이션 얇은 라인.

## 15. FAQ (판매자) — 실제 정책 근거

1. **Q. 입점 수수료가 어떻게 되나요?**
   A. 기본 플랫폼 수수료는 매출의 **5%** 입니다(`COMMISSION_DEFAULTS.PLATFORM_FEE_PCT=5`). 셀러별로 어드민이 `sellers.commission_rate` 로 조정할 수 있습니다. 위탁판매는 별도(10%) 케이스가 있습니다.

2. **Q. 상품을 빠르게 많이 등록할 수 있나요?**
   A. CSV 대량등록(한글 헤더 매핑)과 Cafe24 외부몰 연동 동기화를 지원합니다(`BulkUploadModal`, `cafe24.routes.ts`).

3. **Q. 정산은 언제, 어떻게 받나요?**
   A. 전월 1일~말일 단위로 정산 신청(`/api/seller/settlements/request`) → 어드민 승인 후 송금됩니다. 최소 출금액은 10,000원입니다. 신청 시 PIN 인증이 필요합니다.

4. **Q. 사업자등록 없이도 정산받을 수 있나요?**
   A. **현금 정산은 사업자 검증(`verified`/`exempt`)이 필요**합니다. 미검증 시 교환권/포인트(딜)로 수령할 수 있습니다(412 `BUSINESS_REGISTRATION_REQUIRED`).

5. **Q. 세금(원천징수)은 어떻게 처리되나요?**
   A. 사업자 미검증 셀러는 사업소득 **3.3%**(default) 또는 기타소득 **8.8%**(단발성)가 원천징수됩니다. **사업자 검증 시 원천징수 면제**(세금계산서 발행). 기타소득은 연 300만원 초과 시 종합소득 합산 대상입니다.

6. **Q. 라이브 방송으로 팔 수 있나요?**
   A. 네. YouTube Live가 즉시 지원되며 RTMP/OBS로 송출합니다. TikTok·치지직·SOOP은 준비 중(2026 Q3~Q4 예정)입니다. 방송 권한은 크리에이터/겸업 역할에 부여됩니다.

7. **Q. 크리에이터가 내 상품을 어떻게 팔아주나요?**
   A. 크리에이터가 링크샵에 상품을 "핀(추천)" 하거나 라이브에서 판매하면, 발생한 구매가 크리에이터에게 귀속되어 성과형 보상이 적립됩니다(핀 1.0%, 공구 양쪽 0.5%, 매장영입 1.5% 등).

8. **Q. 결제는 안전한가요?**
   A. 토스페이먼츠 V2 결제를 단일 SSOT로 처리하며, 결제 금액은 서버에서 재검증합니다. 환불은 자동(최대 5회 재시도) 처리됩니다.

9. **Q. 배송/주문 관리는 어떻게 하나요?**
   A. 셀러 주문 관리 페이지(`SellerOrdersPage`)에서 주문을 처리하고, tracker.delivery 연동으로 배송이 6시간 주기로 자동 추적됩니다.

10. **Q. 사업자번호는 어떻게 검증되나요?**
    A. 가입 시 형식(`XXX-XX-XXXXX`) 검증 후, 대표자명·개업일을 제공하면 국세청(NTS) 진위확인을 비동기로 거쳐 자동 승인(`approved`)됩니다.

11. **Q. 라이브에서 경매·타임딜 같은 특별 판매도 되나요?**
    A. 네. 라이브 방송 중 **경매**(실시간 입찰, 입찰 시 딜 보증금 hold), **타임딜**(한정 시간/수량 1~99% 할인, 시간·수량 소진 시 자동 종료), **타임딜 공동구매 모드**(목표 인원 달성 시 추가 할인), **선물하기**(시청자가 상품을 선물 발송)를 지원합니다. 모두 표준 결제·정산 흐름을 타며 기본 수수료 5% 가 동일 적용됩니다.

12. **Q. 펀딩(선주문)처럼 재고 만들기 전에 팔 수 있나요?**
    A. 펀딩 모델(`live_fundings`)이 있습니다. 목표 금액·리워드 등급·마감일을 정하고 진행률을 실시간으로 보여주며, 마감 후 발송합니다. 재고 리스크를 줄이고 수요를 먼저 검증할 수 있습니다. (셀러 직접 생성 UI는 일부 후속 `[확인 필요]`.)

13. **Q. 교환권/기프티콘도 온라인으로 팔 수 있나요?**
    A. 네. 유어딜 **자체 교환권**(`vouchers`)을 발행·판매하면 구매자가 QR/코드 + PIN으로 사용합니다(원자적 1회 사용, 부분 환불 지원). 선물하기와 결합해 바이럴됩니다. (KT alpha 등 **외부 기프티콘몰 연동은 현재 미구현** `[확인 필요]` — 자체 발행 체계.)

14. **Q. 브랜드가 우리 상품을 팔아줄 크리에이터를 직접 섭외할 수 있나요?**
    A. 캐스팅 마켓플레이스(`casting_requests`)로 광고주(브랜드)가 캠페인을 제시하면 어드민 검토 후 적합 셀러에게 제안이 전달되고, 셀러가 수락/거절합니다. 협업 라이브·광고로 확장할 수 있습니다. (제안비 중개 수수료 정책은 `[확인 필요]`.)

15. **Q. 혼자 운영이 버거운데 대신 관리해줄 수 있나요?**
    A. 에이전시에 매니징을 위임할 수 있습니다(소속/이전 흐름). 입점 경로에 따라 플랫폼 수수료 일부가 에이전시(30%)/영입 크리에이터(20%)에게 **내부 분배**되며, 이는 입점사가 추가로 내는 비용이 아닙니다. 에이전시 기능 전체는 별도 자료를 참고하세요.

16. **Q. 여기서 말하는 "공구"는 오프라인 동네딜 공구와 같은 건가요?**
    A. **아닙니다.** 이 소개서의 공구는 **온라인 즉시판매형 단일가 공구(B2C)** — 인원 무관 즉시 구매입니다. 오프라인 매장이 손님을 모으는 동네딜 공구는 별도 **오프라인 공구 소개서**가 다룹니다(§1-A 경계 참조).

`[디자인 지시]` FAQ는 아코디언/Q&A 2열 카드. 4·5번(정산·세금)은 강조 박스. 11~16번(라이브 수익화·펀딩·교환권·캐스팅·에이전시·공구 경계)은 "새 판매 모델" 그룹으로 묶어 별도 칩.

`[디자인 지시: 그라데이션]` **FAQ 카드 = subtle, 강조 항목만 액센트.**
> - Q&A 카드 면: 카드 subtle 그라데이션(`linear-gradient(180deg,#FFFFFF,#FFF1F3)` + border `#FFE4E9`). "Q" 뱃지 원(28~32px)은 시그니처 그라데이션 채움 + 흰 "Q".
> - **4·5번(정산·세금) 강조 박스**: 좌측 4px warm 그라데이션 액센트 바 + border 를 시그니처 그라데이션 stroke(1.5px)로 한 단계 강조. 본문 `text-gray-900`/`text-gray-700`(흰 텍스트 금지).
> - 답변 내 핵심 수치(5%, 3.3%/8.8%, 1.0%) 는 시그니처 그라데이션 텍스트로 포인트.

`[디자인 지시]` 마지막 슬라이드 전면 CTA + QR코드(가입 링크) + 입점 문의 연락처 자리.

`[디자인 지시: 그라데이션]` **CTA 슬라이드 = 시그니처 버튼 + glow(덱의 마무리 임팩트).**
> - 슬라이드 배경: 흰색 + 옅은 mesh, 또는 상단/배경 띠에 deep 그라데이션 풀폭(표지와 수미상관). 헤드라인 "지금 입점하세요"는 deep 면 위 흰 텍스트(스크림) 또는 흰 배경 위 시그니처 그라데이션 텍스트 중 택1.
> - **주 CTA 버튼 "지금 입점 신청하기"**: 시그니처 그라데이션 채움(`linear-gradient(135deg,#FF0033,#FF3D6E)`) + 흰 텍스트 + **glow**(`box-shadow:0 8px 28px rgba(255,0,51,.38)`). 가장 강한 시각 요소.
> - **보조 CTA**("셀러 가이드"/"데모"): 시그니처 그라데이션 **테두리(outline) 버튼**(채움 X, border 1.5px 그라데이션 + 흰 배경 + 그라데이션 텍스트).
> - **QR 코드**: 흑백 유지(스캔성), 프레임 테두리만 시그니처 그라데이션. 입점 문의 연락처는 흰 카드 위 `text-gray-700`.
> - 풀강도 그라데이션 면(deep 띠) + 주 CTA 버튼 1개 — 슬라이드당 한도 준수.
