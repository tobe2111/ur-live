# B2B 도매(유통스타트) 집중 90일 계획

> 객관적 평가: **B2B 도매(utongstart.com)** 가 유어딜에서 가장 방어 가능한 wedge(쐐기)다.
> 이 문서는 그 근거 + 90일 실행 계획 + KPI + 디프라이오리타이즈 대상을 정리한다.
> 모든 주장은 이미 출고된(또는 이번 작업으로 출고되는) 기능에 연결한다.
> 실데이터(현재 GMV/유저 수)는 `[확인 필요]` 로 표기 — 사용자 확인 후 채울 것.

---

## 1. 왜 소비자(백화점식 전개)보다 B2B 도매인가

| 축 | 소비자(교환권/공구/쇼핑) | B2B 도매(유통스타트) | 함의 |
|---|---|---|---|
| AOV(객단가) | 커피 한 잔~수만원 | 박스/MOQ 단위 사입(`min_order_qty`/`order_multiple`, BIZ-8) → 수십만~수백만원 | 같은 트래픽 대비 GMV 레버리지 ↑ |
| Retention(반복) | 소비자 변덕·일회성 | 유통사는 **재고가 떨어지면 다시 사입** → 구조적 반복발주(`/recent-items` 빠른 재주문) | LTV 예측 가능 |
| CAC | 소비자 광고 경쟁 치열 | 사업자 대상 영업/소개 — 모수 작지만 전환 깊음 | 채널 효율 ↑ |
| 방어성(moat) | 카탈로그/가격은 복제 쉬움 | **제조사 직거래 풀 + 등급가 엔진 + 여신/정산 신뢰 인프라** = 복제 어려움 | 진입장벽 형성 |

핵심: 소비자몰은 "또 하나의 쇼핑몰"이라 차별화가 어렵지만, 도매는 **공급(제조사) 확보 + 자본/신뢰 인프라**가 해자가 된다.
유어딜은 이미 가격 SSOT·서버 재계산·원가 마스킹(supply_price/supplier_id 비노출)·oversell CAS·Toss SSOT·등급가 엔진·바로빌 세금계산서·정산 클로백까지 갖춰 — **B2B 신뢰 인프라가 이미 동급 국내 도매몰 중 탄탄한 편**(roadmap 총평).

`[확인 필요]` 현재 도매 GMV / 활성 유통사 수 / 활성 제조사(공급사) 수 — 의사결정 기준선.

## 2. 공급측 확보 모션 (제조사 확보가 1순위)

도매몰의 성패는 **카탈로그(제조사 상품)의 폭과 가격 경쟁력**이다. 이미 출고된 셀프서브 입점이 이를 가능케 한다.

- **제조사 셀프서브 입점/카탈로그**: `supplier-dashboard.routes.ts` — 제조사가 직접 상품 등록(`supply_price`/권장소비자가/바코드/`pack_size`/`order_multiple`/`lowest_price_url`), 가격변경 승인 큐, 드랍쉽 송장.
- **위탁/무재고(드랍쉽)**: `creditSupplierOnOrder` (INC-8) 가 공급자 원본 재고를 차감 → 유통사는 재고 없이 판매, 제조사가 직배송. "사입 0원" 메시지의 실체.
- **공급 범위 선별**: `supply_visibility`(ALL/APPROVED_CHANNEL) + `product_distributor_access` — 제조사가 특정 유통채널에만 노출 가능(독점/선별 공급 → 제조사 유인).
- **검증/신뢰**: 어드민 최저가 검수(`lowest_price_checked`) + BIZ-5 네이버 쇼핑 `lprice` 참고값 병기(자동승인 X) → "검증된 도매가" 포지셔닝.

### 90일 공급 확보 우선순위
1. **앵커 제조사 10~20곳** 직접 영업(식품/뷰티/리빙 — 데모 카테고리와 일치). 셀프서브로 온보딩 마찰 최소.
2. 카테고리당 SKU **깊이 우선**(넓이보다) — 한 카테고리에서 "여기 다 있다" 경험.
3. OEM/ODM 매칭(`oem_requests`) + 견적(BIZ-3 `wholesale_quotes`) 로 대량/재판매 진입점 확보.

`[확인 필요]` 현재 입점 제조사/SKU 수, 카테고리 분포.

## 3. 여신(외상) 정책 — 자본 리스크 프레이밍 (보수적 시작)

여신은 "사입 0원" 약속을 완성하는 **가장 큰 사업 레버이자 가장 큰 자본 리스크**다. 인프라는 이미 출고(BIZ-2 v1).

- 출고된 것: `sellers.distributor_credit_limit` / `outstanding_balance` / `credit_frozen` + `wholesale_credit_ledger`(charge/repayment/adjust 감사) + `ON_CREDIT` 주문 경로(`wholesale.routes.ts /orders`) + 어드민 한도/동결/상환 관리(`distributor-admin.routes.ts`).
- 구조적 리스크: **여신 주문에서도 제조사 정산은 즉시 적립**(`creditSupplierOnWholesaleOrder`) → 플랫폼이 채권을 떠안는다. 유통사가 안 갚아도 제조사는 지급됨. 통제 수단은 **한도 + 동결**뿐.

### 보수적 정책 (90일)
1. **승인 + GMV 임계 유통사만** 여신 부여(가입 직후 pending 은 불가 — 코드 게이트 이미 존재).
2. **소액부터**: 1차 한도 = 직전 N개월 평균 발주액의 1배 이하. 점진 상향.
3. **등급 연동**: 좋은 등급(반복발주 입증) → 한도 상향. (auto-grade 와 연계 가능하나 v1 은 수동 한도 권장.)
4. **연체 → 자동 동결 cron**(follow-up, 현재 수동). 월마감 청구서/명세 자동화(현재 수동 상환 기록).
5. **비상 차단**: 사고 시 `credit_frozen=true` 또는 한도 0 — 코드 롤백 없이 즉시.

`[확인 필요]` 여신을 실제 켤지 여부 + 1차 총 여신 익스포저 상한(플랫폼이 감내 가능한 채권 총액).

## 4. 차별화 (복제하기 어려운 신뢰 자산)

1. **제조사 직거래 도매가**: 중간 유통 마진 제거 + 등급가 엔진(`distributor-pricing.ts`, A/B/C/D/OEM/SPECIAL) + 수량구간 할인(`product_qty_tiers`) + 상품별 마진 override. 원가/제조사 신원은 **절대 비노출**.
2. **무재고 위탁(드랍쉽)**: 사입 부담 없이 판매 → 제조사 직배송. 소상공인 진입장벽 제거.
3. **검증**: 최저가 검수 + (BIZ-5) 네이버 lprice 참고 → "바가지 아님" 신뢰.
4. **클레임/RMA 신뢰**(BIZ-1): 유통사 발의 클레임(`POST /api/wholesale/claims` → `wholesale_claims`) + **분쟁 중 제조사 정산 자동 HOLD**(`held_at`, `holdSettlements`) → 바이어 보호. 국내 도매몰에서 드문 안전장치.
5. **정산/세무 투명성**: 정산 클로백(지급 후 환불도 회수), 세금계산서/거래명세서(바로빌), 매입 역발행 기록(TAX-1), aging 리포트.

## 5. KPI 목표 (90일)

> 기준선이 `[확인 필요]` 이므로 **상대 목표 + 추적 지표**로 정의. 절대값은 기준선 확정 후 사용자와 합의.

| KPI | 정의 / 추적 위치 | 90일 목표(방향) |
|---|---|---|
| **도매 GMV** | `wholesale_orders` 순매출(subtotal − refunded, PAID/SHIPPED/PARTIAL/DONE/ON_CREDIT) — `auto-grade` GMV 산식과 동일 | 월 MoM 성장률 설정 `[확인 필요]` 기준선 |
| **활성 공급사(제조사)** | 최근 N일 내 주문 발생 supplier 수 (`wholesale_order_items.supplier_id` distinct) | 앵커 10~20 → 확대 |
| **활성 유통사** | 최근 N일 내 도매 주문한 `distributor_seller_id` distinct | `[확인 필요]` |
| **반복발주율** | 2회 이상 발주 유통사 비율 (`/recent-items` 가 지원하는 재주문) | 핵심 retention 지표 — 우상향 |
| **미수금(여신) 한도/익스포저** | `Σ outstanding_balance` vs 총 여신 한도 + 연체율 | **상한 내 유지**(리스크 KPI) |
| **클레임 해결 SLA** | `wholesale_claims` open→resolved 시간 | 짧게 유지(신뢰) |
| **GMV per 활성 유통사** | GMV / 활성 유통사 | AOV·deepening 신호 |

추적 인프라: 어드민 `/admin/distributor/orders`(모니터), `tax-summary`/`tax/aging`(정산/미수), `auto-grade/settings`(GMV 집계 재사용). 공급사 분석(BIZ-6 매출 시계열/객단가/재고회전율)은 follow-up.

## 6. 디프라이오리타이즈 (지금 하지 말 것)

- **백화점식 소비자 전개**: 소비자몰 카테고리 무한 확장 — 차별화 약하고 CAC 비쌈. 소비자 surface 는 **현 수준 유지/최적화**(이미 로딩 최적화 잠금 영역)면 충분.
- **다통화/다국가/세율 추상화**(SCALE-1): roadmap 도 "현 단계 과투자, 보류" 명시. 국내 B2B 집중.
- **최저가 자동승인**: 동명이품 오탐 위험 — 참고값 병기까지만(BIZ-5), 게이트 강제는 product intent 확정 후.
- **자동 강등(demote)**: auto-grade 는 promote-only 유지 — 매출 일시 하락으로 가격 올리면 유통사 이탈. 강등은 수동만.
- **공급사 API 인입/대규모 통합**: 앵커 제조사 셀프서브로 충분. 트래픽/SKU 가 증명되기 전 과투자 금지.
