# 유어딜 라이브 핵심 흐름 점검 (대표님용)

> **목적**: 자동 검사(audit-gate 31개)로는 못 잡는 것 — **실제 결제·외부 응답·기기별 화면** — 을
> 대표님이 라이브(live.ur-team.com)에서 직접 한 번씩 눌러보며 "진짜 되는지" 확인하는 체크리스트.
> 개발 환경에선 실결제가 안 되므로 **이 점검은 대표님만 할 수 있는 마지막 검증**입니다.
>
> 사용법: 위에서부터 순서대로. 각 항목 = **[할 일]** → **[정상이면]** → (왜 중요한지).
> 하나라도 이상하면 그 항목 번호와 증상을 캡처해서 Claude 에게 주세요.
>

## 🔑 1단계 — 로그인 (모든 것의 입구)

로그인이 깨지면 나머지 전부 막히므로 **기기 3종을 꼭 따로** 확인. (특히 아이폰)

- [ ] **1-1. PC 크롬 카카오 로그인**
  - 할 일: PC 크롬에서 live.ur-team.com → 카카오로 로그인
  - 정상이면: 로그인 후 홈으로 들어오고, 새로고침해도 로그인 유지
  - 왜: 가장 기본 경로

- [ ] **1-2. 📱 아이폰 사파리 카카오 로그인** ⭐ *가장 잘 깨지는 곳*
  - 할 일: 아이폰 **사파리**에서 접속 → 카카오 로그인
  - 정상이면: 로그인 후 홈 유지 (다시 로그아웃되거나 무한 새로고침 X)
  - 왜: 아이폰은 로그인 쿠키를 까다롭게 처리 → 여기만 조용히 깨지는 사고가 과거에 있었음

- [ ] **1-3. 📱 카카오톡 인앱 브라우저 로그인**
  - 할 일: 카카오톡으로 자기한테 링크 보내고 → 카톡 안에서 열어 로그인
  - 정상이면: 로그인 성공, 화면 갇힘 없음
  - 왜: 카톡 인앱 브라우저는 일반 브라우저와 다르게 동작

- [ ] **1-4. 계정 전환**
  - 할 일: 로그아웃 → **다른 카카오 계정**으로 로그인
  - 정상이면: 이전 계정의 이름·링크샵·포인트가 안 섞이고 새 계정으로 깨끗이 바뀜
  - 왜: 공용/공유 기기에서 계정이 섞이면 큰 사고

## 💳 2단계 — 결제 (실제 돈)

각 결제는 **실제 카드/실제 돈**으로 1회씩. 소액으로. (테스트 후 환불은 3단계에서)

- [ ] **2-1. 딜 충전**
  - 할 일: 마이 → 딜 충전 → 1만원 충전
  - 정상이면: 결제 완료 화면 → 잔액이 정확히 **+10,000딜**
  - 왜: 가장 단순한 결제. 금액 1원 단위 정확성 확인

- [ ] **2-2. 이용권/상품 구매 (카드결제)**
  - 할 일: 이용권 하나 골라 카드로 결제
  - 정상이면: 결제완료 화면(영수증/카드정보 표시) → **주문내역**과 **보관함**에 바로 뜸 → 알림함에 "주문완료"
  - 왜: 가장 흔한 거래. 결제 후 즉시 수령물이 보여야 함

- [ ] **2-3. 교환권(기프티콘) 구매**
  - 할 일: 교환권 상품 구매
  - 정상이면: 결제 후 **문자(기프티콘)가 실제로 도착**
  - 왜: 외부(KT) 발송이 진짜 가는지는 라이브에서만 확인 가능

- [ ] **2-4. 딜 + 카드 혼합결제**
  - 할 일: 딜 일부 + 카드로 나눠서 결제
  - 정상이면: 딜이 쓴 만큼 정확히 차감 + 카드는 나머지만 청구
  - 왜: 딜 차감이 어긋나면 플랫폼 손해/고객 불만

- [ ] **2-5. 숙소 예약 결제**
  - 할 일: 숙소 날짜 잡고 결제
  - 정상이면: 예약 확정. (같은 방을 동시에 둘이 잡으면 한 명만 성공해야 함)
  - 왜: 날짜 재고가 꼬이면 오버부킹 → 현장 사고

- [ ] **2-6. 동네딜(공구) 참여 결제**
  - 할 일: 동네딜 상품 하나 참여 결제
  - 정상이면: 참여 확정 + 확정 알림
  - 왜: 공구 흐름 결제 확인

## ↩️ 3단계 — 환불 (돈 되돌리기)

- [ ] **3-1. 주문 전액 환불**
  - 할 일: 위에서 산 주문 하나를 환불 요청 → (어드민) 환불 처리
  - 정상이면: **카드 환불됨** + 썼던 **딜 복원** + 쿠폰 복원 + (해당되면) 에이전시/공급자 적립도 자동 취소
  - 왜: 환불 시 되돌릴 게 많음 — 하나라도 빠지면 돈이 샘

- [ ] **3-2. 부분 반품**
  - 할 일: 여러 개 산 주문에서 일부만 반품
  - 정상이면: 반품한 만큼만 비례 환불 + 딜/쿠폰 비례 복원
  - 왜: 부분 계산이 가장 틀리기 쉬움

## 🎟️ 4단계 — 수령·사용

- [ ] **4-1. 이용권 매장 사용**
  - 할 일: 보관함 이용권 → 매장에서 QR/PIN 으로 사용 처리
  - 정상이면: "사용완료" 로 바뀌고 재사용 불가
  - 왜: 중복 사용되면 매장 손해

- [ ] **4-2. 디지털 상품 열람**
  - 할 일: 디지털 상품 구매 후 보관함에서 열기
  - 정상이면: 바로 접근 가능
  - 왜: 결제했는데 못 받으면 클레임

## 🔗 5단계 — 링크샵 (내 쇼핑몰)

- [ ] **5-1. 내 링크샵**
  - 할 일: `live.ur-team.com/u/내핸들` 접속
  - 정상이면: 내 상품·이용권·핀이 보이고, 첫 화면이 깜빡이거나 옛 화면 잔상 없음
  - 왜: 사업자 유저의 본인 쇼핑몰 = 얼굴

- [ ] **5-2. 깨끗한 공유 링크**
  - 할 일: `/u/내핸들?embed=1` 접속
  - 정상이면: 상·하단 네비 없이 매장만 깔끔하게
  - 왜: 외부 공유용 standalone 링크

## 📊 6단계 — 정산 확인 (어드민 관점)

- [ ] **6-1. 결제 → 정산 반영**
  - 할 일: 어드민 → 정산 센터에서 위 결제 주문들이 뜨는지
  - 정상이면: 주문이 목록에 있고 수수료/셀러몫 금액이 맞음 (₩NaN 같은 거 없음)

- [ ] **6-2. 에이전시 매장영입 커미션** (해당 시)
  - 할 일: 에이전시가 영입한 매장 주문이면 → 에이전시 커미션 적립됐는지
  - 정상이면: 적립 기록 생성

- [ ] **6-3. (선택) 수수료 규칙 비교**
  - 할 일: 어드민 → 정산/재무 → "수수료 규칙 비교" → "과거 주문으로 즉시 백필"
  - 정상이면: 새 규칙 vs 현행 비교표가 뜸 (실제 돈 무영향)
