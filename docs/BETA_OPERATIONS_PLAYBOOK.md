## Week 2: 첫 외부 셀러 영입 (2-3명)

### 영입 타겟 (우선순위)
1. **본인 단골 가게** (관계 기반) — 음식점 / 헤어샵 / PT 매장
2. **인플루언서 친분** — 팔로워 5천 ~ 5만 마이크로
3. **에이전시** — 본인 인맥 있는 동네 마케팅 회사 (월 정액 50만 + GMV 30% 분배)

### 영입 스크립트 (음식점 사장님)

> "안녕하세요 사장님. 사장님 가게 단골인 [본인 이름]입니다.
>
> 제가 동네 공동구매 플랫폼을 만들었는데, **사장님은 가입도 메뉴 등록도 안 하셔도 돼요**.
> 메뉴판 사진 한 장만 주시면 30초만에 등록해드리고, **사장님은 카카오톡으로 받은 링크 클릭만 하시면** 사용된 voucher 확인 + 정산 받을 수 있어요.
>
> 5명 모이면 10% 할인, 10명 모이면 20% 할인 식으로 단계별 할인이라 사장님 손해 없이 손님 끌어오는 캠페인이에요.
>
> 첫 달은 수수료 3% 만 받을게요 (원래 5%). 일주일 안에 1명도 안 모이면 캠페인 그냥 자동 취소되고 손님 돈 자동 환불됩니다. **사장님 리스크 0** 이에요.
>
> 5분 만에 등록 도와드릴 수 있는데, 메뉴판 사진 한 장 보내주실 수 있을까요?"

### 영입 후 시드 작업 (셀러당 30분)
1. 사장님께 메뉴판 사진 받기
2. `/seller/meal-voucher/new` 본인이 대신 등록 (OCR 활용)
3. 마감 7일 / 목표 인원 5명 / 가격 30-50% 할인 (눈에 띄는 deal)
4. Magic Link 알림톡 자동 발송 → 사장님 확인
5. `/group-buy/<id>` URL share 도구로 본인 인스타/카톡 채널에 1차 노출

- 미달성 시 자동 환불 → 셀러 알림톡 → "다음엔 목표 인원 더 낮추거나 할인율 더 크게" 코칭

## 🚨 위기 대응 시나리오

### 1. 첫 공구 미달성 → 셀러 이탈
**즉시**: 사장님께 직접 연락 (전화). 1주일 더 운영 무료 + 다음 공구는 본인이 직접 share 책임.
**예방**: 첫 공구 등록 시 목표 인원 5명으로 낮게 (10명 X), share 책임 본인이 짊어짐.

### 2. Voucher 사용 거부 / 매장 폐업 → 분쟁
**즉시**: `/admin/disputes` 큐 확인 → AI 자동 분류 결과 + 본인 검토 → 24시간 내 환불.
**예방**: 셀러 가입 시 사업자등록증 권장 (선택), 신용도 높은 가게 우선.

### 3. PG 수수료로 마진 0
**즉시**: 충전 UI 권장 변경 (5만/10만 강조 + 보너스 딜).
**예방**: 셀러 차등 수수료 (1억+ 3%, 1천만+ 4%) — 대형 셀러 retain.

### 4. AI binding 미설정 → OCR 실패
**즉시**: OCR fallback 메시지로 "수동 입력" 안내.
**해결**: Cloudflare Dashboard 에서 AI binding 추가 (1분, 무료).

### 5. CI 배포 실패
**즉시**: `git push --force` 금지. GitHub Actions 결과 확인 → 에러 메시지 보고 fix → 새 commit.
**예방**: Pre-commit hook 자동 검증 (이미 설정됨).
