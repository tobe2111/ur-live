# A/B 테스트 양식 + 체크리스트 (유어딜/유통스타트)

> 기능 변경(노출/문구/플로우)을 "감"이 아니라 **데이터로 결정**하기 위한 재사용 양식.
> 새 실험마다 아래 양식을 복사해 `docs/experiments/<날짜>-<이름>.md` 로 채운다.

## ⚙️ 현재 인프라 현실 (먼저 읽기)

이 코드베이스에는 **실험(variant 버킷팅 + 측정) 인프라가 아직 없다.** 있는 것은:
- `src/shared/config/feature-flags.ts` — **글로벌 on/off 부울 플래그**(SSOT). variant 분배 X.
- `getFeatureFlag(flag, userId)` — user 기반 롤아웃 훅(있으나 측정 미연동).

→ A/B 를 제대로 돌리려면 아래 중 하나 필요(체크리스트 0번 참조):
1. **(권장·경량)** 결정적 해시 버킷팅 — `hash(userId + 실험키) % 100 < N` 으로 variant 배정(서버/클라 동일 함수) + 기존 이벤트 로깅으로 지표 수집. 새 인프라 거의 0.
2. (중량) 전용 `experiments` 테이블 + 노출/전환 로깅 + 대시보드. ROI 높을 때만.

## 📋 A/B 테스트 양식 (복사해서 사용)

```
# 실험: <한 줄 이름>          예) 도매 대량주문 패널 노출 승격
- 작성자 / 날짜:
- 상태: 설계 / 진행중 / 분석중 / 종료(채택·기각)

## 1. 가설 (Hypothesis)
"<무엇을> 바꾸면 <누구의> <지표>가 <방향/크기> 변할 것이다. 왜냐하면 <근거>."
예) 카탈로그 상단에 '대량주문(엑셀)' 진입 버튼을 상시 노출하면, 판매사의
   대량주문 시도율이 ↑할 것이다. 현재는 토글(bulkOpen) 안에 숨어 발견율이 낮으므로.

## 2. 지표 (Metrics) — 사전 확정 (사후 변경 금지)
- 🎯 1차 지표(Primary, 1개만): 예) bulk-preview 호출한 판매사 비율 (주간)
- 🛡️ 가드레일(Guardrail, 회귀 감시): 예) 카탈로그→일반 장바구니 담기율, 페이지 LCP, 객단가
- 📈 보조 지표(2차): 예) 업로드→카트 전환율, 평균 주문 라인 수

## 3. 변형 (Variants)
- A (Control, 현행): <현재 동작 그대로>
- B (Treatment): <바뀐 동작 1개만 — 한 실험 = 한 변경>
  (※ B 에 변경 2개 섞지 말 것 — 원인 분리 불가)

## 4. 대상·배정 (Audience & Assignment)
- 대상: <예: 로그인 판매사 전원 / 신규 가입 7일 이내 / 특정 등급>
- 제외: <예: 어드민·내부계정·viewer 직원>
- 트래픽 분할: A 50% / B 50% (또는 90/10 안전 시작 → 점진 확대)
- 배정 키: userId(또는 seller_id) 해시 — **세션 아닌 사용자 단위**(같은 유저는 항상 같은 변형)

## 5. 표본·기간 (Sample size & Duration)
- 현재 기준값(baseline): <1차 지표 현재 값 %>
- 최소 탐지 효과(MDE): <예: 상대 +15%>
- 필요 표본/기간: <대략 — 표본 계산기 또는 "주당 N명 × 최소 2주, 풀 주 단위">
- ⚠️ 최소 1~2주 + **온전한 주 단위**(요일/주말 효과 제거). 유의미 나와도 조기 종료 금지.

## 6. 성공 기준 & 의사결정 (Decision rule) — 사전 확정
- 채택(ship B): 1차 지표 +X% 이상 & 가드레일 무회귀 & p<0.05(또는 베이지안 P(B>A)>95%)
- 기각(keep A): 1차 지표 개선 없음 또는 가드레일 회귀
- 보류(재실험): 표본 부족/혼재 → 조건 바꿔 재설계

## 7. 구현 메모
- 플래그/버킷 키:  예) exp_bulk_panel_promote
- 변경 파일:  예) src/pages/WholesaleCatalogPage.tsx (진입 버튼), 이벤트 로깅 지점
- 롤백: 버킷 함수가 100% A 반환하도록 1줄 — 즉시 전원 현행

## 8. 결과 (종료 후 작성)
- 기간 / 표본(A,B): 
- 1차 지표 A vs B (+신뢰구간/유의성): 
- 가드레일: 
- 결정: 채택 / 기각 / 보류 — 사유:
- 후속:
```

## ✅ 실행 체크리스트

### 0. 사전 인프라 (최초 1회)
- [ ] variant 배정 방식 결정 — 결정적 해시 버킷팅(권장) vs 전용 테이블
- [ ] 배정 함수 작성: 서버·클라 **동일 결과**(같은 userId+실험키 → 같은 변형). 순수함수 + 단위테스트
- [ ] 노출(exposure)·전환(conversion) 이벤트 로깅 지점 확보 (어떤 변형을 봤는지 + 목표 행동)
- [ ] 어드민/내부계정/viewer **제외** 규칙

### 1. 설계 (런치 전)
- [ ] 양식 1~7 전부 채움 — **지표·성공기준을 런치 전에 확정**(사후 p-hacking 방지)
- [ ] 1차 지표 1개로 압축 (여러 개면 위양성↑)
- [ ] 가드레일 지표 명시 (성능/매출/핵심 전환 회귀 감시)
- [ ] 한 실험 = 한 변경 (B 에 변경 1개만)
- [ ] 표본/기간 산정 — 최소 주 단위 × 1~2주
- [ ] 90/10 안전 시작 여부 결정(고위험 변경이면)

### 2. 구현 (코드)
- [ ] 변형 분기 + 배정 함수 배선 (CLAUDE.md: dual-mode 분기 유지, 한 commit = 한 원인)
- [ ] 노출 이벤트 발화(변형 본 시점) + 전환 이벤트 발화
- [ ] 롤백 1줄 경로 확보 (버킷 강제 A)
- [ ] 테마/i18n/잠금파일 규칙 준수 (로딩·결제 잠금 영역이면 AskUserQuestion)
- [ ] tsc 0 · build 0 · 관련 단위테스트

### 3. 런치 & 모니터링
- [ ] 배정 균형 확인(A/B 트래픽 ~50:50, SRM=표본비율불일치 없나)
- [ ] 첫 24h 가드레일·에러율·성능 점검(즉시 회귀 시 롤백)
- [ ] 양 변형이 실제로 노출되는지 1건씩 수동 확인(QA)

### 4. 분석 (종료 시)
- [ ] 사전 정한 기간/표본 충족했나 (조기종료 금지)
- [ ] 1차 지표 차이 + 신뢰구간/유의성
- [ ] 가드레일 무회귀 확인
- [ ] 세그먼트 쪼개기는 **탐색용**(확정 결론 X — 위양성)
- [ ] SRM·봇·중복계정 등 오염 점검

### 5. 결정 & 마무리
- [ ] 사전 의사결정 규칙대로 채택/기각/보류
- [ ] 채택 시: 플래그 100% 전환 → 한두 주 후 분기/죽은 코드 정리
- [ ] 기각 시: 변형 제거(코드 청소) + 학습 기록
- [ ] 양식 8번(결과) 작성 + `docs/experiments/` 보관 → 지식 누적

## 🧪 worked example (현 컨텍스트)

| 항목 | 값 |
|---|---|
| 실험 | 도매 대량주문(엑셀) 패널 진입 승격 |
| 가설 | 카탈로그 상단 상시 진입 버튼 → 판매사 대량주문 시도율 ↑ (현재 `bulkOpen` 토글에 숨음) |
| 1차 지표 | 주간 활성 판매사 중 `/api/wholesale/orders/bulk-preview` 1회+ 호출 비율 |
| 가드레일 | 일반 장바구니 담기율, 카탈로그 LCP, 주문 객단가 |
| 변형 | A=현행(토글) / B=상단 상시 버튼 + 1줄 설명 |
| 배정 | seller_id 해시 50/50, 어드민·viewer 제외 |
| 기간 | 2주(온전한 주) |
| 성공 | bulk-preview 호출 비율 상대 +15% & 가드레일 무회귀 |
| 롤백 | 버킷 함수 강제 A (1줄) |

> 이 실험을 실제로 돌리려면 체크리스트 0번(배정 함수 + 노출/전환 로깅)부터.
> 원하면 경량 결정적 버킷팅 유틸(`src/shared/ab.ts`, 순수함수+테스트)을 먼저 깔아줄 수 있음.
