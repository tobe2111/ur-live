# 유어애즈 × 유어딜 — 판매채널 번들 (크로스서비스 설계)

> 2026-06-27 대표 "모두 다" 중 하나. **단, CLAUDE.md 서비스 분리 룰**(도매몰/유어딜/유어애즈 경계)상
> 이건 *세 서비스를 잇는* 변경이라 정산·이행·소유권 결정이 선행되어야 함 → **코드 전 설계+결정**.
> (나머지 둘 — 소싱 리포트·예산 페이싱 — 은 유어애즈 내부라 바로 구현 완료.)

## 컨셉
유어애즈 고객사(광고주)의 상품을 **유어딜 소비자 채널(공구/교환권)에 노출** → "광고"만이 아니라
*실판매처*까지 제공. 보라웨어엔 없는 우리만의 수직통합(소싱→광고→판매).

## 왜 바로 코드 못 까나 (분리 위반 위험)
유어애즈 고객사 = `sellers`(마케팅 owner_type) 일 수도, 외부 광고주일 수도 있음. 유어딜 공구/교환권은
**소비자 결제·재고·정산·CS** 가 붙은 실거래. 광고주 상품을 유어딜에 띄우려면:
- **누가 파나**: 광고주가 유어딜 사업자 유저(판매승인)여야 함? 아니면 우리가 위탁판매?
- **이행/CS**: 배송·교환권 발급·환불 책임 주체?
- **정산**: 판매대금 흐름(광고주 ↔ 플랫폼 수수료 ↔ 유어딜 정산엔진)?
- **소유권 플래그**: `products.is_supply_product`/`seller_id` 처럼 "유어애즈 유입 상품" 구분 플래그 필요?

→ 이 결정 없이 코드를 깔면 유어딜 소비자 카탈로그/정산에 누수(과거 도매 상품 누수 사고와 동일 클래스).

## 결정 필요 (대표)
- [ ] **A. 광고주 자격**: 유어딜 사업자 유저(판매승인)만 번들 가능 vs 우리가 위탁판매 대행
- [ ] **B. 채널**: 공구(공동구매) / 교환권 / 일반쇼핑 중 무엇으로 노출
- [ ] **C. 이행·CS·정산 주체**: 광고주 직접 vs 플랫폼 대행(+수수료율)
- [ ] **D. 노출 심사**: 자동 vs 어드민 승인 게이트

## 구현안 (결정 후 — 분리 준수)
1. 광고주 상품을 유어딜에 **복제 등록**(도매 재판매 복제본 패턴 재활용 — `supply_source_id` 처럼 `urads_source_id` 구분 플래그) → 소비자 쿼리는 기존 격리 룰로 자동 포함/제외 제어.
2. 정산: 유어딜 기존 정산엔진(Toss/딜/sellers.commission) 재활용 + 유어애즈 유입분 수수료 라인.
3. 어드민 승인 게이트(D) → 품질/분리 보호.
4. 유어애즈 대시보드에 "유어딜에 판매하기" 진입 + 상태.

## 현황
- 🟡 **설계만** — 결정 A~D 대기. 결정되면 분리 가드(`check-consumer-product-supply-isolation` 류) 확장 + 복제본 플래그로 안전 구현.
