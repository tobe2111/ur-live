# 블로그 UI 개편 — 토스 테크 스타일 (2026-07-01 대표 시안)

대표가 [toss.tech](https://toss.tech) 블로그 스크린샷 2장을 공유하며 "블로그 UI 를 이렇게" 요청.

## 시안 요약

### 1) 상단 히어로 (featured 캐러셀)
- 상단 네비: 로고 + 카테고리(Engineering / Design / Product) + `구독하기`/`채용 바로가기` 버튼 + 검색 아이콘.
- 히어로 카드: **좌측 큰 2줄 제목** + 회색 부제(요약), **우측 큰 라운드 이미지**(일러스트).
  - 예: "토스팀이 AI 파도를 마주하는 방법: AI Surf Day" / "파도를 멈출 수는 없지만, 서핑하는 방법은 배울 수 있다."
- 좌하단에 **원형 화살표 버튼 2개(‹ ›)** — featured 글 캐러셀 이동.

### 2) "전체 아티클" 리스트
- 섹션 제목 **"전체 아티클"**(굵게).
- **세로 리스트 행**(그리드 아님). 각 행:
  - 태그 칩: **카테고리(파란 "Engineering")** + **팀/작성자(회색 칩)**
  - **굵은 제목**(진한 잉크)
  - 회색 **요약** 1~2줄
  - 일부 행은 **우측 정사각 라운드 썸네일**(있는 글만)
- 하단 **페이지네이션**: `‹ 1 2 3 4 5 ›`

## 현재 → 시안 차이

| 요소 | 현재(유어딜) | 시안(토스) |
|---|---|---|
| 상단 | 뒤로가기 + 타이틀 + "판매 시작하기" | **히어로 featured 캐러셀**(큰 제목+이미지+‹›) |
| 목록 | 균일 카드 **그리드**(2~4열) | **세로 리스트 행** + 우측 썸네일 |
| 태그 | 상단 필터 칩 행 | 행 내 카테고리+팀 칩(상단 필터는 유지/축소) |
| 하단 | 없음(전체 노출) | **페이지네이션 1~5** |

## 유어딜 데이터 매핑
- 카테고리 칩 = `tags[0]`(파란 스타일) · 두 번째 칩 = `author`(회색).
- 히어로 = 최신 글 상위 N개(≈5) 캐러셀. 이미지 = `thumbnail_url` 또는 그라디언트+이모지 커버(기존 `blogCover`).
- 페이지네이션 = 이미 전체 글을 클라에 받으므로(limit=100) **클라이언트 페이지네이션**(페이지당 ≈8).

## 구현 체크리스트
- [ ] 히어로 캐러셀(좌 제목/요약 + 우 이미지 + ‹› 화살표, 자동/수동 이동)
- [ ] "전체 아티클" 세로 리스트 행(카테고리+팀 칩 · 제목 · 요약 · 우측 썸네일)
- [ ] 클라이언트 페이지네이션(‹ 1..N ›)
- [ ] 태그 필터 유지(축소 스타일) — 선택 시 히어로 숨김 + 리스트만 필터
- [ ] 다크모드 `dark:` 전 토큰 · `min-h-[100dvh]` · 스켈레톤 로딩 유지
- [ ] 보존(회귀 금지): `__SSR_INITIAL_BLOG__` 0-RTT 소비 · `stripBold` · SEO · 태그 유니버스 클라 파생

## 보존 불변식(중요)
- SSR 시드(`readBlogListSeed`) + `refetchOnMount:'always'` — 첫 페인트 0-RTT.
- 제목/요약 `stripBold` — `**` 노출 방지.
- 다크/라이트 양테마(테마 가드 strict).
