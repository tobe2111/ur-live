# 유통스타트 도매몰 — 쇼핑몰형 UI 기획 (시스템 사양 기반)

> 기반: [`wholesale-b2b-mindmap.md`](./wholesale-b2b-mindmap.md) (PDF 스펙) + 기존 구현([`wholesale-b2b-spec.md`](./wholesale-b2b-spec.md)).
> 목표: 현재 기본 그리드(`/wholesale`)를 **B2B 도매 쇼핑몰**로 재설계. 유통회원(바이어)이 "둘러보고 → 담고 → 대량 주문"하는 흐름 최적화.
> 작성: 2026-06-04. 상태: **기획안 (사용자 방향 확정 대기)**.

---

## 1. 설계 원칙 (B2B ≠ B2C)
| 원칙 | 이유 |
|---|---|
| **가격이 주인공** | 바이어는 "내 등급 공급가 + 마진"으로 의사결정. 가격·재고·MOQ를 카드에서 즉시 |
| **정보 밀집 + 신뢰** | 소비자몰의 감성보다 효율. 라이트 테마, 정돈된 그리드, 군더더기 X |
| **제조사 신원 비공개** | 스펙 룰 — 공급가만 노출, supplier_id/원가 절대 X (서버에서 이미 차단) |
| **등급 맥락 상시 노출** | 내 등급(C/B/A)·마진%를 헤더에 고정 → "이 가격이 내 가격" 인지 |
| **대량 거래 우선** | 장바구니=여러 상품 일괄 선결제 주문. 엑셀 주문·OEM 진입을 일급 동선으로 |

**비주얼 톤(권장)**: 도매꾹/도매매의 정보밀집 + 쿠팡/스마트스토어의 모던 깔끔함 하이브리드. 라이트 테마, 브랜드 액센트 `#FF0033`(인트로와 통일), 카드 컴팩트.

## 2. 정보 구조 (IA)

```
utongstart.com (= /wholesale)
├─ /wholesale/intro      공개 랜딩(소개·가입)        ← 이미 구현
├─ /wholesale            🆕 쇼핑몰 홈 (재설계 핵심)
├─ /wholesale/category/:key  카테고리별 목록
├─ /wholesale/search?q=  검색 결과
├─ /wholesale/product/:id    상품 상세            ← 개선
├─ /wholesale/cart       🆕 장바구니 (일괄 주문)
├─ /wholesale/checkout   주문/결제(선결제)        ← 이미 구현
├─ /wholesale/orders     주문내역                ← 이미 구현
├─ /wholesale/statement  거래명세서/세금계산서       ← 이미 구현
└─ /wholesale/oem        OEM/ODM 신청            ← 이미 구현
```

## 3. 쇼핑몰 홈 (`/wholesale`) — 섹션 구성 (스펙 매핑)

| 섹션 | 내용 | 스펙 근거 |
|---|---|---|
| **① 헤더(고정)** | 로고 · **검색바** · 내 등급 배지(C/B/A + 마진%) · 장바구니(수량) · 주문/거래명세서/OEM 메뉴 | 등급별 공급가 노출 / OEM 지원 |
| **② 카테고리 네비** | 가로 스크롤 칩(전체/식품/뷰티/리빙/패션/디지털…) | 제품 카테고리 |
| **③ 히어로 배너** | 특가·공지 캐러셀 (관리자 설정 or 특가상품 자동) | 특별가격(임박/덤핑) |
| **④ 🔥 특가·임박 (SPECIAL)** | 기간한정 SPECIAL 등급 상품 가로 캐러셀 + 남은시간 | 특별가격 등급 = 임박/덤핑, 선정 유통사만 |
| **⑤ 📢 추천 상품 제안** | 관리자가 나에게 제안한 상품(wholesale_proposals) | 관리자→유통회원 제안 |
| **⑥ ⭐ 베스트셀러** | sold_count 상위 | — |
| **⑦ 🆕 신상품** | 최근 등록 | — |
| **⑧ 카테고리별 추천 그리드** | 무한스크롤 메인 카탈로그 | 제품확인(가시성 가드 적용) |
| **⑨ 모바일 하단바** | 홈 / 카테고리 / 장바구니 / 주문 / 마이 | — |

> ④⑤는 데이터가 **이미 백엔드에 존재**(SPECIAL 등급 가격, proposals). ⑥⑦은 카탈로그 정렬로 파생.

## 4. 상품 카드 (도매 전용)

```
┌──────────────────┐
│  [상품 이미지]      │ ← dominant_color placeholder + cfImage(WebP) + fade
│  🔥특가  -28%       │ ← SPECIAL 배지 / 할인율(권장소비자가 대비)
├──────────────────┤
│ 상품명 (2줄)        │
│ ₩12,000  내 등급가  │ ← 공급가 강하게(빨강/굵게). "C등급" 작게
│ 권장가 ₩19,900      │ ← strike (마진 여력 암시)
│ 재고 1,240 · 식품    │
│ [− 10 +] [장바구니]  │ ← 수량 스텝퍼 + 담기 (호버/탭 시)
└──────────────────┘
```
- **제조사명 절대 X** (스펙). 카테고리·재고·내 등급가만.
- React.memo + dominant_color + cfImage (동네딜/쇼핑과 동일 최적화 패턴).

## 5. 상품 상세 (`/wholesale/product/:id`) 개선
- 좌: 이미지 / 우: 상품명 · **내 등급 공급가(대)** · 권장소비자가 · 재고 · 수량 스텝퍼 · [장바구니 담기] [바로 주문]
- 탭/섹션: 상세설명 · 배송 안내 · **정산 안내**(브랜드=익일/일반=7일 — 스펙) · 반품/교환 정책
- 같은 카테고리 추천

## 6. 장바구니 (`/wholesale/cart`) 🆕
- 여러 상품 담아 **한 번에 선결제 주문**(기존 `POST /api/wholesale/orders` items[] 그대로 활용)
- 상품별 수량 조정 · 합계(공급가 합) · 배송지 · [엑셀로 대량 주문 업로드] 진입(기존 order-template)

---

## 7. 신규로 필요한 백엔드(최소, 전부 additive)
- `GET /api/wholesale/home` — 홈 한 번에: { special[], proposals[], best[], new[], categories[] } (등급가 서버 계산, 가시성 가드). **1 콜로 홈 구성 → SSR inject 가능(0-RTT)**.
- (선택) `GET /api/wholesale/search?q=` — 이미 catalog 의 search 파라미터로 대체 가능.
- 나머지(catalog/orders/oem/statement)는 기존 그대로.

## 8. 성능 (다른 페이지 교훈 반영 — 서버비용 0 지향)
- `/wholesale` 홈에 **SSR 슬롯 신규**(`__SSR_INITIAL_WHOLESALE__`) + cron prewarm + consume-once → 0-RTT (동네딜과 동일 패턴, 이미 검증).
- 카드 dominant_color + cfImage + React.memo.
- 지도 SDK 없음(도매몰엔 불필요).

---

## 9. 단계별 구현 (제안)
1. **Phase 1 — 홈 뼈대**: 헤더(검색+등급배지+장바구니) + 카테고리 칩 + 메인 그리드(카드 개선). `GET /api/wholesale/home` + SSR.
2. **Phase 2 — 큐레이션 섹션**: 특가·임박 / 추천제안 / 베스트 / 신상품.
3. **Phase 3 — 장바구니**: 담기 → /cart → 일괄 주문(+엑셀 업로드 동선).
4. **Phase 4 — 상세 개선** + 검색 결과 페이지.

각 Phase tsc/build/test 검증 + 단일 commit.
