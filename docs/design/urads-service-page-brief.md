# 유어애즈 (UR Ads) — 종합 마케팅 솔루션 서비스 페이지 · 디자인 외주 브리프

> 외주 디자이너/에이전시에게 그대로 전달용. 작성 2026-06-26.

## 1. 한 줄 정의
**유어애즈(UR Ads)** = 네이버 광고·커머스를 한곳에서 자동화하는 **B2B 종합 마케팅 솔루션 SaaS**.
(유어팀 그룹의 3번째 서비스 — 유어딜=소비자 / 유통스타트=도매 / **유어애즈=마케팅**.)

## 2. 타겟 & 톤
- **타겟**: 스마트스토어·쇼핑몰을 운영하는 사업자(셀러)와, 그들을 관리하는 대행사/에이전시.
- **톤**: 신뢰감 있는 B2B SaaS. 데이터·자동화·성과 중심. 과장 없이 "숫자로 증명".
- **레퍼런스(경쟁/벤치마크)**: 보라웨어(boraware.kr), 비드스토리, 애드매치, 그리고 글로벌 SaaS 대시보드(Linear, Vercel, Stripe Dashboard)의 정돈된 데이터 UI.
- **분위기 키워드**: 자동화 / 실시간 / 절감(CPC↓) / 통합 / 안심(합법·공식 API).

## 3. 브랜드
- 이름: **유어애즈 / UR Ads** (유어팀·유어딜과 같은 "유어/UR" 계열).
- 로고: "UR Ads" 워드마크 + 심볼(상승 그래프/타겟 모티프).
- 컬러: 유어팀 계열 톤 유지(기본 잉크/뉴트럴 + 포인트 1색). 다크/라이트 양모드 지원.
- 분리 원칙: 유어딜(소비자, 다크) / 유통스타트(도매) 와 **시각적으로 구분되는 독립 정체성**.

## 4. 페이지 2종
### (A) 랜딩(소개) 페이지 — 신규 고객 유치
- Hero: "광고 입찰부터 발주까지, 하나로 자동화" + 무료체험 CTA + 핵심 수치(예: CPC 최대 15%↓, 24/365 자동).
- 기능 6 섹션(아래 5번) — 각 기능 1스크린(아이콘+설명+미니 시각화).
- 사회적 증명(고객 로고/후기/누적 절감액), 요금제 표, FAQ, 최종 CTA.
- 톤: 보라웨어식 "포인트 01~06" 흐름 + 모던 SaaS 랜딩.

### (B) 대시보드(앱) — 로그인 후 실사용
- 좌측/상단 네비 + 메인 데이터 영역(**PC 전체 너비**, 모바일 대응).
- 멀티테넌트(대행): 고객사 전환 셀렉터.
- 위젯: 통합실적 카드, 자동입찰 현황, 발주 수집 리스트, AI 추천 패널.

## 5. 기능 섹션 (모두 디자인 — 네이버 공식 API 기반)
각 기능은 **랜딩의 소개 블록 + 대시보드의 실제 화면** 2벌이 필요.

| # | 기능 | 핵심 메시지 | 화면 요구 |
|---|---|---|---|
| 1 | **자동입찰** | 목표순위·최대입찰가만 → 최저 CPC 자동 | 키워드 테이블(목표순위/현재순위/입찰가/CPC), 시간대·요일 전략 설정, 대량/CSV |
| 2 | **부정클릭 방어** | 의심 클릭 탐지 + IP 차단으로 광고비 보호 | 의심 클릭 타임라인, IP 차단 목록, 절감 추정 |
| 3 | **키워드 확장** | 상품에 맞는 고매출 키워드 자동 발굴 | 연관키워드(검색량/경쟁도), 추가/제외 관리, 트렌드 그래프 |
| 4 | **통합 실적** | 광고비→클릭→주문→매출(ROAS) 한 화면 | 퍼널/ROAS 대시보드, 기간 비교, 채널별 |
| 5 | **AI 마케터** | 데이터를 읽고 다음 액션을 제안 | 채팅/제안 카드("이 키워드 입찰 올리세요"), 자동 리포트 |
| 6 | **발주 수집** | 여러 스토어 주문을 자동 한곳에 | 주문 통합 리스트, 스토어 연동 상태, 발송처리 |

## 6. 디자인 산출물(요청)
1. 랜딩 페이지 풀 시안 (PC + 모바일)
2. 대시보드 핵심 4화면 (통합실적 / 자동입찰 / 발주수집 / AI마케터) — PC + 모바일
3. 디자인 시스템(컬러·타이포·버튼·카드·테이블·차트 스타일) + 다크/라이트
4. 로고/워드마크 + 아이콘 6종(기능별)
5. Figma 원본 + 개발 핸드오프(스펙)

## 7. 제약(개발 측)
- **PC 전체 너비** 대시보드(유어딜 모바일 액자 X — 도매몰과 동일).
- **다크/라이트 양모드** 토큰.
- 모바일 반응형 필수(셀러가 폰으로도 확인).
- 차트는 가벼운 라이브러리 전제(과한 3D/애니메이션 지양).

## 8. 합법성 메모(디자인 카피에 반영)
- "**네이버 공식 API 기반**" 강조(크롤링 아님 → 안심). 순위는 공식 추정치 기반임을 과장 없이.

## 참고: 네이버 API로 가능한 범위 (개발 확인)
- **자동입찰**: 검색광고 API — Estimate(목표순위/중간/최소 입찰가) + 키워드 입찰 변경. (실시간 정확순위 X → 공식 추정)
- **키워드확장**: RelKwdStat(연관키워드+검색량) + 데이터랩(트렌드/쇼핑인사이트) + 쇼핑 검색.
- **통합실적**: StatReport/MasterReport(노출·클릭·비용·전환) + 커머스 API(주문/매출) → ROAS.
- **부정클릭**: 네이버가 1차 필터 + API의 IP Exclusion + 자체 클릭로그 이상탐지(보완).
- **AI마케터**: 위 데이터 → 자체 AI(Workers AI/LLM) 진단·제안.
- **대행/멀티테넌트**: ManagedCustomerLink(광고) + 커머스 SELF/솔루션 연결(발주).
- **광고비**: Bizmoney 잔액 모니터링/알림.
