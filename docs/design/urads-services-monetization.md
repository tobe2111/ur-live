# 유어애즈 — 추가 서비스 & 수익화 전략

> 2026-06-27 대표 질문: "마케팅 대행사로서 고객사에 줄 추가 서비스? 유어애즈 수익화는?"
> 이미 구현된 것(보라웨어 5종 + 가격모니터링 등)을 토대로 *확장 서비스 후보* + *수익화 모델* 정리.
> 현 상태: 전부 무료/무제한(게이트 0). 수익화하려면 플랜 게이트 + 결제 연동이 전제.

## A. 추가 마케팅 서비스 후보

### A-1. 지금 API로 바로/적게 추가 (네이버, 고정IP 불필요)
| 서비스 | API | 연동 | 가치 |
|---|---|---|---|
| **광고 소재 A/B 관리** | 검색광고 Ad(소재) | 필요 | 제목/설명 변형 등록·성과비교·우승소재 자동적용 |
| **예산 자동 페이싱·알림** | 검색광고 Budget | 필요 | 일예산 과속/과소 감지 → 알림/자동조절 |
| **노출순위 추적 대시보드** | StatReport(평균노출순위) | 필요 | 자동입찰과 시너지(공식 지표, 스크래핑 X) |
| **쇼핑인사이트 소싱 리포트** | 데이터랩 쇼핑인사이트 | 불필요 | 뜨는 카테고리/키워드 → 도매몰 시너지(뭘 팔지) |
| **경쟁사 모니터링** | 쇼핑검색+데이터랩 | 불필요 | 경쟁사 가격/노출 추적(가격모니터링 확장) |
| **평판 모니터링 + 알림** | 오픈API 블로그/뉴스 | 불필요 | 정기 모니터링 + 부정 언급 알림(현 조회를 구독화) |

### A-2. 새 외부 통합 (네이버 밖 — 진짜 "통합")
- **다매체 통합관리/실적**: Google Ads · Kakao Moment · Meta(인스타/페북) Ads API → 보라웨어 통합실적의 멀티매체 버전.
- **GA4/전환 추적 연동**: 광고→실제 매출 연결(ROAS 정확도). **부정클릭 픽셀 인프라 재활용 가능**.
- **알림톡/이메일 리포트 구독**: 통합실적+AI마케터 데이터를 주간/월간 자동 발송(Aligo·Resend 이미 보유).

### A-3. "사람+도구" 운영 서비스 (소프트웨어 아님 — 마진 큼)
- **매니지드 대행**: 자동입찰 엔진 + 사람 검수로 우리가 직접 운영(보라웨어 "프리미엄 무료대행" 모델 = 광고비 수수료).
- **콘텐츠 제작**: AI마케터로 소재/블로그 초안 → 사람 다듬기.
- **세팅 대행/컨설팅**: 초기 계정·키워드 설계.

### A-4. 🌟 유어딜/도매몰 시너지 (보라웨어엔 없는 우리만의 무기)
- **유어딜 판매채널 번들**: 광고주 상품을 유어딜 공구/교환권으로 노출 → 광고 + *실판매처* 동시 제공.
- **풀스택 수직통합**: 도매몰(소싱) → 유어애즈(광고) → 유어딜(판매) → 정산. "광고만"이 아니라 매출까지.
- **AI마케터 무제한**: 자체 Claude 보유 → 한계비용 낮아 상위 티어 강력한 훅.

## B. 수익화 모델

### B-1. 모델 옵션
1. **SaaS 구독(월정액·티어)** — 보라웨어식. 기능/키워드수/사이트수 차등 + 14일 무료체험.
2. **성과 기반(대행 수수료)** — 관리계정 운영 시 광고비의 10~20%(에이전시 표준).
3. **사용량(PPC-style)** — 자동입찰 키워드당 / 부정클릭 1만건당(보라웨어 ₩12,460/1만건 관측).
4. **부가 서비스** — 콘텐츠 제작·컨설팅 별도 과금.

### B-2. 추천 = 하이브리드 (퍼널형)
```
무료(미끼)        : 키워드도구·가격모니터링·평판조회 (읽기) — 가입 유도
─────────────────────────────────────────────
베이직(₩/월)      : 부정클릭 1사이트 + 자동입찰 N키워드 + 통합실적
프로(₩/월)        : 다사이트 + 키워드 무제한 + AI마케터 + 우선지원
─────────────────────────────────────────────
매니지드(성과)    : 광고비 X% — 우리가 직접 운영(자동입찰+사람)
부가             : 콘텐츠/컨설팅/유어딜 판매번들 — 별도
```
- **결제 인프라 재활용**: 유어딜 Toss/딜포인트/사업자 정산 엔진 그대로 → 신규 PG 불필요.
- **락인**: 유어딜 판매채널 번들 + AI마케터로 차별화 → 단순 입찰툴(보라웨어)보다 이탈 어려움.

### B-3. 수익화 구현 전제 (코드)
1. **플랜 게이트**: `ad_subscriptions(seller_id, plan, status, period_end)` + `requirePlan(seller, feature)` 헬퍼. 유료 기능(자동입찰 규칙생성·부정클릭 사이트등록·AI마케터)에 게이트.
2. **사용량 미터링**: 자동입찰 키워드 수·부정클릭 이벤트 수·AI 호출 수 카운터(티어 한도).
3. **결제/업그레이드**: `/ads/pricing` + 유어딜 결제 helper 호출(잠금파일 미수정, helper만 호출).
4. **무료체험**: period_end 기반 14일 trial.

## C. 우선순위 제안
1. **수익화 토대(플랜 게이트 + 가격 페이지)** — 모델 무관 공통 인프라. 가격은 config 로 분리(대표가 숫자만 결정).
2. **소싱 리포트(데이터랩 쇼핑인사이트)** — 도매몰 시너지 + 연동 불필요(빠름).
3. **노출순위 추적 + 예산 페이싱** — 자동입찰 고도화(연동 후).
4. **다매체(구글/메타/카카오)** — 큰 통합, 후순위.
