# 🏭 도매몰 통합 셸 (Unified Wholesale Shell) — 설계 제안

> 대표 피드백 (2026-06-29): "우리 도매몰은 너무 제조사/판매사 대시보드와 메인 도매몰 페이지와 분리된 느낌이야."
> → 카탈로그(메인 매장) ↔ 판매사 대시보드 ↔ 제조사 대시보드가 **각자 다른 사이트처럼** 느껴지는 문제. 한 제품처럼.

## 1. 현재 구조 — 3개의 분리된 셸 (분리감의 원인)

| 표면 | 라우트 | 셸/네비 크롬 | 로그인 | 느낌 |
|---|---|---|---|---|
| **카탈로그(메인 매장)** | `/wholesale`(+best/new/margin/premium/brands) | **자체 풀헤더**(몰 브랜드·검색·카테고리·로그인) — 스토어프론트 | `/wholesale/login` (seller_token) | 쇼핑몰 |
| **판매사 대시보드/앱** | `/wholesale/dashboard·deposits·orders·statement·…` | `WholesaleLayout` → **WholesaleUtilBar**(얇은 다크 바: 공지·예치금·충전·대시보드·로그아웃) | 동일 | 관리도구 |
| **제조사 대시보드** | `/supplier`(+wholesale-orders) | **WholesaleDashboardShell**(좌측 사이드바, 완전 별도) | `/supplier/login` (supplier_token) | 별도 사이트 |

**분리감이 생기는 지점**
1. **크롬이 3종류** — 카탈로그의 풀헤더 ↔ 대시보드의 얇은 유틸바 ↔ 제조사의 사이드바 셸. 이동할 때마다 화면 골격이 통째로 바뀜 = "다른 곳에 왔다" 체감.
2. **카탈로그와 판매사 대시보드가 상단바를 공유 안 함** — 유틸바는 대시보드 페이지엔 있는데 **정작 메인 카탈로그엔 없음**(카탈로그는 자체 헤더). 그래서 매장↔내 대시보드 전환이 매끄럽지 않음.
3. **제조사는 완전한 섬** — 다른 셸·다른 로그인·다른 토큰. '제조사 대시보드' 버튼으로만 건너가고, 건너가면 유통스타트가 아닌 듯한 사이트.
4. **"내 공간" 단일 진입점 부재** — 주문/예치금/정산/거래내역서/내 상품이 표면마다 흩어져 있음.

## 2. 이상적인 형태 — "하나의 도매몰 셸, 안에서 역할만 바뀐다"

핵심: **모든 도매 표면(`/wholesale*`·`/supplier*`)이 동일한 글로벌 상단바(+필요 시 좌측 레일)를 공유**하고, 역할별 콘텐츠는 그 **틀 안에서 스왑**된다. 대시보드는 *별도 페이지*가 아니라 *매장 안의 패널*.

### 2-1. 통합 글로벌 상단바 `WholesaleAppBar` (전 표면 1개)
오늘의 `CatalogHeader`(계정 클러스터) + `WholesaleUtilBar` 를 **하나로 병합**해 카탈로그·판매사·제조사 모두에 마운트.
- **좌**: 로고(→/wholesale) + 1차 네비. 구매자=둘러보기/베스트/신상/마진순 · 제조사=내 상품/주문/정산.
- **중앙**: 검색(카탈로그 맥락).
- **우(역할 인지 계정 클러스터)**: 예치금 잔액+충전(판매사) / 정산 잔액(제조사) · 알림 · **"내 공간" 드롭다운**(내 대시보드 / 주문 / 예치금 / 거래내역서 / 로그아웃). 비로그인=로그인/회원가입.

### 2-2. 대시보드 = 셸 안의 탭 패널 (별도 사이트 아님)
- **판매사**: `/wholesale/dashboard` 가 같은 상단바 아래 탭(개요/주문/예치금/거래내역서/찜). 카탈로그↔대시보드 전환이 크롬 변화 없이 *같은 틀 안 이동*. (이미 `WholesaleLayout` 절반 존재 → **카탈로그까지 같은 바로 끌어들이기**.)
- **제조사**: `/supplier` 가 **동일 상단바** + 제조사용 좌측 레일(상품/주문/정산/출금/채팅). 유통스타트 같은 제품에 '제조사 패널'이 얹힌 모습.

### 2-3. 역할 인지 단일 진입 `/wholesale`
서버/클라가 역할로 분기하되 **틀은 동일** — 구매자=스토어프론트, 제조사=공급자 홈, 그러나 셸은 같음. (2026-06-29 추가한 제조사→/supplier 안내가 통합 셸에선 *같은 틀 안 전환*이 되어 더 자연스러움.)

### 2-4. 일관 디자인 토큰 + 동선 직조
- 전 표면 `WT` 토큰 통일(카드/버튼/간격/로딩 — 로딩은 2026-06-29 이미 통일).
- "떠난 느낌" 제거: 상품 상세에서 판매사=내 주문 담기/내 대시보드, 제조사=카탈로그 미리보기 링크.

## 3. 단계별 구현 (안전·점진·서비스 격리)

- **Phase 1 (기반)**: `WholesaleAppBar` 추출 = CatalogHeader 계정부 + WholesaleUtilBar 병합 → **카탈로그도 `WholesaleLayout` 아래로** 넣어 카탈로그↔판매사 대시보드가 동일 상단바 공유. (체감 분리 즉시 절반 해소)
- **Phase 2**: `/supplier` 를 동일 `WholesaleAppBar` + 몰 톤 좌측 레일로 — 제조사 섬 제거. 데이터/로직 무변경(셸만 교체).
- **Phase 3**: "내 공간" 드롭다운 · 상시 예치금/정산 칩 · 크로스링크 · 빈/로딩 상태 일관.

## 4. 비범위 / 주의
- 서비스 격리 유지: `/wholesale`·`/supplier`·`src/features/supply`·`Wholesale*`/`Supplier*` 만. 소비자(유어딜)·어드민 콘솔 무관.
- 토큰/세션·결제·정산 로직 **무변경** — 본 작업은 셸/네비/레이아웃 통합(프레젠테이션)만.
- 라이트 고정 B2B 테마 유지(dark variant 없음).
