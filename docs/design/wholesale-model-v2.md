# 유통스타트 도매 모델 v2 — admin-gated 가격/등급/노출

> 출처: 2026-06-17 대표 확정 (채팅). ⚠️ **머니 크리티컬 + 잠금 영역** — 구현 전 본 문서 + 마이그레이션
> 계획 + staging 검증 필수. 확정 전 코드 변경 금지.

## 핵심 전환 (현행 → v2)
| 영역 | 현행 (retail-down, 등급=가격) | v2 (cost-up, 등급=노출) |
|---|---|---|
| 제조사 입력 | 원가 + 권장소비자가(판매가) | **공급가**(제조사가 받을 금액) |
| 가격 산식 | 공급가 = 판매가 × (1 − 등급마진%) | **유통사가 = 공급가 × (1 + 마진%)** |
| 마진 결정 | 등급별 보장마진(일반15/프로30/프리미엄38) | **어드민이 상품 승인 시 마진% 지정**(기본 10%) |
| 정산 | 제조사 = 공급가×90% / 플랫폼 = 공급가×10% | **제조사 = 공급가 전액 / 플랫폼 = 공급가×마진%** |
| 등급 역할 | 가격 차등(등급별 다른 가격) | **노출 큐레이션 — 어드민이 등급별 노출 상품 지정** (가격 차등 X, 모든 등급 동일가) |
| 부가세 | (별도/미정) | **노출가 = 부가세 포함** |

## 플로우
1. **제조사 등록**: 공급가 입력 → `supply_approval_status='pending'`.
2. **어드민 승인 (게이트)**: 어드민이 **마진%(기본 10%) + 노출 범위 + 등급 노출**을 정하고 승인 → `approved` + `is_active=1`.
3. **유통사 노출**: 가격 = 공급가 × (1+마진%) (부가세 포함, 단일가). 단 **본인 등급에 노출 지정된 상품만** 보임.
4. **결제/정산**: 유통사가 (공급가×(1+마진%)) 결제 → 제조사 공급가 전액, 플랫폼 마진.

## 기존 인프라 (재사용)
- ✅ 승인 플로우 — `supply_approval_status` (pending/approved/rejected)
- ✅ 상품별 마진 — `products.supply_margin_override_pct` (이미 존재! 카탈로그 쿼리에서 `margin_override` 로 읽음)
- ✅ 노출 범위 — `supply_visibility` (ALL/APPROVED_CHANNEL/UTONGSTART_ONLY)
- ✅ 등급 — `sellers.distributor_grade` (현재 가격용 → 노출 큐레이션용으로 의미 전환)
- ✅ 정산 SSOT — `splitWholesaleUnit()` (식만 교체)

## 신규 필요
- 🔧 가격엔진 `distributorPriceFromRetail` → `공급가 × (1+마진%)` (또는 신규 함수). 잠금/머니.
- 🔧 정산 `splitWholesaleUnit` → 제조사=공급가, 플랫폼=공급가×마진%. 잠금/머니.
- 🆕 **등급별 노출 큐레이션** — 어드민이 등급↔상품 매핑. 데이터 모델(예: `product_grade_visibility` 또는 상품에 `min_grade`) + 어드민 UI. 카탈로그 WHERE 에 등급 노출 필터 추가.
- 🔧 제조사 입력폼 — 공급가 입력(원가/판매가 정리).
- 🔧 카탈로그/상세/장바구니 가격 표시 — 새 산식.
- 🔧 환불 역전 — 새 정산식 대칭.

## 열린 질문 (확정 필요)
1. ✅ **권장소비자가(판매가) = 유지** (2026-06-17 대표 확정 "A"). **표시용·선택 입력** — 가격 계산엔
   안 씀. 유통사 화면에 권장가(취소선) + "되팔면 마진 N%" 가이드. 제조사가 비우면 마진 가이드만 미표시.
2. ✅ **기본 마진율 = 10% default + 어드민 상품 승인 시 조정** (`products.supply_margin_override_pct`,
   미지정 시 10%). 부가세 포함.
3. ⬜ **등급별 노출 모델** (Phase 2 — 미확정): ⓐ 상품마다 "최소 등급" / ⓑ 등급↔상품 다대다 매핑. → Phase 2 시작 전 확정.
4. ⬜ **원가**: 제조사 원가 입력 여부 — 정산 하한 불필요(제조사=공급가 전액)이므로 **공급가만으로 충분**(원가 칸 제거 가능). Phase 1 에서 확정.

## 구현 순서 (확정 후)
1. 설계 확정(본 문서) → 2. 가격엔진+정산 식 교체 + 단위테스트 → 3. 등급 노출 큐레이션(데이터+어드민 UI) → 4. 입력폼/표시 정합 → 5. 환불 역전 대칭 → 6. **staging 결제 E2E 필수** → 7. 운영 반영.
