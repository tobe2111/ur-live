# 유어애즈 — 부정클릭 방지 시스템 설계 (보라웨어 레퍼런스)

> 시안: `urads-boraware-reference.md` §2. 보라웨어 "국내유일 100% 자동관리! 검색광고 부정클릭 방지시스템".
> 다른 4종(자동입찰/키워드확장/통합실적/AI마케터)과 달리 **아키텍처가 근본적으로 다름** — 광고 API 호출이 아니라
> *광고주 랜딩페이지 방문자 추적* + *부정 IP 차단*. 그래서 별도 설계 문서 + 대표 결정 2건 선행.

## 무엇을 하는가 (보라웨어 6블록 → 기능)
1. 단 한 번 설정으로 부정클릭 100% 자동관리.
2. IP 바꿔 접속하는 악성 부정클릭 차단(반복 클릭 탐지).
3. (옵션) 무효클릭 접속자 광고 차단.
4. 접속자 위치정보 제공(부정클릭 집중 지역).
5. 접속자 활동성 분석(단골 vs 부정).
6. PC·모바일 자동보고(IP 포함).

## 동작 원리 (3단계)
1. **수집**: 광고주 랜딩페이지에 **추적 스크립트(픽셀)** 1줄 삽입 → 방문 발생 시 우리 엔드포인트로
   `{ 광고주ID, IP, UA, referrer, 방문시각, (네이버 광고 유입 여부=gclid류/utm), 화면/행동 }` 전송.
2. **판정**: 같은 IP/디바이스가 짧은 시간에 **N회 이상 광고 클릭** + 체류 0초 / 전환 0 → 부정 의심.
   화이트리스트(단골=전환 이력 IP)는 제외. 규칙 + (후속) 이상탐지.
3. **차단**: 의심 IP 를 네이버 검색광고 **"노출제한 IP"** 목록에 등록 → 그 IP 엔 광고 미노출.

## ⚠️ 결정 필요 2건 (대표)
### 결정 A — 개인정보(PIPA) 처리
방문자 **IP·위치·행동 로그 수집**은 개인정보. 합법 운영 조건:
- 광고주(우리 고객사)가 자기 사이트 **개인정보처리방침에 "부정클릭 방지 위한 접속기록 수집/위탁" 고지** + 우리는 **수탁자(위탁처리)**.
- 우리 쪽: 수집 최소화(IP 해시 저장 옵션) · 보관기간 한정(예: 90일) · 목적 외 사용 금지 · 마케팅 재사용 금지.
- 2026-04-22 네이버 광고주 *크롤러* 제거(PIPA/정보통신망법)와 **다른 건**(이건 자사 픽셀의 1자 수집 — 합법 가능) 이나, 처리방침/위탁계약은 필수.
- **결정**: 이 방식으로 갈지(픽셀+위탁), 아니면 IP 해시만 저장하는 보수적 버전으로 갈지.

### 결정 B — 네이버 "노출제한 IP" 자동 등록 가능 여부
- 노출제한 IP 는 검색광고센터 UI 기능(계정당 최대 600개). **검색광고 공식 API 가 이 관리(추가/삭제)를 노출하는지 라이브 확인 필요.**
- **API 지원 O** → 완전 자동(보라웨어처럼). **API 지원 X** → 반자동: 우리가 *의심 IP 리스트*를 만들어 광고주에게 제공 → 광고주가 센터에 붙여넣기(또는 우리가 위임받아 수동).
- 이 환경은 egress 차단이라 확인 불가 → 배포 후 또는 문서 확인 필요.

## 단계별 구현안 (결정 후)
- **Phase 1 (수집·판정, write 0 — PIPA 결정 A 후)**: 픽셀 엔드포인트 `POST /api/ads/clickguard/hit`(레이트리밋·CORS) + `ad_click_events` 테이블(광고주별, IP 해시 옵션, 90일 TTL) + 의심 IP 리포트 `GET /api/ads/clickguard/report`(IP별 클릭수·지역·의심도). **차단 없이 "탐지·리포트"만** = 즉시 가치 + 저위험.
- **Phase 2 (차단 — 결정 B 후)**: API 지원 시 의심 IP 자동 등록 / 미지원 시 리포트 export(복붙용) + 가이드.
- **Phase 3**: 단골 화이트리스트(전환 IP) · 위치 히트맵 · 자동 일일보고(이메일/알림톡).

## 스키마(초안 — Phase 1)
```
ad_click_events(
  id, seller_id, advertiser_key, ip_hash, ua, referrer, landing_url,
  is_ad_inflow INT, dwell_ms INT, created_at,  -- 90일 TTL(cron 정리)
  INDEX(seller_id, created_at), INDEX(seller_id, ip_hash)
)
```
- `advertiser_key` = 광고주별 발급 토큰(픽셀에 박힘, 위조 방지용 도메인 검증).
- IP 는 해시(salt) 저장 옵션 — 동일성 판정엔 해시로 충분, 원문 IP 보관 최소화.
