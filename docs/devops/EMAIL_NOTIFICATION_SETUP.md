# 📧 라이브 스트림 생성 이메일 알림 기능

## 개요
셀러가 새로운 라이브 스트림을 개설하면 자동으로 관리자 이메일(jiwon@ur-team.com)로 알림을 전송합니다.

---

## 구현 방식

### 무료 이메일 전송: MailChannels
- **서비스**: MailChannels (Cloudflare Workers 전용)
- **비용**: 100% 무료
- **제한**: 없음 (Cloudflare Workers 사용 시)
- **API 키**: 필요 없음
- **설정**: 추가 설정 불필요

### 기술 스택
```
Cloudflare Workers → MailChannels API → 이메일 전송
```

---

## 이메일 내용

### 제목
```
[유어 라이브] 🎉 새 라이브 스트림 생성: {라이브 제목}
```

### 포함 정보
1. ✅ **라이브 상태**: 🔴 라이브 중 / 📅 예약됨 / ⏸️ 대기 중
2. ✅ **라이브 제목**: 셀러가 입력한 제목
3. ✅ **판매자명**: 셀러의 display_name 또는 username
4. ✅ **플랫폼**: 📺 YouTube 또는 🎵 TikTok
5. ✅ **예약 시간**: 예약된 경우에만 표시
6. ✅ **라이브 ID**: 고유 식별 번호
7. ✅ **바로가기 링크**: https://live.ur-team.com/live/{streamId}

## 이메일 디자인

### HTML 이메일
- 📱 모바일 최적화 (max-width: 600px)
- 🎨 그라데이션 헤더 (보라-핑크)
- 🔘 버튼 스타일 링크
- 📊 정보 박스 레이아웃
- 🏢 회사 정보 푸터

### 텍스트 이메일 (폴백)
- HTML을 지원하지 않는 이메일 클라이언트용
- 동일한 정보를 텍스트로 제공

---

## 발송 흐름

```
1. 셀러가 "라이브 생성" 버튼 클릭
   ↓
2. POST /api/seller/streams 호출
   ↓
3. DB에 라이브 스트림 저장
   ↓
4. 이메일 전송 (비동기)
   ├─ 성공: 로그 기록
   └─ 실패: 에러 로그 (라이브 생성은 성공)
   ↓
5. 셀러에게 성공 응답 반환
```

**중요**: 이메일 전송 실패해도 라이브 생성은 성공합니다!

## 주의사항

### ⚠️ 발신자 이메일
현재 발신자: `noreply@live.ur-team.com`

**중요**: 이 이메일 주소는 실제로 존재하지 않아도 됩니다!
- MailChannels는 도메인 소유권 검증 없이 전송 가능
- 단, 스팸으로 분류될 가능성 있음

**개선 방법** (선택사항):
1. Cloudflare Email Routing 설정
2. 실제 도메인 이메일 사용 (`support@live.ur-team.com`)
3. SPF/DKIM 레코드 추가 (도메인 DNS 설정)

### 📊 전송 제한
- MailChannels: 제한 없음 (Cloudflare Workers 사용 시)
- 단, 과도한 전송 시 일시적 차단 가능
- 예상 사용량: 하루 10~50개 정도면 전혀 문제 없음

### 🔒 보안
- API 키 불필요 (MailChannels)
- 추가 인증 불필요
- Cloudflare Workers에서만 작동
