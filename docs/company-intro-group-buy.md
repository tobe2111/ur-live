## 2. 시장 문제 (Why we exist)

### 2.1 기존 공동구매의 한계
| 기존 플랫폼 | 한계 |
|---|---|
| 카카오 선물하기 / 쿠팡 / 11번가 | 배송 상품 중심, **오프라인 매장 voucher** 약함 |
| 위메프 / 티몬 | 가격 경쟁만, **라이브 송출** X |
| 인스타그램 라이브 / 유튜브 라이브 | 결제·정산·매장 voucher **자체 인프라 없음** |
| 식권대장 / 식권나라 | B2B 식권 only, **C2C 공구 (매장 직접 운영)** 안 됨 |
| 데일리호텔 / 야놀자 | 호텔 only, **식사/미용/기타** 카테고리 없음 |

### 2.2 셀러가 겪는 진입 장벽
- **사업자등록증 없는 개인** (학생 인플 / 부업 셀러 / 한 끼 식당 사장님) 은 정산 어렵음
- 부가세 신고 의무 + 매장용 통장 + 세금계산서 발행 등 진입 비용 큼
- 결과: 시장에 잠재력 있는 셀러가 진입 못 함

### 2.3 소비자가 겪는 문제
- 오프라인 매장 vouchers (식사권/숙소권) 가 흩어져 있음 — 통합 검색 X
- 라이브 송출 보면서 매장 voucher 구매하는 흐름 없음
- 모집형 공구 진행 상황 (모집률, 마감 시간) 실시간 확인 어려움

→ **유어딜은 위 3가지 문제를 한 번에 풀어내는 통합 플랫폼**.

## 3. 핵심 가치 제안 (Value Proposition)

### 3.1 셀러에게
1. **사업자 등록 없어도 수익 가능** — 상품권 (기프티쇼) 또는 자체 포인트 (딜) 로 정산
2. **라이브 X 공구 X 일반 판매를 한 화면에서** — 인플은 라이브 위주, 매장 사장님은 공구 위주, 둘 다 가능
3. **수수료 5%** (기본, 어드민 협의 가능) — 업계 최저 수준
4. **즉시 정산** (상품권 모드) — 송금 D+7 대기 없음

### 3.2 소비자에게
1. **메인 페이지 하나로** 라이브 + 오프라인 공구 + 온라인 쇼핑 통합 탐색
2. **4 카테고리** (🍽️ 식사권 / 💇 미용 / 🏨 숙소 / 🎯 기타) 단순 명확
3. **모집형 공구** — 가격 최대 70% 할인 + 사회적 증명 (구매자 수 실시간 노출)
4. **카카오 1초 로그인** — 한국 사용자 진입 마찰 최소
5. **알림톡** — 공구 마감 임박 / 사용 가능 매장 알림

### 3.3 에이전시에게
1. **다수 셀러 일괄 관리** — 본인 사업이 아닌 셀러 영입/계약/매칭 도구
2. **PK 이벤트, 캠페인, 인센티브** — TikTok Backstage 학습 기반 도구
3. **라이브 / 공구 셀러 동시 관리** — Mode 토글로 view 전환

## 4. 타겟 사용자 (Persona)

### 4.1 소비자
| 페르소나 | 특징 |
|---|---|
| **30대 직장인** | 점심 식사권, 퇴근 후 미용실, 주말 호캉스 |
| **20대 학생** | 학교 근처 카페/맛집 공구, 후원형 라이브 |
| **40-50대 주부** | 동네 미용실, 가족 외식, 펜션 |

핵심 사용 패턴:
- 모바일 first (95%+ 모바일 사용)
- 알림톡 → 클릭 → 결제 (전체 흐름 3분 이내)
- 메인 → 카테고리 → 매장 상세 → 결제 (최대 5 step)

### 4.2 셀러 (3 타입)
| 타입 | 비중 | 특징 |
|---|---|---|
| **라이브 셀러** (influencer) | 30% | 유튜브/인스타 라이브 송출, UR특가 상품 판매, 후원 받음 |
| **공구 셀러** (store_owner) | 50% | 오프라인 매장 운영, 식사권/숙소 voucher 발행 |
| **혼합** (both) | 20% | 라이브 송출 + 매장 보유 — 둘 다 활용 |

수익 패턴:
- 라이브 셀러: 라이브 시 상품 판매 + 후원 (15% 수수료)
- 공구 셀러: voucher 발행 → 모집 → 매장 방문 시 사용
- 혼합: 라이브 송출 중 본인 매장 voucher 노출

### 4.3 에이전시
- 다수 셀러 관리하는 MCN / 광고 대행사 / 지역 협회
- 자체 셀러 영입 / 라이브 캠페인 운영 / 정산 대행
- 평균 담당 셀러 수: 10-30명

## 5. 핵심 서비스 — 공동구매 (Group Buy)

### 5.1 정의
**일정 기간 + 최소 인원 모집 조건을 충족하면 상품/서비스를 큰 할인가에 구매할 수 있는 모집형 결제 시스템.**

### 5.2 4 카테고리 (2026-05-17 6종 → 4종 통합)
| 카테고리 | 이모지 | 예시 |
|---|---|---|
| 식사권 | 🍽️ | 한정식 1인 4만원→2.5만 / 카페 음료 10잔 1만원 |
| 미용 | 💇 | 헤어컷+염색 8만→4.5만 / 네일 5만→2.8만 |
| 숙소 | 🏨 | 펜션 1박 12만→7만 / 모텔 6만→3.5만 |
| 기타 | 🎯 | 클라이밍 1일권 / 펫 그루밍 / 액티비티 |

### 5.3 모집 흐름
```
1. 셀러가 공구 등록
   - 정가, 공구가, 최소 인원 (예: 20명), 마감 시간 (예: 7일 후)
2. 소비자 결제 (선결제, 카드/계좌)
   - 결제는 즉시 진행, 매장 voucher 발급 보류
3. 모집 기간 진행
   - 메인 페이지에 "현재 N명 모집, 마감 X시간" 실시간 표시
   - 사회적 증명: "20명 이상 → HOT" / "마감 1시간 → 마감임박" 배지
4. 마감 시점
   ├ ✅ 달성: voucher 자동 발급 + 카카오 알림톡 발송
   └ ❌ 미달: 결제 자동 환불 (카드 취소)
5. 매장 방문 시 voucher 사용 (QR 코드 또는 코드 입력)
```

### 5.4 라이브 커머스와의 결합
- 라이브 송출 중 셀러가 "지금 50% 할인" 클릭 → 메인 페이지에 즉시 노출
- 시청자가 라이브 보면서 동시에 구매 (TikTok Shop 패턴)
- 라이브 분석 페이지에 "이번 방송 매출 ₩X" 즉시 표시

## 6. 기술 스택 (Tech Stack)

### 6.1 인프라
- **Cloudflare Pages** (with `_worker.js`) — 단일 배포 단위
- **Cloudflare D1** — SQLite-compatible distributed DB
- **Cloudflare R2** — 이미지/영상 저장
- **Cloudflare Workers Cron** — 정산/sync/모집 마감 처리
- 도메인: `live.ur-team.com` (한국) / `ur-live.pages.dev` (기본)

### 6.2 프론트엔드
- **React 18** + **TypeScript** + **Vite**
- **Tailwind CSS** (다크/라이트 테마 토글)
- **i18next** — 6개 언어 (한/영/일/중/스/프)
- 모바일 first + PC 반응형 (`md:`, `lg:`, `xl:`, `2xl:`)

### 6.3 백엔드 (Worker)
- **Hono v4** (Web framework)
- D1 prepared statements (SQL injection 방어)
- JWT 인증 (셀러/어드민/에이전시 분리)
- Rate limit (`RATE_LIMIT_KV`)
- Turnstile (봇 차단 — 후원/결제 진입점)

### 6.4 인증
- **카카오 OAuth** (한국 — 1초 로그인)
- **Firebase Auth** (글로벌 — 이메일/소셜)
- 한국 라우팅: `isKorea()` 분기 — Firebase 0 호출

### 6.5 결제 / 알림
- **토스 페이먼츠** — 카드/계좌/간편결제
- **카카오 알림톡** — 공구 마감/매장 방문 안내
- **YouTube Live** — 셀러 라이브 송출 (Persistent stream)

### 6.6 모니터링
- **Sentry** — 에러 트래킹 + 소스맵 자동 업로드
- 어드민 라이브 모니터링 — 실시간 시청자 / 매출 / 채팅
- OME 헬스 체크 + 알람 (시청자 급감 시)

## 9. 셀러 정산 시스템 (핵심 차별화)

### 9.1 사업자 등록 검증 게이팅
```
셀러 가입 시:
  business_registration_status = 'pending' (default)

어드민 검증 후:
  ├ verified → 모든 정산 방식 가능 (현금/상품권/딜)
  ├ rejected → 사유 안내 + 재신청
  └ exempt   → 어드민 수동 처리
```

### 9.2 정산 방식 3가지
| 방식 | 사업자 등록 | 원천징수 | 환금성 | 즉시성 |
|---|---|---|---|---|
| **현금 (계좌입금)** | 필수 | 면제 (세금계산서 별도) | 100% | D+7 |
| **모바일 상품권** (기프티쇼) | 불요 | 8.8% 차감 | 표면 100%, 실제 거래시장 80% | 즉시 |
| **포인트 (딜)** | 불요 | 8.8% 차감 시점 분기 | 플랫폼 내 사용 only (현금화 X) | 즉시 |

### 9.3 비사업자 셀러의 딜 사용 정책
- 딜로 정산 받음 → **플랫폼 내 결제만 가능**
- 환급 (현금화) 시도 시 차단: "사업자 등록증을 등록하면 현금 환급이 가능합니다" 안내
- 가입 시 사업자등록증 업로드 → 검증 후 환급 unlock

### 9.4 세무 자동화
- 비사업자에게 지급 시 자동 8.8% 원천징수
- `tax_withholding_log` 에 모든 거래 기록
- 연 누계 300만원 초과 시 자동 경고 + 종합소득 신고 안내

## 10. 비즈니스 모델 / 수익 구조

### 10.1 수수료 구조
| 항목 | 수수료 | 비고 |
|---|---|---|
| 셀러 매출 (기본) | 5% | `platform_settings.commission_rate_default` |
| 셀러 매출 (협의) | 3-10% | 어드민이 셀러별 조정 (sellers.commission_rate) |
| 후원 (라이브 도네이션) | 15% | 별도 |
| 광고 슬롯 (메인 상단) | 입찰가 | 매일 18시 낙찰 |
| 정산 상품권 발급 | KT Alpha 마진 (3-15%) | 별도 |

### 10.2 비용 구조
- Cloudflare 인프라 ~$50-200/월 (스케일 따라)
- 토스 결제 수수료 2.7% (매출에서 자동 차감)
- 카카오 알림톡 발송 ~₩6/건
- KT Alpha 보증금 + 후불 결제

### 10.3 매출 시뮬레이션
```
가정: 셀러 1000명, 평균 월 매출 ₩500,000 / 셀러

총 거래액 (GMV): ₩500,000 × 1000 = ₩500M / 월
플랫폼 수수료 5%: ₩25M / 월
후원 매출 (15%): 추가 ₩2-5M
광고 슬롯: 추가 ₩3-10M
────────────────
순 매출: ₩30-40M / 월
```

- 토스 Idempotency-Key 필수
- 결제 금액 서버 재계산 (클라이언트 신뢰 X)

- IDOR 방어: 리소스 소유권 체크
