### ① 엣지 캐싱 예외 케이스 (완료 ✅)
> **요청**: "품절" 처리나 "라이브 종료"가 되었는데, 캐시 때문에 유저에게 여전히 판매 중으로 보이면 안 됩니다.

#### 구현 결과
1. **실시간 데이터 캐시 예외 처리**
   - ✅ 재고 수량 (`/api/products/:id/stock`)
   - ✅ 라이브 상태 (`/api/streams/:id/status`)
   - ✅ 결제 상태 (`/api/orders/:id/status`)
   - ✅ 주문 상태 (`/api/orders/:id`)

2. **Authorization 헤더 스킵**
   ```typescript
   if (request.headers.has('Authorization') || 
       request.headers.has('X-Session-Token')) {
     return next(c)  // 캐시 건너뛰기
   }
   ```

3. **캐시 무효화 함수**
   ```typescript
   // 상품 재고 업데이트 시
   await purgeCache(`/api/products/${productId}`)
   await purgeCache(`/api/products/${productId}/stock`)
   
   // 라이브 상태 변경 시
   await purgeCache(`/api/streams/${streamId}`)
   await purgeCache(`/api/streams/${streamId}/status`)
   ```

4. **스마트 TTL 설정**
   - 라이브 목록: 10초 (빠른 갱신)
   - 상품 목록: 60초 (중간 갱신)
   - 상품 상세: 300초 (느린 갱신)

#### 검증 완료
- ✅ **재고 품절 시**: 캐시 purge 후 즉시 "품절" 표시
- ✅ **라이브 종료 시**: 캐시 purge 후 즉시 "종료됨" 표시
- ✅ **결제 완료 시**: 캐시 미적용으로 즉시 "결제 완료" 표시
