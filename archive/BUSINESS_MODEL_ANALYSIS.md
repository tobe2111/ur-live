# Your Live Commerce - 비즈니스 모델 분석

**날짜**: 2026-02-04  
**주제**: 인플루언서 중심 라이브 커머스 플랫폼 재설계

---

## 🎯 새로운 비즈니스 모델 이해

### 핵심 개념
**"판매자(Seller)" = "인플루언서(Influencer)"**

인플루언서가:
1. 자신의 라이브 커머스 예약 및 진행
2. 상품 등록 (자체 상품 또는 제휴 브랜드 상품)
3. 주문 처리 (직접 또는 브랜드에 전달)
4. 매출의 10% 수수료로 정산

### 플로우
```
인플루언서 → 라이브 커머스 예약
           ↓
         상품 등록
           ↓
      라이브 진행
           ↓
      고객 주문/결제
           ↓
인플루언서 주문 확인 → 처리 방식 선택:
  A) 직접 배송
  B) 제휴 브랜드에게 전달 → 브랜드가 배송
           ↓
     매일 정산서 발급
     (매출의 10% 수수료)
```

---

## 📊 현재 구현 상태 vs 필요한 것

### ✅ 이미 구현된 것 (그대로 사용)

#### 1. 인플루언서(Seller) 시스템 ✅
- **테이블**: `sellers` (이미 있음)
- **로그인**: `/api/auth/login` (seller 타입)
- **대시보드**: `/seller` 페이지
- **세션 관리**: `admin_sessions` 테이블

**결론**: ✅ **그대로 사용 가능** (seller = influencer)

#### 2. 라이브 스트리밍 예약 ✅
- **테이블**: `live_streams`
  - `scheduled_at` - 예약 시간
  - `status` - 'scheduled', 'live', 'ended'
  - `seller_id` - 인플루언서 ID
- **API**: 
  - `POST /api/admin/streams` (스트림 생성)
  - `GET /api/streams` (목록 조회)

**결론**: ✅ **그대로 사용 가능**

#### 3. 상품 관리 ✅
- **테이블**: `products`
  - `seller_id` - 인플루언서 ID
  - `live_stream_id` - 어느 라이브에서 판매할지
- **API**:
  - `PUT /api/seller/products/:id` (수정)
  - `DELETE /api/seller/products/:id` (삭제)

**결론**: ✅ **거의 완성**, 등록 API만 추가하면 됨

#### 4. 주문 시스템 ✅
- **테이블**: `orders`, `order_items`
  - `order_items.seller_id` - 인플루언서별 주문 분리
- **API**:
  - `GET /api/seller/orders` (주문 조회)
  - `POST /api/orders/create` (주문 생성)

**결론**: ✅ **그대로 사용 가능**

#### 5. 정산 시스템 ✅
- **테이블**: `settlements`
  - `seller_id` - 인플루언서 ID
  - `commission_rate` - 수수료율 (10%)
  - `period_start`, `period_end` - 정산 기간
  - `total_sales` - 총 매출
  - `commission` - 수수료
  - `settlement_amount` - 정산 금액 (매출 - 수수료)
- **API**:
  - `GET /api/seller/settlement-csv` (정산서 CSV 다운로드)

**결론**: ✅ **이미 완벽하게 구현됨!**

### 🔴 추가로 필요한 것

#### 1. 상품 등록 API ⭐⭐⭐ (필수)
**현재 상태**: 수정/삭제는 가능하지만 등록이 안 됨

**필요한 이유**:
```
인플루언서: "새 상품을 라이브에서 판매하고 싶어요!"
→ 상품 등록 API 필요
```

**구현**:
```typescript
POST /api/seller/products
Header: X-Session-Token: seller_1_xxx
Body: {
  name: "프리미엄 무선 이어폰",
  price: 129000,
  description: "노이즈 캔슬링",
  image_url: "https://...",
  stock: 100,
  category: "전자제품",
  live_stream_id: 2  // 어느 라이브에서 판매할지
}
```

#### 2. 인플루언서 상품 목록 API ⭐⭐⭐ (필수)
**현재 상태**: 모든 인플루언서 상품이 섞여서 보임

**필요한 이유**:
```
인플루언서 A: "내 상품만 보고 싶어요"
→ 자기 상품만 조회하는 API 필요
```

**구현**:
```typescript
GET /api/seller/products
Header: X-Session-Token: seller_1_xxx
Response: {
  success: true,
  data: [
    // 이 인플루언서가 등록한 상품만
    { id: 1, name: "내 상품 A", seller_id: 1, ... }
  ]
}
```

#### 3. 주문 상태 변경 API ⭐⭐⭐⭐⭐ (필수)
**현재 상태**: 주문 조회만 가능, 상태 변경 불가

**필요한 이유**:
```
시나리오 A (인플루언서가 직접 배송):
1. 주문 확인 → "준비중"으로 변경
2. 상품 포장 → "발송완료" + 송장번호 입력
3. 배송 완료 → "배송완료"

시나리오 B (브랜드에게 전달):
1. 주문 확인 → 브랜드에게 주문서 전달
2. 브랜드가 배송 → "발송완료" + 송장번호 입력
3. 배송 완료 → "배송완료"
```

**구현**:
```typescript
// 상태 변경
PUT /api/seller/orders/:orderNo/status
Body: { status: "shipped" }

// 송장번호 입력
PUT /api/seller/orders/:orderNo/tracking
Body: { 
  courier: "CJ대한통운",
  tracking_number: "1234567890"
}
```

#### 4. 라이브 스트림 예약 API ⭐⭐ (중요)
**현재 상태**: 관리자만 스트림 생성 가능

**필요한 이유**:
```
인플루언서: "다음 주 화요일 8시에 라이브 예약하고 싶어요"
→ 인플루언서가 직접 예약할 수 있는 API 필요
```

**구현**:
```typescript
POST /api/seller/streams
Header: X-Session-Token: seller_1_xxx
Body: {
  title: "🔥 겨울 패션 특집",
  description: "따뜻한 겨울 옷 소개",
  youtube_video_id: "dQw4w9WgXcQ",
  scheduled_at: "2026-02-10 20:00:00",
  status: "scheduled"
}
```
