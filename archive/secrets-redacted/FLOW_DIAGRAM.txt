┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                    1. 로그인 (3가지 경로)                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

[경로 A] 카카오 OAuth
    User clicks "카카오로 시작하기"
      ↓
    LoginPage.tsx → Kakao OAuth URL
      ↓
    Kakao Login Success → code
      ↓
    KakaoCallbackPage.tsx
      ├─ POST /api/auth/kakao/callback → customToken
      ├─ signInWithCustomToken(customToken)
      ├─ idToken = await user.getIdToken(false)  ✅ 캐시 사용
      ├─ useAuthStore.setAuth(user, idToken, '')  ✅ 토큰 저장
      └─ navigate(returnUrl)

[경로 B] onAuthStateChanged (자동 재인증)
    App.tsx → useAuthKR.initializeAuth()
      ↓
    onAuthStateChanged(firebaseUser)
      ├─ Check sessionStorage flag (중복 방지)
      ├─ idToken = await firebaseUser.getIdToken(false)  ✅ 캐시 사용
      ├─ useAuthStore.setAuth(user, idToken, '')  ✅ 토큰 저장
      └─ sessionStorage.setItem(`auth_processed_${uid}`, 'true')

[경로 C] firebase_token URL 파라미터 (딥링크)
    App.tsx → URL param detection
      ↓
    const firebaseToken = params.get('firebase_token')
      ├─ signInWithCustomToken(firebaseToken)
      ├─ idToken = await user.getIdToken(true)  ✅ 강제 갱신
      ├─ useAuthStore.setAuth(user, idToken, '')  ✅ 토큰 저장
      └─ window.history.replaceState() (URL 정리)

┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              2. API 요청 (Authorization 헤더 주입)                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    api.get('/api/cart')
      ↓
    Request Interceptor (api.ts Line 103-178)
      ├─ 공개 API? → Skip auth
      ├─ Seller/Admin API? → JWT token (localStorage)
      ├─ ✅ [우선 1] useAuthStore.accessToken 존재?
      │   └─ Yes → Authorization: Bearer {accessToken}
      └─ [우선 2] getCachedFirebaseToken() (Fallback)
          └─ Authorization: Bearer {firebaseToken}
      ↓
    Backend receives request with Authorization header ✅

┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                                3. Backend 인증 검증                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    GET /api/cart (with Authorization: Bearer {token})
      ↓
    requireAuth() Middleware (auth.ts Line 254-298)
      ├─ Extract token from Authorization header
      ├─ Try JWT verification (seller/admin)
      │   └─ Success → c.set('user', jwtPayload)
      └─ ✅ Try Firebase ID Token verification
          ├─ verifyFirebaseToken(token, projectId)
          │   ├─ Parse JWT (header, payload, signature)
          │   ├─ Fetch Google public keys (1hr cache)
          │   ├─ Verify RS256 signature (Web Crypto API)
          │   ├─ Check exp < now ✅
          │   ├─ Check iat <= now + 600 ✅
          │   ├─ Check iss === 'https://securetoken.google.com/{projectId}' ✅
          │   ├─ Check aud === projectId ✅
          │   └─ Check sub exists ✅
          └─ Success → c.set('user', { id: firebasePayload.sub, ... })
      ↓
    Cart Route Handler (cart.routes.ts Line 97-158)
      ├─ const user = getCurrentUser(c)
      ├─ const userId = await getUserDbId(db, user.id)
      ├─ SELECT * FROM cart WHERE user_id = ?
      └─ return { success: true, data: { items, summary } }
      ↓
    Response: 200 OK ✅

┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              6. 401 에러 자동 갱신 (선택적)                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    API Request → 401 Unauthorized
      ↓
    Response Interceptor (api.ts Line 181-264)
      ├─ Check originalRequest._retry === false
      ├─ Set _retry = true (무한 루프 방지)
      ├─ getCachedFirebaseToken(true)  ← forceRefresh
      │   └─ user.getIdToken(true) from Firebase
      ├─ Update Authorization header
      └─ Retry original request
      ↓
    Retry Success → 200 OK ✅
    OR
    Retry Failed → Clear auth → Redirect to /login
