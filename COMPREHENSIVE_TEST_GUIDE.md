## 🔐 Phase 2: Kakao 로그인 플로우 테스트 (3분)

### 목표
- Kakao 인증 연동 정상 작동 확인
- 세션 유지 확인
- 로그아웃 기능 확인

### 테스트 단계

**1. 메인 페이지 접속**
```
URL: https://live.ur-team.com
```

**2. 라이브 페이지 접속**
```
URL: https://live.ur-team.com/live/1
예상: YouTube Live 영상 재생, 상품 카드 표시
```

**3. 로그인 시도**
- 우측 상단 **로그인** 버튼 클릭
- Kakao 인증 페이지로 리다이렉트 확인

**4. Kakao 계정으로 로그인**
- Kakao 계정 입력 및 로그인
- 약관 동의 (최초 1회)
- 콜백 처리 확인

**5. 로그인 성공 확인**
```
✅ 우측 상단에 사용자 이름 표시
✅ 로그아웃 버튼 표시
✅ 브라우저 개발자 도구 → Application → Local Storage 확인:
   - userId
   - userName
   - session
```

**6. 로그아웃 테스트**
- 우측 상단 **로그아웃** 버튼 클릭
- 로그인 버튼으로 변경 확인
- Local Storage 세션 삭제 확인

**✅ 성공 기준**
- Kakao 로그인 및 콜백 정상 작동
- 세션 정보 Local Storage 저장
- 로그아웃 시 세션 정리

## 📺 Phase 3: 라이브 스트리밍 및 재고 표시 테스트 (3분)

### 목표
- YouTube Live 연동 확인
- 상품 정보 표시 확인
- **재고 표시 및 품절 처리 확인** ← 새로 추가됨!

### 테스트 단계

**1. 라이브 페이지 확인**
```
URL: https://live.ur-team.com/live/1
```

**2. YouTube 영상 재생 확인**
```
✅ YouTube Live 영상이 배경에서 재생
✅ 전체 화면 배경 모드
✅ 음소거 해제 시 소리 재생
```

**3. 상품 카드 표시 확인**
```
✅ 하단에 상품 카드 표시
✅ 상품 이미지
✅ 상품명
✅ 가격
✅ 재고 수량 ← 새로 추가!
```

**4. 재고 표시 테스트 (NEW!)**

**Case A: 정상 재고 (재고 > 5)**
```
예상 표시: "재고 XX개" (회색)
버튼 상태: "담기" 버튼 활성화 (파란색)
```

**Case B: 재고 부족 (재고 ≤ 5)**
```
예상 표시: "재고 X개 남음" (주황색)
버튼 상태: "담기" 버튼 활성화 (파란색)
경고 메시지: 시각적 경고
```

**Case C: 품절 (재고 = 0)**
```
예상 표시: "품절" (빨간색)
버튼 상태: "품절" 버튼 비활성화 (회색)
클릭 시: "품절된 상품입니다" 알림
```

**5. 브라우저 개발자 도구 콘솔 확인**
```javascript
// 예상 로그:
✅ 상품 로드 완료: { id, name, price, stock }
✅ 재고 정보: stock 필드 확인
```

**✅ 성공 기준**
- YouTube Live 정상 재생
- 상품 정보 정확히 표시
- 재고 수량 실시간 표시
- 재고 상태별 색상 구분 (회색/주황색/빨간색)
- 품절 시 버튼 비활성화

## 🛒 Phase 4: 장바구니 기능 테스트 (5분)

### 목표
- 장바구니 담기 기능 확인
- **재고 기반 담기 차단 확인** ← 새로 추가됨!
- 장바구니 조회 확인

### 테스트 단계

**1. 로그인 상태 확인**
```
없으면: 로그인 후 진행
```

**2. 정상 재고 상품 담기**
- **"담기"** 버튼 클릭
```
✅ "장바구니에 추가되었습니다!" 알림
✅ 재고 수량 1 감소 (예: 10개 → 9개)
✅ 재고 부족 구간 진입 시 색상 변경
```

**3. 재고 부족 경고 확인**
- 재고 5개 이하까지 담기 시도
```
✅ 주황색 경고 표시: "재고 X개 남음"
✅ 버튼 여전히 활성화
```

**4. 품절 처리 확인**
- 마지막 1개까지 담기
```
✅ 재고 0개로 변경
✅ 빨간색 표시: "품절"
✅ 버튼 비활성화: "품절"
✅ 클릭 시: "품절된 상품입니다" 알림
```

**5. 재고 부족 에러 테스트 (동시 주문 시나리오)**
- 브라우저 개발자 도구 → Console
```javascript
// 강제로 재고 부족 상황 시뮬레이션
// 백엔드가 재고 부족 응답을 보내면:
✅ "재고가 부족합니다. 다시 시도해주세요." 알림
✅ 상품 정보 자동 재로드
✅ 최신 재고 수량 표시
```

**6. 장바구니 페이지 이동**
- **"결제"** 버튼 클릭
```
URL: https://live.ur-team.com/cart
```

**7. 장바구니 내용 확인**
```
✅ 담은 상품들 목록 표시
✅ 각 상품: 이미지, 이름, 가격, 수량
✅ 소계 금액
✅ 배송비: 3,000원
✅ 총 결제 금액
```

**✅ 성공 기준**
- 장바구니 담기 성공
- 재고 실시간 감소 반영
- 재고 부족/품절 시각적 경고
- 품절 시 담기 차단
- 재고 부족 에러 시 자동 재로드
- 장바구니 페이지 정상 표시

## 💳 Phase 5: 결제 플로우 테스트 (7분)

### 목표
- NicePay 결제 연동 확인
- 주문 생성 및 재고 차감 확인
- 결제 완료 처리 확인

### 테스트 단계

**1. 배송지 정보 입력**
```
받는 사람: 홍길동
전화번호: 010-1234-5678
주소: 서울시 강남구 테헤란로 123
상세주소: 101동 1001호
우편번호: 06234
```

**2. 결제하기 버튼 클릭**
```
✅ 주문 생성 API 호출 (POST /api/orders)
✅ 브라우저 개발자 도구 → Network 탭에서 확인:
   - 응답: { success: true, data: { orderId, orderNumber, totalAmount } }
```

**3. NicePay 결제창 팝업 확인**
```
✅ 새 창으로 NicePay 결제창 열림
✅ 주문 정보 표시:
   - 주문번호: ORD...
   - 상품명: (상품명)
   - 결제 금액: XX,XXX원
```

**4. 테스트 카드로 결제**
```
카드번호: 5465-7801-2345-6789
유효기간: 12/25
CVV: 123
생년월일: 900101
비밀번호 앞 2자리: 00
```

**5. 3D Secure 인증 (필요 시)**
```
✅ 인증 화면 진행
✅ 인증 완료
```

**6. 결제 결과 페이지 확인**
```
URL: https://live.ur-team.com/payment-result?resultCode=0000&...

✅ 브라우저 개발자 도구 → Console 확인:
   - resultCode: 0000
   - TID: nictest00m...
   - Amount: 결제금액
```

**7. 서버 승인 API 호출 확인**
```
✅ Network 탭에서 POST /api/payments/nicepay/callback 확인
✅ 응답: { success: true, ... }
```

**8. 주문 완료 페이지 리다이렉트**
```
URL: https://live.ur-team.com/order-complete?orderNo=ORD...

✅ 주문 번호 표시
✅ 주문 내역 보기 버튼
```

**9. 재고 차감 확인 (중요!)**
- 다시 라이브 페이지로 이동: https://live.ur-team.com/live/1
```
✅ 주문한 상품의 재고 감소 확인
✅ 예: 결제 전 9개 → 결제 후 8개
```

**✅ 성공 기준**
- 주문 생성 성공
- NicePay 결제창 정상 작동
- 테스트 결제 완료
- 서버 승인 API 성공
- 주문 완료 페이지 표시
- **재고 정확히 차감됨**

## 🔄 Phase 7: 취소/환불 플로우 테스트 (5분)

### 목표
- 주문 취소 기능 확인
- 환불 처리 확인
- **재고 복구 확인** ← 중요!

### 테스트 단계

**1. 취소 가능한 주문 확인**
```
조건:
- 결제 상태: 승인 (completed)
- 배송 상태: 배송 전 (pending)
- 취소 여부: 취소되지 않음
```

**2. 주문 취소 시도**
- 주문 상세 모달에서 **"주문 취소"** 버튼 클릭
```
✅ 확인 팝업: "주문을 취소하시겠습니까?"
```

**3. 취소 확인**
- **"확인"** 버튼 클릭
```
✅ API 호출: POST /api/orders/{orderNumber}/refund
✅ 브라우저 개발자 도구 → Network 탭에서 확인:
   - 요청: { reason: "단순 변심", cancelAmt: 금액 }
   - 응답: { success: true, data: { refund_amount, refund_tid, status: "cancelled" } }
```

**4. 취소 완료 메시지**
```
✅ "주문 취소가 완료되었습니다" 알림
✅ "환불은 영업일 기준 3-5일 소요됩니다" 안내
✅ 모달 자동 닫힘
✅ 주문 목록 새로고침
```

**5. 취소된 주문 상태 확인**
```
✅ 주문 카드에 "취소됨" 상태 표시
✅ 상세 보기 클릭 시:
   - 결제 상태: 취소됨
   - 취소 사유: 단순 변심
   - 환불 금액: XX,XXX원
   - 주문 취소 버튼 비활성화
   - "이미 취소된 주문입니다" 메시지
```

**6. 재고 복구 확인 (중요!)**
- 라이브 페이지로 이동: https://live.ur-team.com/live/1
```
✅ 취소한 상품의 재고 증가 확인
✅ 예: 취소 전 8개 → 취소 후 9개
✅ 품절이었다면: "품절" → "재고 X개" + 버튼 활성화
```

**7. 취소 불가 케이스 테스트**

**Case A: 배송 시작된 주문**
```
✅ 상세 보기 모달:
   - "배송이 시작되어 취소할 수 없습니다" 메시지 (회색)
   - 주문 취소 버튼 비활성화 또는 숨김
```

**Case B: 이미 취소된 주문**
```
✅ 상세 보기 모달:
   - "이미 취소된 주문입니다" 메시지
   - 주문 취소 버튼 비활성화
```

**✅ 성공 기준**
- 주문 취소 정상 처리
- NicePay 환불 API 성공
- 주문 상태 "취소됨"으로 변경
- **재고 정확히 복구됨**
- 취소 불가 케이스 적절히 처리
