# 🎨 CheckoutPage 디자인 리뉴얼 프롬프트

## 📋 프로젝트 정보
- **페이지**: 주문/결제 페이지 (CheckoutPage)
- **플랫폼**: 라이브 커머스 웹사이트 (https://live.ur-team.com/checkout)
- **기술 스택**: React + TypeScript + TailwindCSS
- **결제 시스템**: 토스페이먼츠 위젯 (SDK 통합)

---

## 🎯 디자인 요구사항

### 1. **전체 레이아웃**
```
현재 구조:
┌─────────────────────────────────────────┐
│ 헤더 (← 장바구니로 돌아가기 | 주문/결제) │
├─────────────────┬───────────────────────┤
│                 │                       │
│   좌측 (2/3)    │    우측 (1/3)       │
│                 │                       │
│ 1. 배송지 정보  │  결제 금액 요약      │
│ 2. 주문 상품    │  - 상품 금액         │
│ 3. 결제 수단    │  - 배송비            │
│    (토스위젯)   │  - 총 결제 금액      │
│                 │  [결제하기 버튼]     │
└─────────────────┴───────────────────────┘
```

**개선 방향:**
- 모바일 우선 반응형 디자인
- 좌우 레이아웃 유지 (PC), 상하 스크롤 (모바일)
- 토스페이먼츠 위젯 영역 절대 수정 금지

---

### 2. **컴포넌트별 디자인 가이드**

#### A. 헤더 영역
```
현재:
[← 장바구니로 돌아가기]
주문/결제

요구사항:
- 뒤로가기 버튼 명확하게
- 제목 강조 (폰트 크기, 굵기)
- 진행 단계 표시 추가 (선택사항)
  예: 장바구니 → [결제] → 완료
```

#### B. 배송지 정보 카드
```
현재:
┌──────────────────────────────┐
│ 📍 배송지 정보    [변경하기] │
├──────────────────────────────┤
│ 홍길동                        │
│ 010-1234-5678                │
│ 서울시 강남구 테헤란로 123   │
│ (우) 06234                   │
└──────────────────────────────┘

요구사항:
- 카드 디자인 개선 (섀도우, 라운드 코너)
- 아이콘 추가 (📍 위치, 📱 전화, 🏠 주소)
- 기본 배송지 뱃지 추가
- 변경하기 버튼 우측 상단 고정
- 배송지 미선택 시 경고 메시지
```

#### C. 주문 상품 목록
```
현재:
┌──────────────────────────────┐
│ 📦 주문 상품 (N개)           │
├──────────────────────────────┤
│ [판매자 그룹별]              │
│ ┌─────────────────────────┐ │
│ │ 판매자A                  │ │
│ │ - 상품1  10,000원 × 2   │ │
│ │ - 상품2  20,000원 × 1   │ │
│ │ 배송비: 3,000원          │ │
│ └─────────────────────────┘ │
└──────────────────────────────┘

요구사항:
- 판매자별 그룹핑 명확하게
- 상품 썸네일 이미지 추가
- 수량 표시 뱃지
- 무료배송 조건 표시 (예: 1,000원 추가 시 무료배송)
- 배송비 무료 시 초록색 뱃지
```

#### D. 결제 수단 (토스페이먼츠 위젯)
```
🚨 중요: 이 영역은 절대 수정 금지!
┌──────────────────────────────┐
│ 💳 결제 수단                 │
├──────────────────────────────┤
│ [토스페이먼츠 위젯 영역]     │
│ - 카드 결제                   │
│ - 계좌이체                    │
│ - 간편결제 등                 │
│ (자동 렌더링)                 │
└──────────────────────────────┘

요구사항:
- 위젯 영역 주변 여백만 조정 가능
- 로딩 상태 UI 개선
- "결제 수단 불러오는 중..." 메시지 디자인
```

#### E. 결제 금액 요약 (우측 사이드바)
```
현재:
┌──────────────────────────┐
│ 결제 금액                │
├──────────────────────────┤
│ 상품 금액    40,000원   │
│ 배송비        3,000원   │
│ ────────────────────────│
│ 총 결제 금액  43,000원  │
│                          │
│ [결제하기 버튼]          │
└──────────────────────────┘

요구사항:
- 금액 숫자 강조 (폰트 크기, 굵기)
- 총 결제 금액 색상 강조 (파란색 또는 브랜드 컬러)
- 할인 금액 있을 시 빨간색 표시
- 결제하기 버튼 크게, 눈에 띄게
- 버튼 로딩 상태 애니메이션
- PC에서 sticky 고정 (스크롤 시 따라옴)
```

### 3. **색상 팔레트**
```css
Primary: #3B82F6 (파란색 - 브랜드 컬러)
Success: #10B981 (초록색 - 무료배송, 성공)
Warning: #F59E0B (주황색 - 경고)
Error: #EF4444 (빨간색 - 에러, 할인)
Gray: #6B7280 (회색 - 부가 정보)
Background: #F9FAFB (밝은 회색 - 배경)
White: #FFFFFF (흰색 - 카드 배경)
```

---

### 4. **타이포그래피**
```
제목 (h1): 24px~32px, font-bold
부제목 (h2): 18px~24px, font-bold
본문: 14px~16px, font-normal
보조: 12px~14px, text-gray-600
강조: font-semibold, text-blue-600
```

---

### 5. **반응형 디자인**
```
모바일 (<640px):
- 상하 스크롤
- 결제 금액 요약 하단 고정 (sticky bottom)
- 버튼 화면 하단 고정
- 카드 간격 좁게

태블릿 (640px~1024px):
- 좌우 레이아웃 시작
- 여백 증가

데스크톱 (>1024px):
- 좌측 2/3, 우측 1/3 레이아웃
- 우측 사이드바 sticky
- 최대 너비 1280px
```

### 6. **애니메이션 & 인터랙션**

- 로딩 스피너 회전 애니메이션
- 에러 메시지 페이드인
- 모달 슬라이드업

2. **기능적 요소**
   - 결제하기 버튼 로직
   - API 호출 코드

   - 상품 그룹핑 로직

```html
<div class="min-h-screen bg-gray-50">

    <!-- 헤더 -->

      <button>← 장바구니</button>
      <h1>주문/결제</h1>

    </header>

    <!-- 메인 콘텐츠 -->
    <div class="grid lg:grid-cols-3 gap-6">
      <!-- 좌측: 주문 정보 (2칸) -->

        <!-- 1. 배송지 정보 카드 -->
        <section class="bg-white rounded-lg p-6 shadow-sm">
          <h2>📍 배송지 정보</h2>
          <div>수취인, 연락처, 주소</div>
          <button>변경하기</button>
        </section>

        <!-- 2. 주문 상품 카드 -->
        <section class="bg-white rounded-lg p-6 shadow-sm">
          <h2>📦 주문 상품</h2>
          <div>판매자별 그룹</div>
        </section>

        <!-- 3. 결제 수단 카드 -->
        <section class="bg-white rounded-lg p-6 shadow-sm">
          <h2>💳 결제 수단</h2>
          <!-- 🚨 토스페이먼츠 위젯 - 수정 금지 -->
          <div id="payment-method" style="min-height: 350px"></div>
          <div id="agreement"></div>
        </section>
      </div>

      <!-- 우측: 결제 요약 (1칸, sticky) -->
      <aside class="lg:col-span-1">
        <div class="bg-white rounded-lg p-6 shadow-sm lg:sticky lg:top-6">
          <h2>결제 금액</h2>
          <div>금액 상세</div>
          <button class="w-full py-4 bg-blue-600 text-white">
            결제하기
          </button>
        </div>
      </aside>
    </div>
  </div>
</div>
```

## 📝 AI에게 요청할 때 프롬프트 예시

### 🔥 추천 프롬프트

```
주문/결제 페이지 디자인을 리뉴얼해줘.

**현재 페이지**: https://live.ur-team.com/checkout
**플랫폼**: 라이브 커머스 웹사이트
**기술**: React + TypeScript + TailwindCSS

**페이지 구조**:
1. 헤더 (← 뒤로가기, 제목)
2. 좌측 (2/3): 배송지 정보, 주문 상품, 결제 수단
3. 우측 (1/3): 결제 금액 요약, 결제하기 버튼

**디자인 요구사항**:
- 모던하고 깔끔한 디자인
- 모바일 우선 반응형
- 카드 기반 레이아웃 (shadow-sm, rounded-lg)
- 파란색 계열 브랜드 컬러 (#3B82F6)
- 아이콘 활용 (lucide-react)
- 금액 숫자 강조 (크고 굵게)
- 결제하기 버튼 눈에 띄게

**중요**: 토스페이먼츠 위젯 영역(<div id="payment-method">)은 절대 수정하지 마. 
이 영역은 외부 SDK가 자동으로 렌더링함.

**참고 디자인**: 쿠팡, 마켓컬리, 토스의 결제 페이지 스타일

TailwindCSS 클래스만 사용해서 JSX 코드 전체를 작성해줘.
```
